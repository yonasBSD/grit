   is never a candidate.
2. These 3 tests clone `$HTTPD_URL/smart/repo2.git` with `BUNDLE_URI_PROTOCOL` UNSET, so the
   server-side git-upload-pack wrapper routes `--advertise-refs` to REAL_UP = Apple Git.
3. Direct proof (built bare repo, set uploadpack.advertiseBundleURIs=true + bundle.*):
   `GIT_PROTOCOL=version=2 /usr/bin/git -c uploadpack.advertiseBundleURIs=true upload-pack --advertise-refs`
   advertises `version 2 / ls-refs=unborn / fetch / object-format` but **NOT bundle-uri**
   (capability added upstream in 2.41+). Same command on homebrew 2.52.0 DOES emit `bundle-uri`.
4. grit client correctly gates discovery on `server_advertises_bundle_uri(&caps)`:
   - `grit/src/file_upload_pack_v2.rs:162-165` (faithful cap check)
   - `grit/src/http_smart.rs:1585-1587` (HTTP clone path — the one these tests exercise)
   → no cap advertised → no git-remote-https child spawned → the `child_start` trace2 the
   tests grep for never fires, exactly as upstream git would behave against the same server.
5. The 3 ported tests are **byte-identical** to `git/t/t5601-clone.sh` (diff = IDENTICAL),
   so this is not a porting bug.
