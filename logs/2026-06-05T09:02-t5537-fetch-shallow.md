## Root causes and fixes

### Subtest 12 — `rm -rf repo-with-sub` race ("Directory not empty")
After a fetch, grit runs `maintenance run --auto --detach`. The detached
maintenance grandchild inherited the fetch's cwd (the repo working tree), so a
live process held that directory as its cwd. On macOS `rm -rf` then fails with
"Directory not empty" — the test's `test_when_finished "rm -rf repo-with-sub"`
cleanup races the background child. Reproduced 16/20 trials.
Upstream avoids this by running maintenance in the foreground under the test
harness (`GIT_TEST_MAINT_AUTO_DETACH=false`), which our ported test-lib.sh does
not set (and which is off-limits to edit).
Fix (grit/src/commands/maintenance.rs): spawn the detached maintenance child
with cwd set to filesystem root and pass absolute GIT_DIR/GIT_WORK_TREE so it
rediscovers the repo. It no longer holds the repo dir; 0/25 trials fail after.

### Subtest 16 — shallow connectivity over HTTP (protocol v2)
Four distinct bugs, fixed in order:

1. test_httpd one-time-script consumed only on success.
   grit/src/bin/test_httpd.rs aborted the connection when the one-time-script
   exited non-zero. Upstream lib-httpd/apply-one-time-script.sh ignores the
   exit status and only compares output: if it differs, serve it and remove
   ("consume") the script. The test's deliberately broken `sed` exits 1 with
   empty stdout; upstream still consumes the script and serves the empty
   (truncated) response. Fixed to mirror upstream (ignore exit status).

2. Shallow pack omitted blobs shared with the cut-off history.
   upload_pack.rs / serve_v2.rs built the depth pack with
   `pack-objects --not <boundary-parent>`, which drops every object reachable
   from the excluded history — including blobs still referenced by an in-depth
   tree (a file added in the first commit and never changed). The shallow repo
   then failed `git fsck` with a missing blob. Fixed by cutting the boundary
   commits' parent chains with `pack-objects --shallow <oid>` instead
   (new compute_depth_boundary_commits + spawn_pack_objects_upload_shallow +
   write_pack_objects_revs_stdin_shallow in pack_objects_upload.rs).

3. shallow-info advertised no `shallow` line (off-by-one).
   shallow_grafts_for_upload_pack_deepen used base = min_hit.unwrap_or(1) for a
   fresh client, over-counting by one so the border landed on the root commit
   (no parents) and no `shallow` line was sent. Fixed to unwrap_or(0); the
   already-shallow client path still uses the real distance.

4. Client consumed the packfile as shallow-info (the decisive bug).
   http_smart.rs read_shallow_info_section looped until a flush, treating the
   v2 section-delimiter `0001` as a no-op. In a v2 fetch response sections are
   separated by a delim, so it swallowed the `packfile` header and the entire
   pack — the fetched objects were silently never stored, leaving the ref
   pointing at a missing object. Fixed to stop at the delim.
