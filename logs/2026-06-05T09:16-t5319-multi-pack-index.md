## Root causes found
1. MIDX written as version 1; upstream Git now writes **version 2** by default
   (`.version = MIDX_VERSION_V2` in git/midx-write.c). The `test-tool read-midx`
   header line (`midx_read_expect`) expects `... 2 <hashlen> ...`. Fixed:
   - `build_midx_bytes` now writes `MIDX_VERSION_V2`.
   - `parse_midx_header` accepts V1 and V2.
   - `format_midx_dump` prints the raw hash length (20 for SHA-1) for the
     `hash_len` field instead of the on-disk hash-version byte.
2. Non-incremental MIDX writes left a spurious empty `multi-pack-index.d/`
   directory. Now removed on the non-incremental path.
3. `multi-pack-index verify` only checked the header. Implemented a full
   `verify_midx()` in grit-lib/src/midx.rs mirroring `verify_midx_file` +
   `load_multi_pack_index` checks (chunk TOC, fanout order, oid lookup order,
   pack-int-id range, object offsets vs pack index, checksum) with the exact
   Git error strings the corruption tests grep for.
4. `--progress` for write/verify/repack/expire now emits a forced progress line
   to stderr (tests only require non-empty stderr under GIT_PROGRESS_DELAY=0).
