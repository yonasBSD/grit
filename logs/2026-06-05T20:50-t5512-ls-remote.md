## Root causes & fixes

### CLI Args (grit/src/commands/ls_remote.rs, grit/src/main.rs)
- Added `--branches`/`-b` (canonical) with `--heads`/`-h` as a hidden deprecated
  synonym. clap reserves `-h`, so `preprocess_ls_remote_argv` in main.rs rewrites
  `-h`->`--heads` and `-b`->`--branches` before parsing. The sole-`-h` help case
  is still intercepted earlier (exit 129), so reaching preprocessing always means
  `-h` is the branches flag.
- Made `<repository>` optional; added `--sort`, `--exit-code`, `--get-url`.

### Default remote resolution
- `resolve_default_remote()` mirrors git's `remote_get(NULL)`:
  branch.<cur>.remote -> sole remote -> "origin"; the chosen remote must have a
  URL or it dies "No remote configured to list refs from."

### From header / quiet
- Print `From <url>` to stderr only when no `<repository>` was given and not
  `-q`. Removed the wrong behaviour where `-q` suppressed ref output entirely
  (git's quiet only hides the URL line).

### hiderefs
- `apply_hiderefs` combines `transfer.hiderefs` + `uploadpack.hiderefs` in
  declaration order; `!`-prefixed rules un-hide; last match wins. Peeled `^{}`
  entries follow their base ref.

### sorting
- `--sort` supports `refname` and `version:refname` (with `-` reverse). Other
  fields need object data; outside a repo they fail with the exact git message
  `fatal: not a git repository, but the field '<f>' requires access to object
  data`. Validated up front in run().

### symref filtering
- lib `Options` gained `all_symrefs`: protocol v2 advertises symref-target for
  all symbolic refs; v0 only HEAD. The CLI sets it from the effective protocol
  version for the local path.
- v0 upload-pack path: `symref_map_from_first_line` collects every `symref=X:Y`
  capability so non-HEAD symrefs (e.g. refs/remotes/origin/HEAD) are shown
  (test 39).

### misc
- `--branches`/`--tags` are now a UNION in the lib filter and v0/v2 parsers
  (was an intersection bug; test 38).
- Pattern matching rewritten to git's `tail_match` semantics: `*/<pattern>`
  wildmatch against `/<refname>` (grit-lib/src/ls_remote.rs).
- `capabilities^{}` pseudo-ref (null OID) skipped so a standards-compliant empty
  remote yields no refs and `--exit-code` returns 2 (test 34).
- Unreadable local dest emits git's two-line
  "'<dest>' does not appear to be a git repository" /
  "Could not read from remote repository." message, exit 128 (test 19).
