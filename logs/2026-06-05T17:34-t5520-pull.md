## Fixes applied (grit/src/commands/pull.rs, merge.rs, main.rs)

1. **Reflog action** (tests 8, 9, 21, 22): `git pull` now sets `GIT_REFLOG_ACTION`
   to `pull <args...>` (in main.rs dispatch, only if not already set), matching
   `builtin/pull.c set_reflog_message`. The downstream merge/rebase then record
   e.g. `pull --rebase . ff: Fast-forward` in the reflog.

2. **Autostash passthrough** (tests 23, 24, 29-52): added `--autostash` /
   `--no-autostash` to pull's Args and a tri-state `resolve_pull_autostash`
   (CLI wins, then `pull.autostash`, else Unset). Threaded into both merge and
   rebase calls. Added `no_autostash` field to merge::Args so pull can force
   stash off even when `merge.autostash=true`.

3. **Rebase flavor** (tests 55, 56, 59-67): `--rebase=merges|m` -> `--rebase-merges`,
   `--rebase=interactive|i` -> `--interactive` (new `PullRebaseKind`).
   `rebase_cli_value_is_valid` and `parse_rebase_value` now accept `m`/`i`.
   `--verify-signatures` with rebase now warns `ignoring --verify-signatures for rebase`.
