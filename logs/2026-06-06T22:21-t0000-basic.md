## Root cause
Subtest 80 sets up an index via `read-tree $tree` (no `-u`), which leaves the
index entries' stat fields zeroed while the worktree still holds the original
files. `git diff-files` must then report every entry as `M` with a null OID on
the worktree side, because the stat tuple disagrees with the index and Git never
re-hashes to clear the change.

grit reported the regular files correctly but silently dropped all symlink
entries (`path0sym`, `path2/file2sym`, ‚Ä¶), so diff-files emitted 4 lines instead
of the expected 8.

The bug was in `read_worktree_info_fast` in
`grit/src/commands/diff_files.rs`. The symlink branch (reached only when lstat
did NOT match the index) re-hashed the symlink target and returned
`WorktreeStatus::Unchanged` whenever the content matched the index OID:

```rust
if meta.file_type().is_symlink() {
    let target = fs::read_link(abs_path)?;
    let oid = Odb::hash_object_data(ObjectKind::Blob, target.as_os_str().as_bytes());
    if oid == index_entry.oid && canonicalize_mode(index_entry.mode) == MODE_SYMLINK {
        return Ok(WorktreeStatus::Unchanged);   // <-- bug: hides stat-dirty symlinks
    }
    return Ok(WorktreeStatus::Modified(MODE_SYMLINK, oid));
}
```

Regular files in the same function do NOT shortcut to `Unchanged` on a content
match; they return `Modified` and let `collect_changes` decide. There the entry
is suppressed only when `content_matches && index_stat_is_trusted && stat_agrees`
‚Äî i.e. only when the on-disk stat is both populated and matches the index. For a
read-tree-zeroed entry that check is false, so the file is correctly reported as
`M`.

## Fix
Removed the content-based `Unchanged` shortcut in the symlink branch so a
stat-dirty symlink returns `Modified(MODE_SYMLINK, oid)`, mirroring the regular
file path. `collect_changes` then applies the identical `index_stat_is_trusted`
/ `stat_agrees` suppression logic. The clean-stat symlink case (lstat matches,
not racy) is still short-circuited to `Unchanged` earlier in the function, and
the racy-clean case is suppressed downstream ‚Äî so `update-index --refresh`
followed by `diff-files` still yields an empty diff (subtest 81 unaffected).
