## Failure #24 "show" — CONFIRMED unfixable within allowed scope (test-lib.sh artifact)

Diff is in the `two` remote's push status:

```
-    ahead forces to main    (fast-forwardable)
-    main  pushes to another (up to date)
+    ahead forces to main    (local out of date)
+    main  pushes to another (local out of date)
```

Root cause: `git remote show two` classifies push status by checking whether the local
object store contains the remote's old OID (`classify_push` in
`grit/src/commands/remote.rs`, mirroring upstream `get_push_ref_states` /
`odb_has_object`). For `fast-forwardable`/`up to date`, the local repo must already hold
`two`'s `main` object.

In **upstream Git**, `setup_repository one` and `setup_repository two` produce
**byte-identical "Initial" commits** (same content, same `test_tick` timestamps), so they
share the same OID — and the clone of `one` therefore already has `two`'s main object.
Verified with real git 2.52.0: both setups yield the same `536d935...` initial SHA.

The grit harness's `tests/test-lib.sh` adds **file persistence** for `test_tick`
(`_TICK_FILE = $TRASH_DIRECTORY/.git/.test_tick`) that upstream does not have. That leaks
the advanced tick counter from the `setup_repository one` subshell into the
`setup_repository two` subshell, so `two`'s Initial gets a later timestamp → different OID
(`de74b1a` vs `9d34b14`). Hence the clone lacks `two`'s main object → grit correctly
reports `local out of date`. Real git run *inside this same harness* would produce the same
"wrong" output. Fixing requires editing `tests/test-lib.sh` (forbidden). Confirmed prior
agent's analysis. **grit code is correct here.**

## Failure #130 "rename moves around the reflog" — root cause = `for-each-reflog-ent` order, blocked by a mis-ported test file

`test-tool ref-store main for-each-reflog-ent <ref>` in grit (`grit/src/main.rs`, ~line
1085) emits reflog entries **newest-first**. Upstream `files_for_each_reflog_ent`
(`git/refs/files-backend.c:2341`) streams the reflog file **front-to-back = oldest-first**;
only `for-each-reflog-ent-reverse` is newest-first. grit's read direction is inverted for
BOTH variants (it reverses the plain one and leaves `-reverse` un-reversed).

The one-line fix is to swap the reverse condition (`if sub == "for-each-reflog-ent-reverse"`
instead of `"for-each-reflog-ent"`). With that fix:

- **t5505 #130 PASSES** → 128/130.
- t1400 #"verifying log" blocks (oldest-first heredocs), t1405 #10, t1406 #11 — all expect
  upstream oldest-first and would move toward correct.
- **t1421 #7 "simple writes" REGRESSES** (10/10 → 9/10).
