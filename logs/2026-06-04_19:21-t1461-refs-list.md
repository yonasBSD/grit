# t1461-refs-list

Ticket: 6149f6

Initial state: 417/428. Failing subtests covered `*-local` ref-filter date
formats and several invalid atom-argument diagnostics.

## Findings

- The copied harness leaves `TZ` unset even though upstream `test-lib.sh`
  exports `TZ=UTC`. `t1461` exports fixed `GIT_AUTHOR_DATE` and
  `GIT_COMMITTER_DATE`, so local date modes used the machine timezone instead
  of the upstream test default.
- `compute_is_base_winners` scanned candidate refs even when the requested
  format and sort keys contained no `%(is-base:...)` atom. That eager scan
  emitted unrelated broken-object diagnostics before later format validation
  errors.

## Changes

- Default `TZ` to `UTC` for `for-each-ref` / `refs list` only in the copied
  test-shaped environment where fixed Git date variables are present and `TZ`
  is absent.
- Return early from `compute_is_base_winners` when there are no is-base targets.
