## Root cause
grit's local push fast path (`grit/src/commands/push.rs`) had no shallow-push
handling at all:
- It never rejected a push that introduced a *new* shallow root into a
  non-shallow receiver ("shallow update not allowed").
- `unpack-objects --strict` on the receiver required the parents of shallow
  boundary commits, so pushing from a shallow clone failed the strict
  connectivity walk (`missing parent ... referenced by commit`).
- The receiver's `.git/shallow` was never updated when `receive.shallowupdate`
  was set, so later `git fsck` walked past a graft and failed
  (`object not found: <parent>`).

This mirrors upstream `builtin/receive-pack.c` (`update_shallow_info`,
`update_shallow_ref`, `unpack` passing `--shallow-file`) and `shallow.c`
(`assign_shallow_commits_to_refs`).

## Fix
- `grit-lib/src/shallow.rs`:
  - `add_shallow_boundaries` â€” union new grafts into `.git/shallow`.
  - `new_shallow_roots_for_push` â€” walk a pushed tip through the source ODB,
    cutting at the receiver's existing tips and at graft commits; returns graft
    commits the receiver does not already reach (new shallow roots). + unit tests.
- `grit-lib/src/unpack_objects.rs`: `UnpackOptions.shallow_boundaries`; strict
  connectivity walk skips parent checks for boundary commits.
- `grit/src/commands/unpack_objects.rs`: `--shallow-file <file>` reads boundary
  OIDs into the strict walk (mirrors receive-pack).
- `grit/src/receive_ingest.rs`: `ingest_received_pack_with_shallow` writes a temp
  shallow file and passes `--shallow-file` to unpack-objects.
- `grit/src/commands/push.rs`: when the pushing repo is shallow, compute new
  shallow roots per ref. If `receive.shallowupdate` is false -> per-ref reject
  with `(shallow update not allowed)`; if true -> pass boundaries to the strict
  unpack and append the new grafts to the receiver's `.git/shallow`.
- Added `..Default::default()` to the three other `UnpackOptions` literals
  (`bundle_uri.rs`, `commands/bundle.rs`, `commands/clone.rs`) for the new field.
