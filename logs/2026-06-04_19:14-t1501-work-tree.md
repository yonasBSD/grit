# t1501-work-tree

Ticket: 9d4618

Initial state: 36/39. Failing subtests covered relative `GIT_DIR`/`GIT_WORK_TREE`,
`diff-index` raw output under a `.git` directory work tree, and an absolute
pathspec outside the repository.

## Findings

- `git diff` resolved cwd-relative pathspecs into repo-relative paths, but then
  filtered entries with the original raw argv tokens. From
  `repo.git/work/sub/dir`, `tracked` therefore filtered against `tracked`
  instead of `sub/dir/tracked`.
- `diff_tree_vs_worktree` refreshes staged additions from the live work tree for
  rename and patch rendering. Raw uncached `diff-index` still needs an all-zero
  new object ID when that side represents live worktree content rather than the
  index entry.
- `git log` validated only clap-collected pathspecs before combining the raw
  `--` tail. `git log HEAD -- /home` therefore skipped the worktree-scope check
  and exited successfully.

## Changes

- Filter `git diff` entries with the resolved pathspec list.
- Teach raw uncached `diff-index` rendering to print an all-zero new OID when an
  added entry's live worktree OID differs from the index OID.
- Validate `git log` pathspec scope after combining explicit `--` pathspecs,
  implied pathspecs, and stdin paths, before resolving them.
