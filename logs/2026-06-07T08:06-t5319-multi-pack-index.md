### 77/78/81 root cause (pack-objects, NOT MIDX)
`select_packs_for_batch` (grit/src/commands/multi_pack_index.rs:561) is byte-faithful
to git `fill_included_packs_batch` (git/midx-write.c:2001): same mtime sort, same
`expected = (referenced<<14)/num_objects * pack_size`, rounded, same `expected >= batch`
skip and `total < batch` loop guard. Per-pack MIDX referenced counts match git
(A=24,B=15,C=9,D=6,E=7; non-overlapping so referenced==num_objects).

The divergence is purely pack BYTE SIZE. Test 74 sets mtimes D=-5,C=-4,B=-3,A=-2,E=newest,
then BATCH = THIRD_SMALLEST_SIZE + 1, computed from grit's OWN sizes via
`test-tool path-utils file-size`. Fill order D,C,B,A,E:
- git: THIRD=1250(B), BATCH=1251. D(504)+C(757)=1261 >= 1251 -> stops; B EXCLUDED.
  test74 -> new pack of {C,D}; test77 expire -> 4 packs {E,A,B,new}. PASS.
- grit: THIRD=2524(B), BATCH=2525. D(672)+C(1224)=1896 < 2525, then B exp=2524<2525
  -> B INCLUDED; total=4420 stops. test74 -> new pack of {B,C,D}; test77 expire
  deletes B,C,D -> 3 packs {E,A,new}, want 4. FAIL. 78/81 cascade off the same packs.

Drilled into pack-D with `verify-pack -v` (same OIDs both binaries):
- git: tree 8d5716 full (166->158); tree 43ddb53 **delta vs 8d5716** (9 delta ->38);
  commits 127 compressed.
- grit: tree 8d5716 full (166->179); tree 43ddb53 **FULL** (135->148, no delta);
  commits 145/146 compressed.

grit pack-objects only deltifies BLOBS (optimize_blob_deltas, pack_objects.rs:4811 —
prefix/LCP + thin-pack boundary). It never delta-compresses trees or commits, so every
tree is stored full. Pack-A spans 8 commits/8 trees, magnifying the gap (5291 vs 2025).
Fixing 77/78/81 requires a general tree/commit delta window byte-comparable to git —
a large pack-objects feature with high regression risk across t53xx/t5616/t77xx. Out of
MIDX core; no MIDX-side lever exists (test derives BATCH from grit's real pack bytes).

### 98 root cause (bitmaps, NOT MIDX)
`git repack -adb --write-midx` writes the `.bitmap` sidecar as a **0-byte placeholder**
(midx.rs write_bitmap_placeholders; verified: multi-pack-index-*.bitmap is empty).
`test-tool bitmap list-commits` (grit/src/main.rs:860 run_test_tool_bitmap) is a stub:
it rev-lists all refs and drops the lexicographic-last commit as a heuristic, gated by
whether pack.preferBitmapTips is non-empty. Test 98 needs: real EWAH commit-bitmap
writing during MIDX bitmap repack, a real bitmap-commit selection algorithm (so some
commits get no bitmap), `pack.preferBitmapTips` pattern->hierarchy matching, and a
list-commits that reads the actual bitmap. ewah_bitmap.rs exists but isn't wired into
MIDX bitmap writing. Large separate subsystem (shared with t5310/t5326/t5319-97).
