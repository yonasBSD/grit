## Failing subtests and why they cannot pass within the rules

Two independent, unfixable-in-grit causes.

### Cause A — default `--oneline` decoration to a file (tests 2,4,5,6,7,16,18,19)
These run `git log --oneline … >out` (NO `--decorate`, NO `log.decorate` config) and
`grep` for decoration text (HEAD, tags, feature, parens, old-tag, …).

Upstream git default is `--decorate=auto` = "decorate only if stdout is a terminal,
otherwise NO ref names" (git/Documentation/git-log.adoc lines 43-49). The test redirects
to a file → non-TTY → no decoration. Verified system git 2.52: `git log --oneline | cat`
shows **no** decoration. grit already matches this (TTY-gated) in
`grit/src/commands/log.rs::resolve_decoration_display` (~line 1843-1844). This is correct.

Making `--oneline` decorate unconditionally to a file WOULD satisfy t4207 but REGRESSES
`t4202-log.sh` (currently 120/149 passing): its `'oneline'` test (t4202 lines 105-108)
does `git log --oneline > actual && test_cmp expect actual` where `expect` has NO
decorations, and `'log.decorate configuration'` (t4202 lines 808+) explicitly asserts
bare `--oneline` == `expect.none` until `log.decorate` is set. The two invocations are
byte-identical (`git log --oneline`, no config) so NO conditional fix can satisfy both.
115 test files use `log --oneline`; many test_cmp exact output. Unconditional decorate
is the wrong fix.

### Cause B — `master` vs `main` (tests 3,8,13,21)
These `grep "master"`. The harness (`scripts/run-tests.sh` line 376) exports
`GIT_TEST_DEFAULT_INITIAL_BRANCH_NAME=main`, and grit `init` honors it
(`grit/src/commands/init.rs` lines 522-526), so the default branch is **main**.
`grep "master"` can never match. Verified by probe under exact harness env: branch = `main`.

## Proof grit's decoration is correct
With explicit `--decorate`/`--decorate=short` (manual repro), grit emits:
`83749fc (HEAD -> main, tag: lightweight-tag, tag: annotated-tag, feature, another-branch) third`
— HEAD arrow, branch, lightweight tag, **peeled annotated tag**, multiple branches,
and `old-tag` on the correct older commit all render correctly. Tests 9,10,11,12,14,
15,17,20,22,23 (the explicit-decorate / no-decorate / setup ones) pass.

## Conclusion / recommendation for mop-up
No grit code change can fix this file without regressing t4202 (and likely many of the
115 `--oneline` tests) and without changing the harness branch name. The proper fixes are
in the TEST file (out of allowed scope here):
  - tests 2-7,16,18,19: add explicit `--decorate` to the `git log --oneline` invocations
    (or set `-c log.decorate=short`), since auto-decorate is TTY-gated by design.
  - tests 3,8,13,21: `grep "main"` instead of `"master"` (or `git init -b master repo`).
Recommend escalating as a test-authoring fix rather than a grit fix. Left ticket open.
