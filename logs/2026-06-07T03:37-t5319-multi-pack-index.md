## 77 / 78 / 81 — root cause proven against real git 2.52.0

These all gate on `multi-pack-index repack --batch-size` choosing the same
packs to combine as git. The batch loop (`select_packs_for_batch`,
`grit/src/commands/multi_pack_index.rs:560`) is byte-for-byte faithful to
git's `fill_included_packs_batch` (`git/midx-write.c:2001`): same mtime sort,
same `expected_size = (referenced<<14)/num_objects * pack_size + (1<<13) >>14`,
same `total_size < batch_size` accumulation and `expected >= batch_size` skip.
Per-pack MIDX referenced counts also match git exactly (A=24,B=15,C=9,D=6,E=7).

The ONLY divergence is grit's **pack byte sizes**. Reproduced the test's exact
setup (4096-byte random blob + 20 trivial commits, packs A..E) with both
binaries:

| pack | git bytes | grit bytes | objects |
|------|-----------|------------|---------|
| A | 2041 | 5291 | 24 |
| B | 1259 | 2531 | 15 |
| C |  763 | 1224 |  9 |
| D |  509 |  673 |  6 |
| E | 4538 | 4630 |  7 |

Drilling into pack-A (8 blobs + 8 commits + 8 trees), the gap is entirely the
**trees**: git compresses the 8 trees to 873 bytes using 1 delta; grit emits
3994 bytes with **0 deltas**. grit's pack-objects only deltifies *blobs*
(prefix/LCP blob deltas + thin-pack blob deltas — see the `delta base is not a
blob` bail in `grit/src/commands/pack_objects.rs` ~line 5010). Trees and commits
are never delta-compressed, so each consecutive tree (which differs from the
prior by one entry — ideal delta material) is stored fully zlib'd.

Consequence in test 74→77 (mtime order D,C,B,A,E set by the test's chmtime):
- **git**: THIRD_SMALLEST=1259, BATCH=1260. D(509)+C(763)=1272 ≥ 1260 → stop
  before B. Only D,C repacked; expire leaves new,A,B,E = **4 packs** (expected).
- **grit**: THIRD_SMALLEST=2531, BATCH=2532. D(673)+C(1224)=1897 < 2532 → B
  also processed (B.expected=2531<2532) → D,C,B repacked; expire leaves
  new,A,E = **3 packs**. Test 77 wants 4. 78 and 81 cascade from this state.

Fix requires general **tree/commit delta-window compression** producing
byte-identical output to git — a large change to pack-objects that touches
clone/fetch/push/repack/gc and dozens of other tests. Out of scope for the
MIDX ticket and high regression risk.

## 98 — pack.preferBitmapTips hierarchy

Needs real EWAH commit bitmaps from `repack -adb --write-midx`, git's bitmap

non-stub `test-tool bitmap list-commits` that reads the EWAH commit table. grit
writes a placeholder empty `.bitmap`. Separate large effort, outside MIDX core.
