## Independent re-verification (NOT a grit bug — environmentally unpassable here)

I re-reproduced the prior agents' (R1/R2/R3) blocker from scratch rather than trusting it:

1. **REAL_GIT selection.** `tests/lib-httpd.sh:30` candidate loop is
   `/usr/bin/git /usr/local/bin/git /bin/git`. On this box `/usr/bin/git` exists and is
   picked first = **Apple Git 2.39.5 (Apple Git-154)**. Homebrew **2.52.0** at
   `/opt/homebrew/bin/git` is never a candidate (and the PATH fallback loop is only reached
   when none of the three absolute paths exist).

2. **Server-side advertise routes to REAL_GIT.** For these 3 tests `BUNDLE_URI_PROTOCOL`
   is UNSET, so the smart-HTTP `git-upload-pack` wrapper
   (`tests/lib-httpd.sh:258-261`) sends `--advertise-refs` to
   `REAL_UP = $REAL_GIT exec-path/git-upload-pack` = Apple Git 2.39.5.

3. **Direct proof Apple Git 2.39.5 does NOT advertise `bundle-uri`.** Built a bare repo
   with `uploadpack.advertiseBundleURIs=true` + `bundle.*` set, then:
   ```
   GIT_PROTOCOL=version=2 /usr/bin/git upload-pack --advertise-refs repo2.git
   ```
   advertises: version 2, agent, `ls-refs=unborn`, `fetch=shallow wait-for-done`,
   `server-option`, `object-format=sha1`, `object-info`. **NO `bundle-uri`.**
   (The `bundle-uri` server capability was added upstream in Git 2.41.)
   The same command against `/opt/homebrew/bin/git` (2.52.0) DOES emit `bundle-uri`.

4. **grit client is correct.** Bundle-uri discovery is gated on the advertised cap:
   - `grit/src/http_smart.rs:1585-1587` — `!bundle_uri_override && server_advertises_bundle_uri(&caps) && transfer_bundle_uri_enabled()`
   - `grit/src/file_upload_pack_v2.rs:162-165` — `server_advertises_bundle_uri` matches `bundle-uri` / `bundle-uri=...`
   - `grit/src/fetch_transport.rs:1414, 2121` — same gate.
   No cap advertised ⇒ no `bundle-uri` command sent ⇒ no `git-remote-https` child
   spawned ⇒ the `child_start` trace2 event the tests grep for never fires. This is
   exactly how upstream git would behave against the same (non-advertising) server.

5. **Ported tests are byte-identical to upstream.** `diff` of the three test blocks
   (`tests/t5601-clone.sh:837-941` vs `git/t/t5601-clone.sh`) = IDENTICAL. No porting bug
   to flip/fix.
