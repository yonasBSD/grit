# t3700 add

## 2026-06-05

- Claimed ticket `7950cc`.
- Target test: `tests/t3700-add.sh`.
- Initial scope: remaining `git add` behavior failures after t3702.
- Reproduced 56/58. Failing subtests were 24 and 25, both around
  `git add --refresh`.
- Root causes:
  - Plain `read-tree HEAD` refreshed stat data even without `-u`, so
    `diff-index HEAD -- foo` could not observe the expected stale-stat `M`.
  - `diff-files` suppressed entries whose content matched the index whenever
    the index was writable, even if cached stat data was stale.
- Fixes:
  - Limit read-tree post-update stat refresh to `read-tree -u`/worktree update
    paths.
  - Keep content-clean but stat-dirty entries visible in `diff-files`, so
    pathspec-limited `git add --refresh bar` clears only `bar` while `baz`
    remains dirty.
