## Differential verdict: TEST bug (BSD wc padding)
Reproduced in /tmp with grit (target/release/grit) and real git 2.52.0 (/opt/homebrew/bin/git):
- grit `show-index` output is BYTE-FOR-BYTE IDENTICAL to git 2.52.0 (`diff` clean).
- Both emit 35 lines; verify-pack reports 35 objects. Counts are genuinely equal.
- On macOS, BSD `wc -l` LEFT-PADS its output: `show_count="      35"`, while
  `grep -c` emits clean `"35"`. The STRING comparison `=` fails on the padding;
  numeric `test -eq` (which parses integers, ignoring leading whitespace) passes.

This is the sanctioned BSD-wc-padding test-bug pattern: a pure comparison-mechanism
fix, no expected value changed, grit provably equals real git.

## Fix
Changed the final comparison in subtest #14 from string `=` to numeric `-eq`:
```sh
test "$show_count" -eq "$verify_count"
```
(plus an explanatory comment).
