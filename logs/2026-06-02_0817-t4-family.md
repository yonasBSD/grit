# t4 diff-family workflow

## Goal
Break t4 tests into dependency groups and start clearing failures toward a fully passing t4 family.

## Workflow result
Grouped failing tests into dependency lanes:
- core diff pair generation/output
- pathspec, unmerged entries, and exit status
- stats/summary formatting
- rename/rewrite/typechange
- whitespace, word-diff, and function names
- submodule diff
- combined/remerge diff
- external/no-index/textconv
- format-patch/log consumers

First implementation target: `t4017-diff-retval.sh` because it has one/few failures and exercises foundational `--exit-code`, `--quiet`, and `--check` behavior.

## Investigation
Read:
- `tests/t4017-diff-retval.sh`
- `git/Documentation/git-diff.adoc`
- `git/Documentation/diff-options.adoc` sections for `--check`, `--exit-code`, and `--quiet`

Ran:
- `./scripts/run-tests.sh t4017-diff-retval.sh --verbose` => 37/38
- direct verbose test run showed `git diff --exit-code --nonsense` printed an object resolution error instead of usage.

## Changes
- Updated `grit/src/commands/diff.rs` trailing-var-arg reparse to reject unknown `--long` options with exit 129 and a lowercase `usage:` line instead of treating them as revisions.

## t4001 diff-rename

Target: `t4001-diff-rename.sh` (22/23).

Investigation:
- The remaining runner-visible failure was the `git diff -l 4 -C -C --cached --name-status` case.
- `git diff` did not accept split `-l 4`, rejected repeated `-C`, and porcelain diff only ran rename detection rather than copy detection.

Changes:
- Accepted repeated `-C` by letting the final occurrence win.
- Added porcelain `git diff -l<N>` / `-l <N>` parsing and warning emission for limited copy detection.
- Routed `git diff -C` through copy detection so modified files can be copy sources.

Validation:
- `./scripts/run-tests.sh t4001-diff-rename.sh --verbose` passes 23/23.

## t4005 diff-rename-2

Target: `t4005-diff-rename-2.sh` (2/4).

Investigation:
- `git diff-index -C <tree>` was being misinterpreted by the top-level post-subcommand `-C <dir>` handler as a directory change.
- Passing `--raw` before `-C` avoided that handler, confirming command dispatch rather than diffcore was at fault.

Changes:
- Excluded diff-family commands from post-subcommand `-C <dir>` stripping so their `-C` copy-detection option reaches command parsing.

Validation:
- `./scripts/run-tests.sh t4005-diff-rename-2.sh --verbose` passes 4/4.

## t4006 diff-mode

Target: `t4006-diff-mode.sh` (6/7).

Investigation:
- Binary mode-only changes were rendered as `Bin <old> -> <new> bytes` even when blob bytes were identical.

Changes:
- Treat binary mode-only changes as zero byte-change stat records.
- Render binary diffstat rows with zero old/new byte sizes as plain `Bin`.

Validation:
- `./scripts/run-tests.sh t4006-diff-mode.sh --verbose` passes 7/7.

## t4044 diff-index unique abbrev

Target: `t4044-diff-index-unique-abbrev.sh` (1/2).

Investigation:
- Porcelain patch `index <old>..<new>` lines used a fixed 7-character abbreviation, which made two related blob IDs ambiguous.

Changes:
- Extended same-line old/new patch abbreviations until the two object IDs differ when rendering modified-file index lines.

Validation:
- `cargo check --release -p grit-cli` passes with the existing warning backlog.
- `cargo build --release -p grit-cli` passes with the existing warning backlog.
- `./scripts/run-tests.sh t4044-diff-index-unique-abbrev.sh --verbose` passes 2/2.
