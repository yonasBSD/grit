# t0001-init.sh — re-init move gitdir with linked worktrees

Ticket: e2fed2

## Failing subtests (before)
- 44: re-init to move gitdir with linked worktrees (absolute)
- 45: re-init to move gitdir within linked worktree (absolute)
- 46: re-init to move gitdir with linked worktrees (relative)
- 47: re-init to move gitdir within linked worktree (relative)

All four call `sep_git_dir_worktree`, which after `git init --separate-git-dir`
compares `rev-parse --git-common-dir` from the main worktree vs the linked
worktree and requires them to match.

## Root cause
The bug was NOT in `init --separate-git-dir` itself — that command correctly
moved the gitdir and rewrote the linked-worktree `.git` / `gitdir` / `commondir`
pointers. The mismatch was in `rev-parse --git-common-dir`:

- linked worktree (has `commondir`): printed the canonical absolute common dir
  (`/abs/seprepo`) — correct.
- main worktree (no `commondir`): grit printed `../seprepo` (relative-to-cwd),
  because the old handler used `PathDefaultMode::RelativeToCwd`.

Real Git implements `--git-common-dir` as
`print_path(repo_get_common_dir(), prefix, format, DEFAULT_RELATIVE_IF_SHARED)`
(`git/builtin/rev-parse.c`). Two facts matter:

1. `get_common_dir_noenv` (`git/setup.c`): if a `commondir` file exists the
   common dir is the canonical absolute realpath it points at; otherwise it is
   the gitdir *string itself* — which after setup is cwd-relative for an in-tree
   `.git` (`.git`, `../.git`) but absolute when `.git` is a gitfile (separate
   gitdir / linked worktree).
2. `DEFAULT_RELATIVE_IF_SHARED` in `print_path` does NOT fall back to cwd as the
   prefix (the `!prefix && (... || def != DEFAULT_RELATIVE_IF_SHARED)` guard).
   With no `--prefix` the prefix stays NULL and `relative_path` returns the path
   verbatim. So for the separate-gitdir main worktree the absolute gitdir is
   printed as-is → `/abs/seprepo`, matching the linked worktree.

The same bug affected `rev-parse --git-path <x>` for a separate-gitdir main
worktree (`../seprepo/objects` instead of the absolute path); the new helper
fixes the common-dir computation underlying that case too.

## Fix
`grit/src/commands/rev_parse.rs`:
- Added `setup_relative_git_dir()` — reproduces the cwd-relative-vs-absolute
  gitdir form Git's `setup_git_directory` leaves in `the_repository->gitdir`.
- Added `computed_common_dir()` — port of `get_common_dir_noenv` (honors
  `GIT_COMMON_DIR`, the `commondir` file, else the setup gitdir).
- Added `print_common_dir_path()` — faithful port of `print_path(..,
  DEFAULT_RELATIVE_IF_SHARED)`: verbatim when no `--prefix`, relative when an
  explicit sharing `--prefix` is given, with `--path-format` absolute/relative
  overrides.
- Rewrote the `ShowGitCommonDir` action to use them.
