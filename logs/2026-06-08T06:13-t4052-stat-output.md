# t4052-stat-output.sh — differential triage (2026-06-08)

Ticket: eb8249 (in-progress, assigned t5).

Baseline: 81/89 on grit. 8 failing subtests: 37, 43, 49, 64, 65, 72, 78, 89.
All are `log --graph --stat` variants.

## Method
Ran the real `t4052-stat-output.sh` through the harness with `tests/grit`
replaced by a symlink to `/opt/homebrew/bin/git` (git 2.52.0), so the
expectations are judged against real git in the exact harness environment.

Real-git harness results for the 8 subtests:

```
ok 37 - log --graph respects COLUMNS with big change
ok 43 - log --graph respects not enough COLUMNS with big change
ok 49 - log --graph respects diff.statGraphWidth
ok 64 - log --stat-width=width --graph with big change
ok 65 - log --stat-graph-width=width --graph with big change

ok 78 - log --graph respects COLUMNS with long name
not ok 89 - diffstat where line_prefix contains ANSI escape codes is correct width
```

## Verdicts

### 37, 43, 49, 64, 65, 72, 78 — GRIT BUG (out of scope)
These PASS on real git 2.52.0 in the harness, so the expectations are correct.
Grit fails them: for `log --graph --stat`, grit does NOT draw the graph
line-prefix (`|`, or colored `<RED>|<RESET>`) on the diffstat/summary lines —
it emits 3 spaces instead — and omits the blank `| ` line before the diffstat.
The diffstat BAR width itself is correct (byte-identical `+` count to real git),
so the ticket's claim that grit "does not subtract the graph prefix width" from
the bar is inaccurate; the actual defect is the missing prefix glyph on the
diffstat rows.

Differential example (subtest 64, `log -1 --stat --stat-width=40 --graph`):
```
GRIT : '   abcd | 1000 ++++++++++++++++++++++++++'   (3 spaces prefix)
REAL : '|  abcd | 1000 ++++++++++++++++++++++++++'    (pipe + 2 spaces)
```
Identical bar; only the graph prefix differs. OUT OF SCOPE — left failing.

### 89 — TEST BUG (fixed) + grit bug (still out of scope)
This FAILS even on real git 2.52.0. The test computes
`MIN_TERM_WIDTH = ${#FILENAME} + 9` as the smallest width that keeps the
filename untrimmed. That offset is wrong for git 2.52.0.

Empirically (real git 2.52.0, `log --graph --stat -n1`, colored `|` prefix),
the constant overhead is 17, not 9, and it is independent of filename length:
```
FN len=21 -> full name first appears at COLUMNS=38 (offset 17)
FN len=33 -> full name first appears at COLUMNS=50 (offset 17)
FN len=51 -> full name first appears at COLUMNS=68 (offset 17)
```
At `len+17` the full name shows; at `len+16` exactly the expected
`...eholder-text-placeholder-text` (trimmed by one char) shows — matching the
test's two assertions. Fix: `+9` -> `+17` (and the explanatory comment).
After the fix, subtest 89 PASSES on real git in the harness.

Subtest 89 still fails on GRIT because grit additionally (a) omits the colored
`<RED>|<RESET>` graph prefix on the diffstat line and (b) does not subtract the
prefix display width, so it never trims the name. That is the same grit graph
bug above — OUT OF SCOPE.

## Net
Test-authoring fix applied to subtest 89 (offset 9 -> 17), differential-verified
against git 2.52.0. Grit pass count unchanged at 81/89: all 8 failures are the
single underlying grit bug (no graph line-prefix on `log --graph --stat`
diffstat lines). File cannot reach fully-passing via test edits alone.

Classification: mixed (1 real test bug fixed; 8 grit-bug subtests remain).
