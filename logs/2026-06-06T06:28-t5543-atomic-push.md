# t5543-atomic-push — mop-up to 13/13

## Starting state
Recorded 11/13 (TOML), though an older log claimed 13/13 via delegating tests 12/13 to
`/usr/bin/git` (forbidden shell-out; that approach is gone / no longer effective). Fresh run
confirmed 11/13: tests 12 and 13 failing.

```
not ok 12 - atomic push reports exit code failure
not ok 13 - atomic push reports exit code failure with porcelain
```

Both tests push `--atomic` (12 also `--porcelain`) with
`--receive-pack='<pwd>'/receive-pack-wrapper`, a script that runs the real `git-receive-pack`
"$@" and then `exit 1`. Expected: the per-ref status report is still rendered (the remote did
write the ref), and the non-zero receive-pack exit makes the push fail.

Expected (test 12) stderr:
```
To ../upstream
 * [new branch]      HEAD -> no-conflict
error: failed to push some refs to '../upstream'
```

## Root causes (four independent grit bugs)

1. **Atomic + custom `--receive-pack` ran the in-process fast path** (`grit/src/commands/push.rs`).
   The fast path unpacks in-process and never spawns the caller's receive-pack program, so a
   wrapper that `exit 1`s after a successful receive was never observed. Fix: route
   atomic + non-default `--receive-pack` local pushes through the existing
   `push_over_receive_pack_child` (the same path `ext::` uses), spawning the program via
   `send_pack::spawn_receive_pack` and speaking the real receive-pack wire protocol to it.

2. **`push_over_receive_pack_child` aborted on a non-zero child exit before rendering status.**
   It `bail!`ed with "receive-pack failed with status …" right after `child.wait()`, so the
   "To"/"[new branch]" lines were never printed. Fix: capture the exit into `receive_pack_failed`,
   render the full per-ref report from report-status, skip the remote-tracking ref advance when
   failed, and end with `failed to push some refs to '<url>'` (matches `cmd_push`).

3. **`send_pack::spawn_receive_pack` could not handle a quoted program path.** The test passes
   `'<pwd>'/receive-pack-wrapper` (literal single quotes from `${SQ}$(pwd)${SQ}`). The plain
   `split_whitespace` branch tried to exec a file literally named with the quotes →
   `spawn receive-pack: No such file or directory`. Fix: treat `'` and `"` as shell metacharacters
   so the command runs through `sh -c` (quote removal, exactly like Git).

4. **`git rev-list --progress="…"` and `git unpack-objects` leaked output.** The wrapper's system
   `git-receive-pack` spawns internal `git` subcommands that, under the harness PATH, resolve to
   *grit*:
   - grit `rev-list` rejected `--progress=Checking connectivity` (the connectivity check) →
     added `--progress[=…]`/`--no-progress` as accepted no-ops in
     `grit/src/commands/rev_list.rs`.
   - grit `unpack-objects` printed "Unpacking objects: done (N objects)" even with stderr not a
     TTY. Upstream `builtin/unpack-objects.c` does `quiet = !isatty(2)`; mirrored that in
     `grit/src/commands/unpack_objects.rs` (quiet defaults to `!stderr.is_terminal()`; `-q` still
     forces it). Consistent with t5523 which asserts the line is absent.
