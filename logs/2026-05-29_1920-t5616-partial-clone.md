## Changes

### Tests 23 / 32 / 33 — tree/sparse/combine clone filters
- `grit/src/commands/clone.rs`: a `file://` partial clone whose filter cannot be reproduced by the
  client-side `copy_objects`+materialize layout (`tree:<n>`, `sparse:oid=…`, or a combine spec
  containing them) is now routed through upload-pack (`args.no_local = true`) so
  `pack-objects --filter` applies the reachability-aware filter server-side. New helper
  `clone_filter_materializable_locally`.
- `grit/src/pack_objects_upload.rs`: trace `pack-objects --filter=<spec>` as a single `=`-joined arg
  (matches the trace string test 23 greps).
- `grit/src/commands/pack_objects.rs`: `pack-objects --revs --filter=<spec>` now enumerates the
  filtered object set via the `rev-list` walk (which honors `tree:<depth>`, `sparse:oid=…`,
  `combine:…`) instead of the flat blob-size post-pass. New helpers `filter_needs_rev_list_walk`,
  `collect_filtered_objects_via_rev_list`.
- `grit-lib/src/rev_list.rs`: `sparse_oid_lines_from_filter` now returns `Result<Option<…>>` and
  emits Git's exact `unable to access sparse blob in '<name>'` / `unable to parse sparse filter data
  in <oid>` errors for unresolvable `sparse:oid` specs (test 33).

### Test 29 — ref-tip blobs in blob:none clone
- `grit/src/commands/clone.rs` `collect_reachable_skeleton_and_blobs`: a blob a ref (or tag chain)
  points at directly is classified as skeleton (kept loose + in the promisor pack), not a deletable
  content blob — so `fsck` no longer reports `missingObject` for a ref-tip blob.

### Test 45 — HTTP ref-tip connectivity
- `grit/src/commands/clone.rs`: after a blob:none HTTP clone, `check_clone_ref_tip_connectivity`
  fails with `did not send all necessary objects` when a ref points directly at a missing,
  non-promised object. Does not descend trees or follow tag chains, so annotated-tag targets stay
  tolerated (test 46 still passes).
- `grit/src/bin/test_httpd.rs`: the `one_time_script` helper now consumes the one-time script only
  when it actually modified the response (matching upstream `apply-one-time-script.sh`), so a
  preceding `ls-refs` round no longer wastes the substitution meant for the fetch packfile.

### Test 47 — REF_DELTA against missing promisor base (partial: producer + index-pack done)
- `grit/src/commands/pack_objects.rs`: `pack-objects --thin` with a `--not` boundary now deltas each
  packed blob against the same-path blob in the boundary trees, emitting a REF_DELTA whose base is
  external to the pack. New helpers `commit_tree_blob_paths`, `collect_tree_blob_paths`,
  `thin_pack_boundary_blob_deltas`. `build_pack` now writes the base OID for an external (empty
  `base_pack`) REF_DELTA base instead of zero bytes (the wire format was previously invalid).
  Verified: the crafted thin pack now contains the two expected REF_DELTAs.
- `grit/src/commands/index_pack.rs`: `--fix-thin` now lazily fetches a REF_DELTA base that is missing
  locally but promised by a promisor remote (`lazy_fetch_thin_delta_bases`), using the caller's
  target ODB rather than the process cwd. Only genuinely-missing bases are fetched.
- `grit/src/http_smart.rs`: trace sent `want <oid>` lines in the v2 fetch so tests can assert which
  objects were requested.
- Remaining 47 blocker: grit's HTTP v2 fetch performs a multi-round negotiation against the
  one-time-script server such that the substituted thin pack is not the pack ultimately ingested
  (grit ends up with the unsubstituted, filtered 2-object pack). Resolving this needs single-round
  fetch behavior for this case; deferred to avoid regressing the many passing fetch/clone tests.
