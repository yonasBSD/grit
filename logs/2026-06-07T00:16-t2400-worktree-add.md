## Diagnosis trap (important for the next agent)

Reproducing `rebase -i` failures *outside* the harness is misleading: an
interactive shell here exports `GIT_EDITOR=true` and the user's global
`core.editor=nvim`, both of which make `git rebase -i` skip the fake editor and
silently complete the rebase. `scripts/run-tests.sh` scrubs these (`env -u
GIT_EDITOR ... EDITOR=: VISUAL=:`), so always reproduce via the runner or with the
same env scrub. Our `tests/test-lib.sh` does **not** scrub arbitrary `GIT_*` vars
the way upstream `git/t/test-lib.sh` line 511 does, so the runner is the only thing
clearing `GIT_EDITOR`.

## Root cause (single bug, two symptoms)

- **Subtest 62** "not allow to delete a branch under rebase": from inside the
  `under-rebase` worktree (mid-rebase, **detached** HEAD), `git branch -D
  under-rebase` wrongly succeeded. Upstream `branch_checked_out()`
  (git/branch.c `prepare_checked_out_branches`) reports a branch as in-use when
  *any* worktree — including the current one — is mid-rebase/bisect on it. grit's
  `delete_branch` only checked `branch_checked_out_in_other_worktree` (HEAD symref
  of *other* worktrees) and `name == current` (normal current branch). Mid-rebase
  the worktree HEAD is detached, so both checks missed it.
- **Subtest 64** "check out from current worktree branch ok": this was a *cascade*.
  Because 62 deleted `under-rebase`, by the time 64 ran `git checkout under-rebase`
  the branch no longer existed → "pathspec 'under-rebase' did not match". With 62
  fixed (branch survives), checkout from the current rebasing worktree works.

## Fix

`grit/src/commands/branch.rs`, `delete_branch`: after the existing
`branch_checked_out_in_other_worktree` and `name == current` guards, also reject
when `worktree_refs::branch_occupied_any_worktree` reports the branch occupied
(this includes the current worktree's rebase/bisect/update-refs state, since
`occupied_branch_refs` reads `rebase-merge/head-name`, `rebase-apply/head-name`,
`BISECT_START`, etc.). Same error message: "cannot delete branch '<n>' used by
worktree at '<path>'".
