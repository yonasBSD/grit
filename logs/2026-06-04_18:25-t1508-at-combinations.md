# t1508-at-combinations

Ticket: 08c9c4 (`t1508-at-combinations.sh`)

## Findings

- Failing subtest 34 showed `newbranch@{1}` returning `fatal: log for 'newbranch' only has 1 entries`.
- Git's ordinal reflog syntax has one boundary slot before the oldest entry: `ref@{1}` with a single non-creation reflog entry resolves to that entry's `old_oid`.
- The old boundary must still reject an all-zero `old_oid`, preserving the existing `@{3}` failure for a branch whose oldest entry is branch creation.
- Failing subtest 35 was caused by a CLI `rev-parse --verify` precheck that rejected any empty reflog selector before the library resolver could apply Git's `ref@{0}` current-ref fallback.

## Changes

- Updated reflog ordinal resolution in `grit-lib/src/rev_parse.rs` to resolve nonzero `old_oid` boundary values and reject all-zero boundaries.
- Removed the `rev-parse --verify` empty-reflog precheck in `grit/src/commands/rev_parse.rs`, leaving selector behavior to the library resolver.
- Refreshed `data/tests/t1/t1508-at-combinations.toml` to 35/35.
