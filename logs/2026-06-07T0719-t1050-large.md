## Result
Fresh run: **28/29** passing — unchanged from all prior rounds. The lone failure

## Diagnosis (re-confirmed independently this round)
Subtest 12 is a **harness-porting bug**, not a grit bug.

The ported `tests/test-lib.sh` (lines 560-568) sets `OID_REGEX`, `ZERO_OID`,
`EMPTY_TREE`, `EMPTY_BLOB`, etc., but omits the single upstream assignment:

```
# git/t/test-lib.sh:1611
OIDPATH_REGEX=$(test_oid_to_path $ZERO_OID | sed -e 's/0/[0-9a-f]/g')
```

`test_oid_to_path` itself IS present (tests/test-lib-functions.sh), only the
assignment line is missing.

Subtest 12 (tests/t1050-large.sh:72) contains:

```
for l in .git/objects/$OIDPATH_REGEX
do
        test_path_is_missing "$l" || return 1
done
```

With `OIDPATH_REGEX` empty, the glob `.git/objects/$OIDPATH_REGEX` expands to the
bare existing directory `.git/objects/`, so `test_path_is_missing` fails
unconditionally — regardless of what grit does.

## Independent verification (both directions)
- **With `OIDPATH_REGEX` set** (replicating upstream): wrote an isolated harness
  test sourcing the real `tests/test-lib.sh`, defined `test_oid_to_path` +
  `OIDPATH_REGEX`, ran exactly subtest 12's body against grit. Result:
  `ok 2 - add a large file or two`. grit's `git add large1 huge large2`
  produces exactly **1 packfile + idx**, `git show-index` reports **2 objects**,
  and **0 loose objects** match the OID-path glob. grit behavior is correct.
- **With `OIDPATH_REGEX` unset** (replicating the ported harness): the same body
  fails because the glob hits the bare `.git/objects/` directory. Confirms the
  failure is solely the unset variable.

## Why it cannot be fixed from grit
`OIDPATH_REGEX` is a parent-shell variable in `tests/test-lib.sh`. No grit
command can set a variable in its parent shell, and the fix (adding the one
missing assignment line) requires editing `tests/test-lib.sh`, which is
forbidden by the working contract (AGENTS.md "Do Not"; causes regressions).

## Status

add / packing / pack-size limit / big-file diff / index-pack is correct (28/29).
