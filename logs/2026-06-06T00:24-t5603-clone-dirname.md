# t5603-clone-dirname — clone output directory name guessing

Ticket: 91356f. Goal: make `tests/t5603-clone-dirname.sh` fully pass (was 25/47).

## Diagnosis

t5603 only exercises the directory-name guessing logic of `git clone` (the fake
`GIT_SSH` wrapper ignores the path and just serves `$TRASH_DIRECTORY`). The 22 failures
fell into three buckets:

1. `foo/.git`, `foo///.git/`, `foo/.git///` variants (tests 6,7,12,13,16,19,20,24,26,27):
   the old per-path guessing did `file_name().strip_suffix(".git")`, which produced `.git`
   / empty instead of stripping the `/.git` segment and keeping `foo`.
2. Empty-path / host-only URLs `ssh://host/`, `host:/`, `user@host:/`, with auth and ports
   (tests 28-37): should fall back to the **hostname**, but grit guessed an empty name.
   Worse, `ssh://host/` was rejected outright by `parse_ssh_url` ("ssh: empty path").
3. Path-with-trailing-colon-number `ssh://.../test:1234` (tests 46,47): should guess `1234`
   (port-like suffix kept for paths), grit produced `test:1234`.

Root cause: grit guessed the dir name from the **already-parsed SSH spec path** (host/auth
stripped), so it could never fall back to the hostname, and its ad-hoc stripping did not
match Git. Git's `git_url_basename` (`git/dir.c`) operates on the **raw URL string**.

## Fix

- Added `grit_lib::transport_path::git_url_basename(repo, is_bundle, is_bare)` — a faithful
  port of Git's `git/dir.c:git_url_basename`, plus `NoDirectoryName` error and a
  `collapse_control_and_whitespace` helper. Operates on the raw URL: skips scheme, strips
  auth greedily to last `@`, strips trailing spaces/slashes/`/.git`, strips a trailing port
  number only for host-only URLs, finds the last component (treating `:` as a separator),
  strips `.git`/`.bundle`, falls back to the host when no path. 9 unit tests cover every
  t5603 case.
- `grit/src/commands/clone.rs`: added `clone_target_dir_name(args)` that uses
  `git_url_basename` on `args.repository` (or strips trailing slashes from an explicit
  `--directory`), passing `is_bare = args.bare && !args.mirror` (mirror keeps the guessed
  name). Wired it into both `run_ssh_clone` (local-resolved SSH) and `run_ssh_network_clone`
  (the path the test actually takes through the GIT_SSH wrapper), replacing the ad-hoc
  `file_name()`/`strip_suffix` blocks.
- `grit/src/ssh_transport.rs`: `normalize_ssh_url_path` no longer rejects an empty path —
  `ssh://host/` is valid (Git passes the empty path to `git-upload-pack`; the clone dir
  falls back to the hostname). No test relied on the "ssh: empty path" error.
