## Analysis (differential vs git 2.52.0 at /opt/homebrew/bin/git)

Subtest 19 ran `git log --oneline -n 1 >../actual` then `grep "HEAD" actual`.
The default `log.decorate=auto` only decorates a TTY, so piped output carries
NO decoration and "HEAD" never appears. This is the known `--decorate=auto` / TTY
test-bug pattern.

Reproduced in /tmp with identical inputs:
- Real git `log --oneline -n 1` (piped): `b3dd2c5 first commit` — no "HEAD".
- Grit `log --oneline -n 1` (piped): `d5545db first commit` — no "HEAD".
  => grit MATCHES real git byte-for-byte (no decoration on pipe). TEST bug.
- With explicit `--decorate`: BOTH real git and grit emit
  `<hash> (HEAD -> main) first commit` — both grep-match "HEAD".

Verdict: test-bug-fixed. Grit is correct; the test's expectation was unattainable
on piped output under default decorate=auto.

## Fix (test-body only)
Subtest 19: added explicit `--decorate` so the intended HEAD decoration is forced
on piped output, preserving the test's intent ("shows HEAD decoration"):
  `grit log --oneline -n 1` -> `grit log --oneline --decorate -n 1`
