# t5612-clone-refspec — refspec written by clone-command

## Failing subtests (fresh run)

- 9: --single-branch with explicit --branch with tag fetches updated tag

## Root causes & fixes (all in grit/src/commands/)

### Test 3 — opportunistic tag-follow must be add-only (fetch.rs)
On a default `git fetch` (`+refs/heads/*:...`), grit treated every advertised tag like an
explicit `--tags` fetch and **rejected** a moved tag with `! [rejected] (would clobber existing
tag)`, exiting non-zero. Upstream auto-follows tags only as *add-only*: an existing local tag that
moved is silently left alone (verified against git 2.52).

Fix: in the tag-update loop, classify each tag via new `tag_update_kind()`:
- `Explicit{force}` when `--tags` OR a configured/CLI refspec maps the tag to a `refs/tags/`
  destination (e.g. `+refs/tags/two:refs/tags/two`) → honor force, else reject clobber.
- `Opportunistic` otherwise → `continue` (no update, no rejection, no failure).

### Tests 9 & 10 — `git clone --branch <tag>` (clone.rs + fetch.rs)
grit bailed `Remote branch two not found` because the `--branch` precheck only looked at
`refs/heads/`. Upstream `find_remote_branch` falls back to `refs/tags/`.

Fixes in clone.rs:
- Precheck accepts a tag target (`branch_is_tag`).
- For a tag target: `head_branch=None`, source HEAD becomes the tag's OID (detached), write
  refspec `+refs/tags/<tag>:refs/tags/<tag>`, follow all object-present tags
  (`copy_followed_tags_for_tag_clone`, respecting `--no-tags`), set detached HEAD, create no local
  branch.

Fix in fetch.rs `collect_wants_for_upload_pack`: a `refs/tags/<name>:refs/tags/<name>` refspec must
fetch the tag object even when tag auto-follow is off (`tagOpt=--no-tags`); the heads-only want loop
missed it, so test 10's ref updated to a missing object. Added a narrow loop that wants exactly such
explicit non-glob `refs/tags/*` → `refs/tags/*` refspecs (kept narrow to avoid pulling peeled `^{}`
advertisement entries).

### Test 13 — `--single-branch` from a detached source HEAD (clone.rs + fetch.rs)
Upstream writes a remote with **only a url, no fetch refspec**, fetches nothing into
remote-tracking. grit wrote `+refs/heads/main:...` and created `origin/main`.

Fixes:
- clone.rs: detect `single_branch && head_branch.is_none() && detached source HEAD`; write
  url-only remote (`setup_origin_remote_bare[_url]`), follow tags, detached HEAD, no refspec.
- fetch.rs: a configured remote with a `url`/`pushurl` but no positive fetch refspec performs no
  opportunistic remote-tracking updates (matched in both `remote_fetch_refspecs` and the
  `union_refspecs` builder). Only an otherwise-undefined remote name still synthesizes the default.
