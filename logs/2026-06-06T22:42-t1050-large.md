## Root cause (independently re-verified)

Subtest 12 (git/t/t1050-large.sh:58-88) contains:

```sh
for l in .git/objects/$OIDPATH_REGEX
do
    test_path_is_missing "$l" || return 1
done &&
```

`OIDPATH_REGEX` is defined in upstream `git/t/test-lib.sh:1611`:

```sh
OIDPATH_REGEX=$(test_oid_to_path $ZERO_OID | sed -e 's/0/[0-9a-f]/g')
```

but the **ported** `tests/test-lib.sh` never assigns `OIDPATH_REGEX` (it
defines ZERO_OID, OID_REGEX, EMPTY_TREE, EMPTY_BLOB around lines 315/566 but
omits OIDPATH_REGEX). `test_oid_to_path` itself IS present in the ported
`tests/test-lib-functions.sh:1810`, so the only missing piece is the single
assignment line in test-lib.sh.

With the variable unset, `.git/objects/$OIDPATH_REGEX` glob-expands to the
bare directory `.git/objects/`, which always exists, so `test_path_is_missing`
fails unconditionally. Verified by hand:

- unset  -> `[.git/objects/]` EXISTS -> test_path_is_missing FAILS
- set    -> literal `.git/objects/[0-9a-f][0-9a-f]/...` -> missing -> PASSES
