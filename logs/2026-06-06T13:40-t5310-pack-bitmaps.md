## Root cause (confirmed, matches all prior comments)
grit's pack-objects/repack writes a **0-byte `.bitmap` placeholder** file
(`grit/src/commands/pack_objects.rs:1177` `std::fs::write(dir.join(format!("{stem}.bitmap")), [])`).
There is no real pack-bitmap writer or reader. `grit-lib/src/ewah_bitmap.rs` (289 lines) has the
EWAH compress/decompress core (used only by the index UNTR extension), but NOT the full
`.bitmap` file format: header (`BITM\1`), commit/type/XOR EWAH bitmaps, name-hash cache,
lookup table, nor the commit-selection algorithm, reader, or trace2 instrumentation.

Verified empirically: `grit repack -adb` in a fresh repo produces a 0-byte `pack-*.bitmap`.

## Every remaining failure requires the real bitmap reader/writer
