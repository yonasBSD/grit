# t0007-git-var — independent re-verification (opus-t5, 2026-06-08)

Ticket: d9f52b. Result: 26/27, BLOCKED. Confirms prior finding — harness
portability bug, NOT a grit bug.

## Failing subtest
- `not ok 6 - get GIT_EDITOR without configuration`
  - `sane_unset_all_editors && test_expect_code 1 git var GIT_EDITOR >out && test_must_be_empty out`
  - grit (and upstream git) exit 0 printing `vi` when `TERM` is a real terminal.

## Root cause (proven)
Upstream `git/editor.c::git_editor()` returns `NULL` (→ `git var GIT_EDITOR`
exits 1, empty) only when `is_terminal_dumb()` is true, i.e. `TERM` unset or
`"dumb"`. Otherwise it falls back to `DEFAULT_EDITOR` ("vi") and exits 0 — there
is no `isatty()` check.

The test depends on `TERM=dumb`. Upstream `git/t/test-lib.sh:654-656`:
```
USER_TERM="$TERM"
TERM=dumb
export TERM USER_TERM
```
The ported `tests/test-lib.sh` dropped these lines: `grep -c TERM= tests/test-lib.sh` = 0,
and there are zero `USER_TERM` references in it. Yet the ported
`tests/test-lib-functions.sh` (lines 156, 169, 176, 185, 195, 206, 217, 222)
still references `USER_TERM` and comments "Use your original TERM instead of
test-lib.sh's 'dumb'" — so the harness still *assumes* test-lib.sh sets
`TERM=dumb`. The porting omitted the assignment; `TERM` therefore inherits the
outer terminal (xterm-ghostty here), `is_terminal_dumb()` is false, and git's
own logic returns "vi"/exit 0.

## Proof
grit `var GIT_EDITOR` is byte-identical to upstream git (HOME=trash, editors
unset, GIT_CONFIG_NOSYSTEM=1):
- `TERM=dumb`  -> empty / exit 1
- `TERM=xterm` -> `vi` / exit 0
- `TERM` unset -> empty / exit 1

Definitive: `TERM=dumb sh t0007-git-var.sh` => `# passed all 27 test(s)` (27/27).

## Conclusion
`grit/src/editor.rs::resolve_git_editor` is a faithful port of `git_editor()`;
no grit change can pass subtest 6 without diverging from upstream git. The only
fix is restoring `TERM=dumb` to `tests/test-lib.sh`, which the working contract
forbids. The other 26 subtests pass (each sets GIT_EDITOR/EDITOR/core.editor
explicitly, independent of TERM). Staying BLOCKED.
