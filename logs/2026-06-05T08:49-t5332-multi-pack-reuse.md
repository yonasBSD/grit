## Failing subtests at start
- 8: reuse objects from first pack with middle gap
- 10: omit delta with uninteresting base (same pack)
- 11: omit delta from uninteresting base (cross pack)
- 12: non-omitted delta in MIDX preferred pack

## Root causes and fixes (all in grit pack-objects / MIDX machinery)

### 1. Delta direction was backwards (fixed tests 10 and 12)
`optimize_blob_deltas` in `grit/src/commands/pack_objects.rs` (the `blobs.len() > 3`
branch) made the **larger** blob the delta target with a smaller-prefix base. Git's
`type_size_sort` + `find_deltas` does the opposite: the **smaller** blob deltas against a
**larger** blob that has it as a prefix. Symptom: `git cat-file --batch-check %(deltabase)`
on `delta:f` returned zeros because grit stored `base:f` as the delta instead of `delta:f`.
Fixed by requiring `b.data.starts_with(&t.data) && b.data.len() > t.data.len()` and picking
the smallest qualifying (closest-size) base.

### 2. `pack-objects --all` wrote commits in OID order, not recency order (fixed test 8)
`collect_pack_objects` collects the `--all` object set into a `BTreeSet` (OID-sorted),
destroying Git's `compute_write_order` recency ordering (newest commit first). Test 8 asserts
F precedes E precedes D in the pack (`f_pos < e_pos < d_pos`); the `&&` chain broke at that
assertion long before the reuse check ever ran (which is why the MIDX write never executed and
the pack showed "reused 0"). Added `order_all_commits_first_parent_chain`, applied in the
`args.all && !args.incremental` write-order block, which reorders commits to the front in
`rev-list --all` recency order, preserving trees/blobs after.

### 3. Cross-pack delta reuse not rejected in MIDX multi-pack reuse (fixed test 11)
`compute_midx_reused_entries` reused a delta whose base, after MIDX deduplication, lives in a
**different** pack. Git's `try_partial_reuse` rejects this via `midx_pair_to_pack_pos`: the base
must be the MIDX's canonical copy in the *same* pack as the delta. Added
`MidxReuseTables::canonical_pack(oid)` (`grit-lib/src/midx.rs`) and require
`canonical_pack(base) == Some(delta_pack_id)` for both OFS and REF delta bases in the multi-pack
reuse eligibility check.
