## Failing subtest
- `not ok 6 - get GIT_EDITOR without configuration`
  - Body: `( sane_unset_all_editors && test_expect_code 1 git var GIT_EDITOR >out && test_must_be_empty out )`
  - In the ported harness this got exit 0 with stdout `vi`/`true` instead of exit 1/empty.

## Differential verdict: TEST/harness portability bug (NOT a grit bug)
Reproduced in /tmp with editors unset, HOME=trash, GIT_CONFIG_NOSYSTEM=1, both
`target/release/grit` and `/opt/homebrew/bin/git` (2.52.0). Byte-identical:

| TERM       | grit            | real git 2.52.0 |
| ---------- | --------------- | --------------- |
| dumb       | empty / exit 1  | empty / exit 1  |
| (unset)    | empty / exit 1  | empty / exit 1  |
| xterm      | `vi` / exit 0   | `vi` / exit 0   |

grit's `var GIT_EDITOR` is a faithful port of upstream `git_editor()` (git/editor.c):
returns NULL (exit 1) only when `is_terminal_dumb()` (TERM unset or "dumb"); otherwise
falls back to DEFAULT_EDITOR "vi" and exits 0. No isatty() involved.

Root cause: upstream `git/t/test-lib.sh` (lines ~654-656) exports `USER_TERM=$TERM; TERM=dumb`
for every test. The ported `tests/test-lib.sh` dropped that line (grep -c 'TERM=' => 0),
so inside the harness TERM inherits the outer terminal (observed `xterm-ghostty`), making
`is_terminal_dumb()` false and `git var GIT_EDITOR` fall back to `vi`/exit 0. Confirming
evidence: `tests/test-lib-functions.sh` lines 156 & 206 still comment "Use your original
TERM instead of test-lib.sh's \"dumb\"" — i.e. the harness still assumes test-lib.sh set
TERM=dumb. Prior agents (ticket d9f52b) proved this but were blocked: the canonical fix is
in tests/test-lib.sh, which their contract forbade editing.

## Fix (test-body only; expected VALUE unchanged)
This task authorizes editing tests/t0007-git-var.sh only. The dropped harness invariant
(`TERM=dumb`) is restored locally inside subtest 6's existing subshell:

```sh
TERM=dumb &&
export TERM &&
sane_unset_all_editors &&
```

This changes only the comparison/run environment to the upstream-intended one; the expected
result (exit 1 / empty stdout) is exactly what real git 2.52.0 produces under TERM=dumb,
so no expected value was changed. Did NOT touch test-lib.sh / test-lib-functions.sh / lib-*.sh.

(test-lib-functions.sh already references USER_TERM and the "dumb" default). Out of scope to edit here.
