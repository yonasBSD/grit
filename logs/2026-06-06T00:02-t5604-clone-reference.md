# t5604-clone-reference — clone --reference / alternates

Ticket: c97711. Goal: 8 failing subtests (8, 16, 19, 24, 27, 31, 33, 34) → fully passing.

Final: **34/34 passing.**

## Root causes & fixes

### Local object copy (`grit/src/commands/clone.rs::copy_objects`)
Rewrote `copy_objects` / replaced `copy_dir_contents` with a recursive
`copy_or_link_objects_tree` faithfully mirroring Git's `copy_or_link_directory`
(builtin/clone.c):
- Copies the **entire** `objects` tree — top-level loose files, hidden files,
  unknown garbage dirs — not just hex fan-out directories. (**test 31**)
- `lstat`s every entry; any symlink aborts with
  `symlink '<rel>' exists, refusing to clone with --local`. (**test 33**)
- If `objects` itself is a symlink, aborts with
  `'<path>' is a symlink, refusing to clone with --local`. (**test 34**)
- `info/alternates` is skipped here (resolved by `merge_alternates_from_source_objects`).

### Local-path clone with `--reference` still copies source objects (**test 19**)
Old code skipped `copy_objects` whenever a reference was given. But Git only
borrows-without-copying for `file://` clones; a genuine local-path
`git clone --reference G F H` copies F's objects AND adds G as an extra
alternate. Changed guard to `copy_source_objects = !has_reference || !is_file_url`.

### `file://` clone preflight: drop wants the reference already has (**test 8**)
`clone_preflight_file_v2_if_needed` now takes the `--reference` object dirs and
filters wants present in them before issuing `command=fetch`, so no `want` is
sent for objects borrowable from the reference (Git's `everything_local`).
New helper `resolve_reference_object_dirs` in clone.rs.

### `--no-local --reference=P` sets up the alternate before fetch (**test 27**)
The `--no-local` branch now calls `append_reference_alternates` *before* the
upload-pack fetch, so negotiation borrows from P and the clone genuinely
depends on it (removing P corrupts R; `--dissociate` repacks S to make it
self-contained).

### Fetch negotiation: `everything_local` want filtering (**test 24**)
`fetch_via_upload_pack_skipping` now filters `compute_wants(...)` through
`filter_wants_already_local` → `want_is_locally_complete` (commit/tag closure
check via `grit_lib::connectivity::push_tip_objects_exist`). A fetch into a repo
with an alternate no longer `want`s objects the alternate already holds.

### Local-path pull copies only the reachable closure (**test 16**)
`pull` from a local/`file://` path used `copy_objects_for_pull` (hardlinked the
remote's whole object store). Changed `pull.rs` to resolve the pulled tips and
use `copy_reachable_objects` + `prune_loose_objects_available_from_alternates`,
so only genuinely-new objects land locally (loose), matching `git fetch`'s local
transport. Made `prune_loose_objects_available_from_alternates` `pub(crate)`.
