# t1050-large

Ticket: eaeba5

Initial state: 16/29. Current state: 28/29.

## Findings

- `git add` wrote large blobs as loose objects, so tests expecting
  `core.bigFileThreshold` pack storage failed.
- Large text blobs were still diffed as text because binary detection only
  checked for NUL bytes and ignored `core.bigFileThreshold`.
- `index-pack` was parsed through the generic clap wrapper, which rejected pack
  path positionals. The t1050 clone also produced two packfiles, so the shell
  glob passed two pack paths to `index-pack --verify`.
- Remaining subtest 12 is blocked by copied harness state: `OIDPATH_REGEX` is
  unset in `tests/test-lib.sh`, so the test expands `.git/objects/$OIDPATH_REGEX`
  to `.git/objects/` and requires the object directory itself to be missing. A
  Git command cannot set that parent-shell variable, and modifying
  `tests/test-lib.sh` is forbidden by AGENTS.md.

## Changes

- Added a large-blob batch for `git add` that writes unique blobs above
  `core.bigFileThreshold` into packfiles and honors `pack.packsizelimit`.
- Added a full-object pack writer helper for command-internal pack creation.
- Taught diff patch/stat paths to treat blobs larger than `core.bigFileThreshold`
  as binary.
- Added a raw `index-pack` argv parser and made `--verify` tolerate multiple
  pack paths supplied by shell globs.

## Validation

- `cargo fmt --package grit-cli`.
- `cargo build --release -p grit-cli` passed with the existing `ext_total`
  warning in `grit/src/commands/diff.rs`.
- `./scripts/run-tests.sh t1050-large.sh` now reports 28/29.
