# t4000-diff-format (ticket 879031)

## Status
Fully passing: 41/41 (was 39/41).

## Failing subtests fixed
- 35: `--no-patch` in `git diff --no-patch --patch-with-raw` is a no-op
- 38: `--no-patch` in `git diff --no-patch --patch-with-stat` is a no-op

## Root cause
`git diff --no-patch --patch-with-raw` (and `--patch-with-stat`) suppressed the
unified patch body. Order matters in upstream git: `-s`/`--no-patch` sets the
output to "no output", and a later `--patch-with-*` re-enables PATCH. grit's
`diff_show_unified_patch_last_wins(&raw_args)` already computed this last-wins
result correctly (returning `true` for `--no-patch --patch-with-raw`), and the
raw/stat header block was emitted. But the actual unified-patch hunk emission
was gated separately on `!args.no_patch` in `grit/src/commands/diff.rs`
(~line 3733). Since `args.no_patch` stays `true` whenever `--no-patch`/`-s`
appears anywhere in argv (it is not order-aware), the hunks were dropped even
though the enclosing `if show_unified_patch` block had already decided to show
them.

`git diff-files` variants (subtests 33, 36) passed because they go through the
order-aware `parse_diff_files_format_argv`, which clears `suppress_diff` on a
later `--patch-with-*`.

## Fix
In `grit/src/commands/diff.rs`, replaced the redundant `!args.no_patch` guard on
`show_unified_after_stat` with `show_unified_patch` (the enclosing last-wins
condition). This block is already inside `if show_unified_patch`, so the guard
was redundant for the normal case and wrong for `--no-patch --patch-with-*`.
