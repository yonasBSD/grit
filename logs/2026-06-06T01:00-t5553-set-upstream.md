## Starting state
Fresh run: 6/21 passing. Tests 5-9 (fetch --set-upstream) and 12-21 (pull --set-upstream) failing.

## Root causes found
1. **fetch --set-upstream set the wrong branch.** Grit derived the config key from the fetched
   ref/destination instead of the *current* HEAD branch. Upstream (`builtin/fetch.c`) sets
   `branch.<current>.remote/.merge`, with merge = the single FETCH_HEAD-only "source ref"
   (a CLI refspec with no `:dst`, or remote HEAD for an anonymous URL fetch).
2. **pull had no --set-upstream.** `git pull --set-upstream` simply forwards the flag to
   `git fetch` (builtin/pull.c). Pull's Args lacked the flag and hardcoded `set_upstream: false`.
3. **pull lacked --tags.** Needed for test 15 (`pull --tags --set-upstream upstream three`).
4. **pull HEAD eager-resolution failure.** `normalize_pull_fetch_refspecs` bailed resolving
   `HEAD` when the remote url is a bare relative name (`parent`, no slash). Fixed to leave the
   refspec as `HEAD` and let fetch resolve the advertised HEAD.
5. **CLI-refspec FETCH_HEAD entries were marked not-for-merge when they had a `:dst`.** Upstream
   marks *every* command-line refspec for-merge ("Merge everything on the command line"), so
   `git pull <remote> main:other2` has a merge candidate (test 13). Changed `dst.is_empty()` to
   `true` for the CLI-refspec branch line.
