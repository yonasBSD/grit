### Colorized add -p (fixes 58, 62, 63, 64, 65, 66, 67)
Added a `ColorCtx` to `grit/src/commands/add_patch.rs` resolved once per `add -p`
invocation. Mirrors git's `add-patch.c` `init_color`/`check_color_config`:

- `want_color_domain(color.interactive|color.diff)`: explicit always/never/true/false
  on the specific key wins, else falls back to `color.ui`, else auto. `color_auto()`
  is true when stdout is a tty OR `GIT_PAGER_IN_USE` is set and TERM != dumb (t3701
  `force_color` exports `GIT_PAGER_IN_USE=true TERM=vt100`).
- Diff slots (`color.diff.meta/frag/context/old/new`) and interactive slots
  (`color.interactive.prompt/header/help/error`) parsed via `grit_lib::config::parse_color`
  with git's defaults (bold / cyan / "" / red / green; bold-blue prompt; etc.).
- `colorize_diff()` colors each rendered diff line by prefix (`@@`=frag, `+`=new,
  `-`=old, ` `=context, file-meta lines=meta), then pipes through
  `interactive.diffFilter` if set. `diff.wsErrorHighlight=all` overlays GIT_COLOR_BG_RED
  on trailing whitespace (fixes 62, greps `old<`).
- The hunk render + file header now go through `colorize_diff` when color is active;
  the prompt is wrapped in the `interactive.prompt` color.
- diffFilter mismatch detection (fixes 66): builds the full plain diff
  (header + all hunk bodies), colorizes, runs the filter, and aborts with
  "mismatched output from interactive.diffFilter" when the filtered line count drops
  (git's parse_diff colored/plain lockstep check).

All default (non-color) paths are unchanged — coloring is gated behind `cctx.any()`.
