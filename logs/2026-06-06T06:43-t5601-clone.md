## Test 108: batch missing blob request during checkout — FIXED

Test does `checkout HEAD^` on a `--filter=blob:limit=0` partial clone with
`GIT_TRACE_PACKET` + `GIT_TRACE2_EVENT` set, and asserts:
- exactly one `fetch> done` line in the packet trace (one negotiation round)
- exactly one `"key":"total_rounds","value":"1"` trace2 data event

Root causes in grit:
1. The checkout lazy-fetch path (`hydrate_tree_blobs_from_promisor` ->
   `flush_promisor_blob_batch`, Local branch) just *copied loose objects* from the
   promisor ODB, never spawning an `upload-pack` negotiation. So no packet trace
   (`fetch> done`) was produced and no `total_rounds` event was emitted.
2. grit never emitted `total_rounds` anywhere (`grep total_rounds` over src was empty).

Fixes (both in grit/src, command wiring + transport):
- `grit/src/commands/promisor_hydrate.rs` `flush_promisor_blob_batch`: for a Local
  promisor source, when `promisor_local_lazy_fetch_prefers_upload_pack()` is true
  (i.e. `GIT_TRACE_PACKET` is active, same gate t4067 already uses), route the whole
  batch through `try_lazy_fetch_promisor_objects_batch` (one `upload-pack` round for
  all missing blobs) instead of loose-copying. Non-traced path unchanged.
- `grit/src/fetch_transport.rs` `fetch_upload_pack_negotiate_pack_bytes` (the
  explicit-OID-wants path, used ONLY by promisor lazy fetch): emit
  `trace2_emit_data_intmax(.., "total_rounds", 1)` after the single negotiation round
  — `negotiation_v2` for v2, `negotiation_v0_v1` for v0/v1 — matching upstream
  fetch-pack.c. Explicit-want lazy fetch is always exactly one round.
- `grit/src/commands/promisor_hydrate.rs` `try_lazy_fetch_promisor_objects_batch`:
  post-fetch retention used `exists_local`, which deliberately SKIPS objects in
  `.promisor`-marked packs. After ingesting the freshly-fetched (thin) pack and
  writing its `.promisor` marker, the wanted objects lived in a promisor pack, so
  `exists_local` reported them absent -> the batch bailed "could not fetch N objects"
  even though the checkout had actually succeeded. Switched the two post-fetch
  `need.retain(... exists_local ...)` checks to `repo.odb.exists(...)` (which includes
  promisor packs), mirroring the single-object path's `repo.odb.read(..).is_ok()`.

Result: 112/115. Manual repro: exit 0, one `fetch> done`, one total_rounds=1 event.
