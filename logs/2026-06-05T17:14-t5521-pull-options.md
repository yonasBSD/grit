# t5521-pull-options — pull option passthrough

Ticket: ba8fe6. Started at 8/22, finished at **22/22**.

## Root causes

All 14 originally-failing subtests were `git pull` CLI flags that grit's `pull`
command did not accept (clap rejected them) or did not forward to fetch/merge:

- `-v`/`--verbose` (6-9) — pull lacked a verbose flag.
- `--all` (12, 13, 15), `--force` (12), `--dry-run` (14, 15).
- `--allow-unrelated-histories` (16).
- `--signoff`/`--no-signoff` (17-20).
- `--no-verify`/`--verify` (21, 22).

## Fixes (all in `grit/src/commands/`)

### `pull.rs`
- Added `Args` fields: `verbose` (Count), `all`, `force`, `dry_run`,
  `allow_unrelated_histories`, `signoff`/`no_signoff` (mutually `overrides_with`),
  `no_verify`/`verify` (mutually `overrides_with` so the last one on the CLI wins,
  matching tests 20 and 22).
- Wired `verbose`/`force`/`dry_run`/`all` into the fetch `Args`, and
  `signoff`/`no_signoff`/`no_verify`/`allow_unrelated_histories` into
  `build_pull_merge_args`.
- `--dry-run`: early-return after fetch in all three pull paths (standard,
  local-path shortcut, `.` shortcut) so nothing is written (matches `pull.c`
  `if (opt_dry_run) return 0;`).
- `--all`: new `run_pull_all` that delegates to `fetch --all` (no repo arg) then
  integrates straight from the FETCH_HEAD that fetch wrote. Key subtlety: do NOT
  re-normalize FETCH_HEAD by branch name — with two remotes both serving `main`
  there are two `branch 'main'` lines, and `fetch --all` already marks only the
  current branch's configured upstream (`branch.<name>.remote`) for-merge.
- Verbosity: `-q`/`-v` are a single signed counter in Git
  (`parse_opt_verbosity_cb`), not two independent flags. `pull -v -q` is quiet but
  `pull -q -v` is verbose (the jump-to-±1 asymmetry). Added
  `compute_pull_verbosity` that walks `std::env::args()` in order (handling short
  bundles like `-qv` and long `--quiet`/`--verbose`/`--no-*`) and overrides
  `args.quiet`/`args.verbose` from the result.

### `show.rs` + `log.rs`
- `git show -s --pretty=format:%(trailers)` printed the placeholder literally:
  `show.rs`'s `apply_format_string` had no `%(...)` branch (only `log.rs` did).
  Added a `%(trailers...)` branch to show.rs that reuses
  `log::parse_trailers_opts` (made `pub(crate)`) and
  `grit_lib::commit_trailers::format_trailers`. This is what tests 17-20 rely on
  to inspect (or confirm the absence of) a `Signed-off-by` trailer on the merge
  commit.
