## Failing subtests 21-27 — root cause (BLOCKER, reconfirmed)
Local `tests/test-lib.sh` line ~1500 overrides `test_ref_exists`:
```sh
test_ref_exists () { git rev-parse --verify -q "$1" >/dev/null 2>&1 ; }
```
This is sourced AFTER upstream `test-lib-functions.sh`'s `show-ref --exists` version, so it wins.
Every one of subtests 21-27 has a `test_ref_exists <broken-named-ref>` PRECONDITION that runs
BEFORE the delete under test. `rev-parse --verify` runs `check_refname_format`, which rejects any
name containing `..` (all the `broken...ref` / `broken...symref` / dangling-symref targets here),
so it exits non-zero and the precondition fails — the actual delete is never reached.

## NEW independent evidence: tested against REAL git 2.39.5 (prior agents used 2.52.0)
Reproduced each precondition with `/opt/homebrew/bin/git` (2.39.5) on identical broken-ref setups:

| subtest | ref under `test_ref_exists` | real git `rev-parse --verify -q` | `show-ref --exists` |
|---------|------------------------------|----------------------------------|---------------------|
| 21/22   | dangling symref badname -> broken...ref | exit 1 | exit 0 |
| 23      | real ref broken...ref (valid OID)       | exit 1 | exit 0 |
| 24/25   | symref broken...symref -> main (exists) | exit 1 | exit 0 |
| 26/27   | dangling symref broken...symref -> idonotexist | exit 1 | exit 0 |

grit's `rev-parse --verify -q` gives the SAME exit 1 in every case. So even REAL git fails these 7
preconditions under the local helper. Upstream `show-ref --exists` (which grit also matches, exit 0)
would pass them. This is a divergence between the locally-copied `tests/test-lib.sh` and upstream
`test-lib-functions.sh`, not a grit bug.
