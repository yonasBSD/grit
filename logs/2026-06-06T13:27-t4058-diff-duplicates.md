# t4058-diff-duplicates — work log (ticket 62eee7)

Date: 2026-06-06

## Starting state
10/16 passing. Failing: 8, 9, 10 (cache-tree corruption detection), 11, 14, 15 (status empty after duplicate-tree checkout/merge/reset).

## Root causes

Two independent problems, both stemming from trees with duplicate path entries
(`outer/inner` appears 4× in tree `two`):

1. **Index not deduplicated (tests 11, 14, 15).** grit's checkout / merge-ff / reset
   flattened a duplicate-entry tree into the index keeping all 4 identical-path
   entries. Real Git's `add_index_entry` replaces same-name entries, so the index
   holds exactly one (the *last* tree entry wins → `blob_two`). The stray dup
   entries made a subsequent `git status` non-empty (`AM outer/inner`).

2. **No cache-tree corruption detection (tests 8, 9, 10).** Real Git primes the
   cache-tree from the source *tree* during `read-tree --reset` / `reset --hard`,
   so the cache-tree's `entry_count` counts duplicates (4) while the dedup'd index
   has 1 entry. Writing the index then fails `cache_tree_verify` with
   "corrupted cache-tree has entries not present in index". `git diff HEAD` hits the
   same corruption via unpack-trees' (ungated) cache-tree fast path.

   NOTE: upstream gates the write-time check on `GIT_TEST_CHECK_CACHE_TREE` (default
   `true` in upstream test-lib.sh). The grit harness `tests/test-lib.sh` does NOT
   export it, so grit must treat *unset* as `true` (check on) to match the tests;
   only an explicit falsy value disables it. Test 10 sets `=false` for the reset
   (so it writes the corrupt cache-tree) but `diff HEAD` still fails (ungated).

## Fixes
- `grit-lib/src/index.rs`: `Index::dedup_paths_keep_last()` — collapse duplicate
  `(path, stage)` keeping the last (matches add_index_entry replace).
- `grit/src/commands/{checkout,merge,reset,read_tree}.rs`: call dedup after building
  an index from a tree.
- `grit-lib/src/write_tree.rs`: `build_cache_tree_from_tree` (raw duplicate-preserving
  entry counts) + `verify_cache_tree` (the `entry_count + pos > cache_nr` check).
- `grit/src/commands/{read_tree,reset}.rs`: prime cache-tree from the *tree* for
  plain reset / read-tree --reset.
- `grit-lib/src/split_index.rs`: run `verify_cache_tree` on index write unless
  `GIT_TEST_CHECK_CACHE_TREE` is explicitly falsy (default on).
- `grit/src/commands/diff.rs`: verify cache-tree on diff (ungated) so `diff HEAD`
  reports corruption.
- `grit-lib/src/error.rs` + `grit/src/main.rs`: `Error::CacheTreeCorrupt` printed
  verbatim as `error: corrupted cache-tree has entries not present in index`.

## Note on ported-vs-upstream divergence
In upstream git/t/t4058, tests 11/14/16 are `test_expect_failure` (known Git bugs).
The ported tests/ file marks them `test_expect_success`. The dedup fix makes grit
genuinely produce empty status here, so the ported expectations pass.
