The generic loop `git -C sub $builtin -h` asserts each builtin prints a usage
synopsis to stdout, exits 129, and writes nothing to stderr. The three commands
above were dispatched without a `-h`/`--help` synopsis short-circuit:
- shortlog: manual arg parser rejected `-h` as `error: unknown switch \`h'`.
- whatchanged: reached the deprecation guard and printed the removal notice to
  stderr, exiting 1.
- index-pack: manual `parse_argv` rejected `-h` as `error: unsupported option: -h`.

## Fix
grit/src/main.rs `dispatch()`: prepend
`commands::upstream_synopsis_help::try_print_upstream_help_and_exit(subcmd, rest)`
to the `shortlog`, `whatchanged`, and `index-pack` arms (same pattern already
used by `bisect`, `archive`, `check-attr`, `interpret-trailers`, ...). All three
have vendored adoc SYNOPSIS entries in the generated
`upstream_help_synopsis.rs`, so the helper prints `usage: git <cmd> ...` to
stdout and exits 129 before any command-specific parsing or the whatchanged
deprecation guard runs.
