## Fixes

### add -p hunk edit: write edit file inside .git (fixes 19, 24, 28)
`run_editor_on_text` wrote the hunk-edit buffer to a `$TMPDIR` NamedTempFile.
Git writes it to `$GIT_DIR/addp-hunk-edit.diff` (via
`strbuf_edit_interactively(..., "addp-hunk-edit.diff", ...)`). t3701's fake editor
does a relative `mv -f patch "$1"`, which broke when `$1` lived in `$TMPDIR`
(cross-dir / perms). Switched to `git_dir.join("addp-hunk-edit.diff")`, threading
`&repo.git_dir` through `edit_hunk_and_apply`. Editor resolution order unchanged
(GIT_EDITOR/VISUAL/EDITOR).

### add -p hunk edit: garbage/headerless patch rejected (fixes 21)
The body parser required a `@@` header (`saw_body`) before processing lines, so a
totally garbled patch (no header) produced empty old/new and silently aborted.
Now anchors the body at the original hunk position when no header is present and
treats unrecognized lines as full-text context; a garbage patch then fails
`locate_hunk` and prints "patch does not apply" + "Your edited hunk does not
apply." (matching git). Bare-context lines (t28's trailing `c`) still work.

### Editor resolution via crate::editor (harness VISUAL=: trap)
The runner sets `EDITOR=: VISUAL=:` and the ported `tests/test-lib.sh` does NOT set
`TERM=dumb` (upstream git/t/test-lib.sh does, at line 654-656). git's `git_editor()`
only consults `VISUAL` when the terminal is NOT dumb, so upstream skips VISUAL and the
fake `EDITOR` wins. In the grit harness TERM=xterm-ghostty (not dumb), so a hand-rolled
`GIT_EDITOR/VISUAL/EDITOR` lookup picked `VISUAL=:` (no-op) and silently no-op'd the edit.
Switched `run_editor_on_text` to `crate::editor::resolve_commit_launch_editor`, which
already treats the harness placeholders `VISUAL=:` / bare `EDITOR=:` the way
`launch_commit_editor` does, so `test_set_editor`'s `EDITOR='"$FAKE_EDITOR"'` is used.
Also runs the editor from the work tree (relative `mv` in the fake editor).

### Empty-file add/delete diff header (fixes 42)
`git diff [--cached]` of an added/deleted EMPTY file (both blobs empty) must emit only
`diff --git` / `new|deleted file mode` / `index`, no `--- / +++` lines. Added a guard in
`write_patch_with_prefix` (diff.rs) skipping the body when status is Added/Deleted and
both contents are empty. Verified no regressions in t4013/t2203/t2200/t4001/t7508.

### Unmerged path in diff --cached (fixes 56)
`git diff --cached` of an unmerged (conflicted) index entry must print a single
`* Unmerged path <name>` line, not a malformed `diff --git` block. Added an early
`cached && status == Unmerged` branch in `write_patch_with_prefix` (diff.rs). No
regressions in t4015/t4002/t7508/t7060/t7512(pre-existing 44/47).

### reset -p skips unmerged + diff-index --diff-filter (fixes 86)
Two parts: (1) `reset_patch` (reset.rs) re-added unmerged paths from the target tree
because they have no stage-0 entry; added an `is_unmerged` guard so `reset -p` is a
no-op for conflicted entries (git skips them). (2) `git diff-index` didn't accept
`--diff-filter=<letters>`; added parsing + an `apply_diff_index_diff_filter` helper
(uppercase=include, lowercase=exclude, like `git diff`). No regressions in
t7102/t7105/t4035.

### add -p validates diff.algorithm (fixes 69)
`git -c diff.algorithm=bogus add -p` must abort with
`error: option diff-algorithm accepts "myers", "minimal", "patience" and "histogram"`
(git's add -p shells to diff-files, which validates). Added an early guard in
`run_add_patch_with_reader` rejecting any diff.algorithm not in
myers/default/minimal/patience/histogram.

### add glob pathspec recurses subdirs (fixes 71)
`git add "*.c"` only staged top-level matches: a wildcard pathspec without long magic
went through `expand_glob_pathspec`, which only `read_dir`s a single level. Git's `*.c`
uses wildmatch without `WM_PATHNAME` so `*` crosses `/` and matches at any depth.
Changed `pathspecs_need_match_walk` to also return true for any wildcard pathspec, so it
routes through the recursive `add_with_pathspec_list` walk + `matches_pathspec_list`.
No regressions in t3700/t3702/t3704/t3910/t6130.
