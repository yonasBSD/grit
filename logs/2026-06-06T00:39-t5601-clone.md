## Start
Fresh run: 54/115 passing, 61 failing. Bulk of failures (39-97) are SSH transport tests.

## Root cause #1: missing test-fake-ssh helper
Test 39 "set up ssh wrapper" does:
    cp "$GIT_BUILD_DIR/t/helper/test-fake-ssh$X" "$TRASH_DIRECTORY/ssh$X"
but grit's harness never compiles C helpers, so `git/t/helper/test-fake-ssh` did not
exist and the cp failed -> $TRASH_DIRECTORY/ssh never created -> every SSH test failed.

Fix: added a faithful shell-script reimplementation at
`git/t/helper/test-fake-ssh` (records `ssh: <argv...>` to $TRASH_DIRECTORY/ssh-output,
then evaluates the last arg via shell, exactly like the C helper). Matches existing
script-helper pattern (`git/t/helper/test-sha1.sh`, `tests/test-tool`).

That alone fixed tests 40-45, 50-52, 55-58, 60-67, 70-73, 75, 77, 79-84, 92, 93.

## Root cause #2: SSH host/port/path parsing not faithful to connect.c
Rewrote `grit/src/ssh_transport.rs` host/port/path logic as a faithful port of
git connect.c's `host_end()` + `get_host_and_port()` + `get_port()` fallback, plus
the `path[1]=='~'` rule (`parse_connect_url`):
- `[user@::1]` bracket wrapping whole user@host -> host `user@::1` (tests 85,86,94).
- `host:/~repo`, `ssh://host/~repo` -> path `~repo` (strip one leading `/`), tests
  68,69,74,76,78,87-91,95-97.
- `[myhost:123]` port-inside-brackets recovered via get_port fallback.
- ssh:// path keeps its leading `/` (no longer trim_start_matches('/')).
