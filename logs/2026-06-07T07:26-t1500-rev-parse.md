## Starting state

## Diagnosis
`git rev-parse --symbolic-full-name --bisect` is expected to print the matching
bisect ref *names* (`refs/bisect/bad-1`, `refs/bisect/bad-3`, `refs/bisect/bad-4`,
`^refs/bisect/good-3`) in sorted-by-name order. grit's `Action::BisectRefs`
ignored the `--symbolic-full-name` flag (the variant payload was bound to `_`)
and always printed object ids, sorted by oid.

Upstream reference (git/builtin/rev-parse.c): `--bisect` iterates refs under
`refs/bisect/bad` (printed via `show_reference`) and `refs/bisect/good` (printed
via `anti_reference`, prefixed `^`). `show_rev` with `SHOW_SYMBOLIC_FULL` dwims
the ref and prints its full name; otherwise it prints the oid. Refs are emitted
in the ref store's sorted-by-name order.

## Fix
grit/src/commands/rev_parse.rs `Action::BisectRefs`:
- Bind the variant payload as `symbolic_full_name`.
- Collect refs per prefix sorted by ref name (matching git's iteration order).
- When `symbolic_full_name` is set, print the ref name (with `^` for good);
  otherwise print the oid.
- Use a plain `starts_with` prefix match (as git does), which already excludes
  `refs/bisect/b` and `refs/bisect/go`.
