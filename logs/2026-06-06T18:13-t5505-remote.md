## #24 "show" — harness artifact in test-lib.sh (NOT a grit bug)

`setup_repository one` and `setup_repository two` each run in their own subshell and call
`test_tick` before `git commit -m Initial`. Upstream `test_tick` is a plain shell variable
reset per-subshell, so BOTH "Initial" commits land at tick 1112911993+60 → byte-identical
SHAs → the clone (made from `one`) already holds `two`'s main object → `git remote show two`
reports up-to-date / fast-forwardable.

The grit-ported `tests/test-lib.sh` (lines 227-264) added **file persistence** for
`test_tick` (`_TICK_FILE = $TRASH_DIRECTORY/.git/.test_tick`) to survive subshell
boundaries. That leaks `one`'s advanced tick into `two`'s subshell:
  subshell one → tick 1112911993
  subshell two → tick 1112912053
(reproduced directly). Different timestamps → different "Initial" SHAs → the clone lacks
`two`'s main object → `classify_push` correctly emits `(local out of date)`. Real git in
this harness produces the same wrong output. Unfixable without editing the forbidden
`tests/test-lib.sh`. grit is correct.

## #130 "rename moves around the reflog" — grit IS wrong, but the fix regresses ported t1421

Root cause CONFIRMED against upstream C and ported tests:

* `git/refs/files-backend.c:2341 files_for_each_reflog_ent` streams the reflog file
  front-to-back with `strbuf_getwholeline` → **oldest-first** (no reversal). Only
  `files_for_each_reflog_ent_reverse` (2227) walks the file backwards → newest-first.
* `grit/src/main.rs:1100-1108` reverses the WRONG variant: it reverses for
  `for-each-reflog-ent` (the non-reverse one) and leaves `-reverse` unreversed. So grit's
  output is inverted vs upstream for BOTH variants. Verified empirically: grit
  `for-each-reflog-ent` emits newest-first, `for-each-reflog-ent-reverse` emits oldest-first.
* The ported `tests/t1405-main-ref-store.sh:79-89` and `tests/t1406-...:75-83` expect the
  UPSTREAM order (non-reverse head=oldest "one", tail=newest "recreate-main"; reverse
  flipped). i.e. the ported t1405/t1406 already AGREE WITH UPSTREAM, so grit currently fails
  those subtests too.

One-line fix (verified): change the guard at main.rs:1106 from
`if sub == "for-each-reflog-ent"` to `if sub == "for-each-reflog-ent-reverse"`.

Measured effect of that fix (rebuilt + ran all four files):
* t5505-remote: 127 → **128** (#130 passes).
* t1421-reflog-write: 10 → **9** (#7 "simple writes" REGRESSES).
* t1405-main-ref-store: 7/16 → 7/16 (unchanged — its reflog-ent subtests #10/#11 already
  fail as a cascade from earlier unrelated rename/for-each-ref failures, not from ordering).
* t1406-submodule-ref-store: 8/15 → 8/15 (unchanged, same cascade situation).

The t1421 #7 regression is a **test-port bug**: ported `tests/t1421-reflog-write.sh:105-108`
heredoc lists the entries newest-first (`second` then `first`), whereas upstream
`git/t/t1421-reflog-write.sh:104-107` lists them oldest-first (`first` then `second`). The
ported heredoc was mis-calibrated to grit's buggy newest-first output. grit's PHYSICAL
reflog file is correctly ordered oldest-first; only the read-back was inverted.
