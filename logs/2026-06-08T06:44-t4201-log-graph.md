# t4201-log-graph — port fix (2026-06-08)

Ticket: 894307. Baseline: 22/23. Final: 23/23 (fully passing).

## Failing subtest

14: `log --graph --reverse shows graph in reverse` (tests/t4201-log-graph.sh:139-146).

The test expected `git log --graph --reverse --oneline` to SUCCEED (head=first,
tail=fifth). This is a TEST-AUTHORING bug: real Git 2.52.0 and grit both REJECT
the combination.

## Differential proof (git 2.52.0 vs grit)

Scratch repo /tmp/t4201scratch/linear, 5 linear commits:

- `/opt/homebrew/bin/git log --graph --reverse --oneline`
  → `fatal: options '--graph' and '--reverse' cannot be used together`, exit 128.
- `target/release/grit log --graph --reverse --oneline`
  → `error: options '--reverse' and '--graph' cannot be used together`, exit 1.

Both reject. grit MATCHES real git's behavior → TEST bug, not a grit bug.

This rejection is the canonical, asserted-everywhere behavior:
- tests/t9260-log-oneline-format.sh:359 `log --oneline --graph --reverse is rejected`
  (test_must_fail; TOML 33/33 fully_passing).
- tests/t4202-log.sh:1931 `--reverse and --graph conflict` (test_must_fail).

Prior agents (comments 1-4 on ticket) correctly diagnosed the bad expectation but
believed they could not edit the test body (could only flip
expect_failure->success). This task EXPLICITLY authorizes editing the test body, so
the legitimate fix is to correct the wrong expectation rather than remove grit's
guard (which would regress t9260 + t4202).

## Fix

Rewrote subtest 14 to assert the rejection, mirroring the canonical t9260 idiom:

    test_expect_success 'log --graph --reverse is rejected' '
        (
        cd linear &&
        test_must_fail git log --graph --reverse --oneline >out 2>err &&
        grep "reverse.*graph\|graph.*reverse" err
        )
    '

## Result

`./scripts/run-tests.sh t4201-log-graph.sh` → 23/23, fully_passing = true.
