# t4210-log-i18n — ticket 349370

Date: 2026-06-06T19:16
Result: 21/21 passing (was 19/21).

## Failing subtests
- 3: log --grep searches in log output encoding (latin1)
- 5: log --grep does not find non-reencoded values (latin1)

## Root cause
`git log --grep` must search the commit message **after re-encoding it to the log
output encoding** (`--encoding` / `i18n.logOutputEncoding`), at the byte level — see
`commit_match()` in `git/revision.c` (`message = repo_logmsg_reencode(commit, encoding)`
then `grep_buffer(...)`).

grit was matching `--grep` against the UTF-8-decoded, stored `info.message` string,
so with `--encoding=ISO-8859-1`:
- `--grep=<latin1 é byte 0xE9>` found nothing (the stored message is UTF-8, byte 0xE9 absent)
- `--grep=<utf8 é bytes C3 A9>` matched both (UTF-8 é present in the stored message)

This is the exact inverse of Git. Two coupled problems:
1. The message must be re-encoded to the output charset before grep matching.
2. The `--grep` needle bytes must be matched at the byte level. grit's top-level argv is
   captured with `to_string_lossy()`, so a raw non-UTF-8 needle byte (latin1 0xE9) becomes
   U+FFFD before it ever reaches the grep code and can never match.

## Fix (grit/src/commands/log.rs)
- Added `GrepFilter` struct carrying `str_res` (existing UTF-8 `regex::Regex` path),
  `byte_res` (`regex::bytes::Regex`), and `reencode_to: Option<String>` (the non-UTF-8
  output encoding). `GrepFilter::matches()` re-encodes the message to that charset and greps
  the raw bytes when an encoding is active; otherwise it greps the UTF-8 string unchanged
  (so the common path is byte-for-byte identical to before).
- Added `recover_raw_grep_pattern_bytes()` which re-scans `std::env::args_os()` to recover
  the exact (possibly non-UTF-8) `--grep` needle bytes, lining them up positionally with
  `args.grep_patterns` (the lossy clap values).
- Added `build_grep_bytes_regex()` / `escape_byte_for_bytes_regex()` to compile a byte-mode
  (`(?-u)`) regex from raw needle bytes, honoring the fixed/basic/extended/perl pattern type
  (BRE→ERE via existing `bre_to_ere`; non-ASCII bytes rendered as `\xHH`).
- Threaded `&GrepFilter` (replacing `grep_res: &[Regex]`) through the four message-grep
  paths: `run_rev_list_log`, `run_reflog_walk`, `WalkCommitsIter`(+`new`), and `walk_commits`.
