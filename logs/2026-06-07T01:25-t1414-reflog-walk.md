## Diagnosis
- `git rev-list -g` (no positional rev) must exit non-zero. Upstream
  builtin/rev-list.c line ~842 triggers `usage(rev_list_usage)` when
  `!revs.commits && reflog_walk_empty(...) && !revs.pending.nr &&
  !revs.rev_input_given && !revs.read_from_stdin`.
- Unlike `git log -g` (which defaults to HEAD via add_head_to_pending and so
  walks HEAD's reflog), `rev-list` supplies no default revision.
- grit's grit/src/commands/rev_list.rs walk_reflog block wrongly defaulted to
  `list_reflog_refs` (all refs) when revision_specs was empty — that is the
  `--reflog`/add_reflog_tips behaviour, not `-g`.

## Fix
- grit/src/commands/rev_list.rs: when walk_reflog && revision_specs.is_empty(),
  print rev-list usage and exit(129) instead of expanding to all reflog refs.
