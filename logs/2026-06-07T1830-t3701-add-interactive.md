### "No newline at end of file" in add -p (test 38 "add first line works")
Two bugs, both around a worktree file whose final line has no trailing newline
(`firstline\nbaseline\ncontent\nlastline` w/ no `\n`):

1. **Staged blob got a spurious trailing newline.**
   `checkout::blend_line_diff_by_hunk_ranges` (shared by add -p / checkout -p staging)
   rebuilt content with `.lines()` + always `push('\n')`, so a no-final-newline file was
   staged WITH a newline → wrong blob hash. Fixed: track which side (source/worktree)
   produced the final emitted line and drop the trailing `\n` when that side had none.
   (grit/src/commands/checkout.rs, blend_line_diff_by_hunk_ranges)

2. **Hunk display omitted `\ No newline at end of file`.**
   `render_hunk_with_offsets` never emitted the marker. Added `old_no_nl`/`new_no_nl`
   params; each body line records whether it is the file's last line on its side, and the
   marker is printed right after it. Threaded the two no-nl flags through all 3 call sites.
   (grit/src/commands/add_patch.rs)

Result: 110/130 (test 38 passes; staged result + displayed hunks both match upstream).
