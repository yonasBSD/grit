## Key harness gotcha (false negatives when running the file directly)

Running `./tests/t3701-add-interactive.sh` directly in the agent shell makes the
editor tests (19, 21, 24, 28, 83) spuriously fail, because the agent shell exports
`GIT_EDITOR=true`. `scripts/run-tests.sh` does `env -u GIT_EDITOR ...`, so under the
official runner those tests PASS. Always reproduce with `GIT_EDITOR` unset (and an
empty `HOME` + `GIT_CONFIG_NOSYSTEM=1` so a global `core.editor` doesn't leak in),
or just trust `run-tests.sh`. The true failing set at 115/130 was:

  43 55 59 60 76 77 78 81 82 84 85 87 113 124 130

## Fix landed: test 60 "brackets appear without color" (115 -> 116)

`grit/src/commands/add_interactive.rs` `print_file_list` / `print_name_list` rendered
the 1-based item number with `{:>3}` (width 3). Git's `print_file_item` uses
`%c%2d:` — selection-marker char + number right-aligned to width 2. So grit printed
`   1: ...` where git prints `  1: ...`. Changed both `{:>3}` -> `{:>2}`. The header's
5 leading spaces line up with `<mark><2-wide>: ` = 1+2+2 = 5. Verified test 60 now
matches exactly; no regressions (full file 116/130 under runner env).

This file was NOT being touched by other agents, so the change is cleanly isolated.
