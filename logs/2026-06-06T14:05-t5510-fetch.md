# t5510-fetch — D/F conflict resolved by --prune (regression fix)

Date: 2026-06-06
Ticket: d590aa (fresh; prior ticket 7ace33 closed at 215/215, this regressed to 214/215)

## Symptom

Fresh run: 214/215. Failing subtest 189 "branchname D/F conflict resolved by --prune".

Scenario: remote has branch `dir/file`; clone; then on remote delete `dir/file`
and create branch `dir`. In the clone, `git fetch --prune` must prune
`origin/dir/file` AND create `origin/dir` (a directory/file conflict in the
remote-tracking namespace).

Observed before fix:
```
error: updating ref refs/remotes/origin/dir: invalid ref: cannot update ref
  'refs/remotes/origin/dir': reference namespace conflict
 - [deleted]    (none)  -> origin/dir/file
 * [new branch] dir     -> origin/dir   # FAILED
```

## Root cause

grit/src/commands/fetch.rs, the inline D/F prune handler in the per-ref update
loop (~line 3291). It only pruned conflicting child refs when the incoming flat
ref existed on disk as a *directory* (`local_ref_path.is_dir()`).

Right after `git clone`, all tracking refs are written to `packed-refs`, not loose.
So `refs/remotes/origin/dir/file` lived only in packed-refs and there was NO
`refs/remotes/origin/dir/` directory. `is_dir()` returned false, the inline handler
was skipped, and the create of `refs/remotes/origin/dir` hit
`packed_ref_namespace_conflict` (refs.rs write_ref) because packed-refs still held
the child. The bulk prune (which would have removed the child) runs only AFTER the
ref updates, too late to resolve the conflict.

## Fix

Replace the `is_dir()` gate with a `list_refs(git_dir, "<local_ref>/")` lookup, which
merges packed + loose refs. When that returns any child refs, prune the stale ones
(guarded: skip any child still mapped this fetch via `updated_refs`) and then clear
now-empty directories so the flat ref can be created. This covers both the loose-dir
and packed-only cases.

## Result

- t5510-fetch: 215/215 (was 214/215).
- Manual repro now exits 0; `origin/dir` resolves to the same OID as remote `dir`.
- cargo test -p grit-lib --lib: 269 passed, 2 failed (the known pre-existing
  ignore::gitignore_glob_tests failures, unrelated).
- No new clippy warnings in changed lines.
