# Libification Phase 0 — scaffolding — 2026-06-08

First commit of the libification effort (plan:
`~/.claude/plans/concurrent-foraging-acorn.md`), on its own `grit-libify`
branch, kept separate from the test-fixing work on `grit-t5-progress`.

Additive only — no behavior change, every existing `grit_lib::<module>::…` path
unchanged:

- New `grit-lib/src/progress.rs`: `ProgressSink` + `NullProgress` (no-op default),
  `Cancel` + `NeverCancel`, and a `&mut T` forwarding impl. This is how
  long-running operations will report progress and observe cancellation without
  the library ever touching the terminal. 3 unit tests.
- New `grit-lib/src/{plumbing,porcelain}/mod.rs`: the two curated namespaces
  (machine-stable building blocks vs. user-facing result-model operations),
  doc-only scaffolding to be populated in later phases.
- `grit-lib/src/lib.rs`: declares the three new modules + a `prelude`
  (`Repository`, `Error`/`Result`, `Object`/`ObjectId`/`ObjectKind`, `Odb`,
  `Index`, `ConfigSet`); doc comment now points newcomers at `Repository` /
  `prelude` / `porcelain` / `plumbing`.

Verified: `cargo build --release -p grit-cli` clean; `cargo test -p grit-lib --lib`
279 passed (incl. the 3 new progress tests); only the 2 known pre-existing
`ignore::gitignore_glob_tests` failures remain.
