## Remaining failures: subtests 21-27

All seven failing subtests delete a dangling/broken symref. They share the same
shape and each begins with the precondition:

```
test_ref_exists refs/heads/<X>
```

The local `tests/test-lib.sh` implements:

```sh
test_ref_exists () {
        git rev-parse --verify -q "$1" >/dev/null 2>&1
}
```

Upstream `git/t/test-lib-functions.sh` (line 261) instead uses:

```sh
git ${indir:+ -C "$indir"} show-ref --exists "$1"
```

These two diverge for dangling/broken symrefs.

## Independent verification against real git 2.52.0

I reproduced every subtest 21-27 precondition with **real git 2.52.0** (the actual
delete ops were stripped, only the `test_ref_exists` precondition tested) using the
local helper's `rev-parse --verify -q`:

| subtest | ref arg                       | real git exit | grit exit |
|---------|-------------------------------|---------------|-----------|
| 21      | refs/heads/badname (dangling) | 1             | 1         |
| 22      | refs/heads/badname (dangling) | 1             | 1         |
| 23      | refs/heads/broken...ref       | 1             | 1         |
| 24      | refs/heads/broken...symref    | 1             | 128*      |
| 25      | refs/heads/broken...symref    | 1             | 128*      |
| 26      | refs/heads/broken...symref    | 1             | 128*      |
| 27      | refs/heads/broken...symref    | 1             | 128*      |

`show-ref --exists` (upstream helper) returns **exit 0** for all of these in grit,
matching upstream — so under the upstream harness these would pass.

Conclusion: the local helper's `rev-parse --verify -q` returns non-zero for these
refs in **real git too**, so even real git 2.52.0 fails these seven preconditions
under the local `tests/test-lib.sh`. This is a harness divergence, not a grit bug.

(*) Note on 24-27: grit's `rev-parse --verify -q refs/heads/broken...symref` exits
128 with `fatal: ambiguous argument 'refs/heads/broken'` — it mis-parses the `...`
as a range operator, whereas real git exits 1 quietly. This is a minor divergence
but does NOT change the subtest outcome: the precondition needs exit **0**, and both
1 and 128 are non-zero failures. Fixing it would not flip any subtest; I left the
shared rev-parse parser untouched to avoid risk to other agents.
