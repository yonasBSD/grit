## Starting state
200/219 passing. The 19 failures were all `git rev-parse --git-path` with
`GIT_COMMON_DIR=bar` returning the wrong base for paths that belong to the
shared common dir, e.g. `logs/refs`, `objects`, `config`, `hooks/me`,
`remotes/bar`, `branches/bar`, `common/file`, etc.

## Root cause
`grit/src/commands/rev_parse.rs` `Action::GitPath` used an ad-hoc, hand-rolled
prefix list to decide whether a path was "common" and then **canonicalized**
`GIT_COMMON_DIR` joined onto the absolute git dir, printing an absolute path
like `/private/tmp/gpt/.git/bar/logs/refs`. Two bugs:

1. The common/worktree-local classification did not match Git's
   `common_list`/trie in `git/path.c` (`update_common_dir` / `check_common`).
   For example `logs/refs` is common but `logs/refs/bisect/foo` is not, decided
   by a longest-`/`-terminated-prefix trie lookup â€” not by simple prefix tests.
2. Git uses the `GIT_COMMON_DIR` env value verbatim (setup.c `get_common_dir`)
   and `replace_dir` swaps the git-dir prefix, so `GIT_COMMON_DIR=bar` yields a
   **relative** `bar/<component>` (printed as-is by relative-if-shared because a
   relative path shares no root with the absolute cwd). grit was canonicalizing
   to an absolute path under `.git`.

## Fix
- Ported Git's `common_list` table + the trie classification semantics into
  `grit-lib/src/git_path.rs` as `is_common_git_path(rel: &str) -> bool`
  (handles `.lock` suffix stripping and consecutive-slash normalization exactly
  like `update_common_dir`). Added a unit test covering all t0060 cases.
- Rewrote the `GIT_COMMON_DIR` branch in `Action::GitPath` to:
  - honor env-var overrides (`GIT_GRAFT_FILE`/`GIT_INDEX_FILE`/`GIT_OBJECT_DIRECTORY`)
    first, matching `adjust_git_path` ordering;
  - when `is_common_git_path` is true, print `<GIT_COMMON_DIR verbatim>/<original
    component>` (preserving interior `//`), no canonicalization;
  - otherwise fall through to the existing per-worktree git-dir resolution.
- Recomputed the real-linked-worktree `is_worktree_local` flag from
  `!is_common_git_path(...)` (the prior hardcoded list was a subset of this
  faithful classifier, so behavior is preserved there).
