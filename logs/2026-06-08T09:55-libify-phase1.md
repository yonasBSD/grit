# Libification Phase 1 — public API curation — 2026-06-08

Additive curation of `grit-lib`'s public API (plan phase 1), on `grit-libify`.
No behavior change; every existing `grit_lib::<module>::…` path still resolves.

- Added seven **domain views** to `grit-lib/src/lib.rs` — `object_store`,
  `references`, `worktree_index`, `revision`, `diffing`, `merging`,
  `configuration` — each a curated table of contents over the flat module list,
  implemented as pure **named module re-exports** (collision-proof, unlike
  globbing large modules together). Crate doc now points readers at them.
- **`pub(crate)` demotion: none.** A fan-in scan of all four consumer crates
  (`grit`, `grit-protocol`, `grit-simple`, `grit-http-server`) found that *every*
  one of the 124 `pub mod` modules is referenced externally — the CLI genuinely
  uses nearly the whole engine surface, so there are no safe demotion candidates.
  Recorded here so the absence is intentional, not overlooked.

Verified: `cargo build --release -p grit-cli` clean; `cargo test -p grit-lib --lib`
279 passed (2 known `ignore` failures only); the new intra-doc links resolve
(the remaining `cargo doc` warnings are pre-existing broken links in unrelated
modules).
