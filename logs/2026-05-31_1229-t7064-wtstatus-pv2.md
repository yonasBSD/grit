## Problem

Status porcelain v2 emitted spurious `C` (new-commits) bits in submodule
sub-state tokens for dirty-but-unchanged-HEAD submodules. Failing tests:
- 22 (AM S..U) -> grit emitted SC.U
- 23 (AM S.M.) -> grit emitted SCM.
- 24 (AM S.M.) -> grit emitted SCM.
- 25 (AM S.MU) -> grit emitted SCMU
- 28 (.M S.M.) -> grit emitted SCM.

## Root cause

`fn format_porcelain_v2` in `grit/src/commands/status.rs` computed the
submodule flags correctly via `submodule_porcelain_flags(work_tree, path,
recorded_gitlink_oid)` (new_commits = sub_head != recorded_gitlink_oid),
then OVERRODE `f.new_commits = true` whenever the unstaged DiffEntry was
`Modified` with `old_oid != new_oid`.

That override was a regression. `diff_index_to_worktree` now emits a
`Modified` gitlink DiffEntry for a dirty-but-unchanged-HEAD submodule with
`old_oid = index gitlink OID` (nonzero) and `new_oid = zero`, so the
`old_oid != new_oid` test is always true for a dirty submodule, wrongly
forcing the `C` bit. (Commit 8d60abfda originally deleted this override
when it changed diff.rs to emit-on-dirty; it later reappeared via a
code-move/merge while diff.rs kept the new behavior.)

## Fix

Deleted the 5-line override block in `format_porcelain_v2`. The `C` bit now
comes solely from `submodule_porcelain_flags` (the HEAD vs recorded-gitlink
comparison), which is the correct source. No change to grit-lib/diff.rs.

File changed: grit/src/commands/status.rs (8 insertions, 6 deletions —
net removal of the override, replaced with an explanatory comment).
