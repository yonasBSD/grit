## What changed

### grit-lib/src/commit_encoding.rs
- Added `find_invalid_utf8` / `is_strict_utf8`, a byte-exact port of Git's
  `commit.c:find_invalid_utf8`. This is stricter than `str::from_utf8`: it
  also rejects surrogates, the non-characters U+xxFFFE/U+xxFFFF, and the
  U+FDD0..=U+FDEF block (which Rust accepts). Added unit tests.

### grit/src/commands/commit.rs
- `raw_to_message_result`: always retain the raw `-F` file bytes (previously
  dropped when the file happened to be valid UTF-8). The commit body is now
  stored verbatim and never decode-then-re-encoded. This was corrupting 7-bit
  encodings like ISO-2022-JP — the ESC control byte was turned into the literal
  HTML entity string "&#65533;" by encoding_rs's ISO-2022-JP encoder.
- COMMIT_EDITMSG re-read after prepare-commit-msg now preserves the verbatim
  raw bytes when the source message carried them.
- Removed the unconditional `encode_unicode` of the body; the body is stored
  verbatim. Author/committer identity is only re-encoded to the charset when it
  actually contains non-ASCII (matches Git storing identity verbatim).
- Added Git's strict UTF-8 validation when the effective encoding is UTF-8:
  prints "Warning: commit message did not conform to UTF-8." (3 lines) without
  aborting. Added the NUL-byte rejection ("a NUL byte in commit log message not
  allowed.") so UTF-16 message files fail.
- `-s` sign-off: when the commit encoding is non-UTF-8 and the assembled message
  contains non-ASCII (the sign-off committer name), the body is encoded to the
  declared charset so the stored bytes match the `encoding` header (previously a
  Latin-1 name was stored as UTF-8 bytes while labeled ISO8859-1 → mojibake).

## t3901 — remaining 2 failures (subtests 17 "am (L/L)", 20 "am (L/U)"): CROSS-LANE (am.rs)
`git am` with a non-UTF-8 i18n.commitEncoding stores the author/committer header
lines as UTF-8 bytes while writing `encoding ISO8859-1`, so a later decode (e.g.
format-patch --encoding=UTF-8 in the test's check_encoding) double-encodes the
name (`=C3=83=C2=81...`).
Fix belongs in grit/src/commands/am.rs (NOT owned by this lane) around the
`CommitData` build (~line 2198): when `encoding` is Some(non-UTF-8), set
`author_raw` / `committer_raw` via
`grit_lib::commit_encoding::encode_header_text(enc, &ident)` for non-ASCII
identities, mirroring commit.rs. The helper already exists in this lane's
commit_encoding.rs. The message-body re-encoding in am.rs is already correct;
only the identity headers need encoding.
