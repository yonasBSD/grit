# t7066-status-ignored

Ticket: ab0b6a

## Start

- Claimed ticket for `tests/t7066-status-ignored.sh`.
- Reproduced `31/32`; the only failure was subtest 23, `ls-files --ignored shows ignored files`.
- The failing command was plain `git ls-files --ignored`, which produced no ignored worktree paths because `ls-files` only loaded ignore matchers for `--exclude-standard` or explicit exclude options.

## Fix

- Updated `grit/src/commands/ls_files.rs` so untracked plain `--ignored` loads the repository standard ignore stack when no explicit exclude source is present.
- Kept explicit `-x`, `-X`, and `--exclude-per-directory` calls on the explicit matcher path to avoid broadening existing exclude-only tests.
