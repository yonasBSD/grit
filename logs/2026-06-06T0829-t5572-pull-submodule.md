# t5572-pull-submodule — mop-up round 1

Ticket: bf62ee (was closed/resolved by a prior agent claiming 69/69 @ 851719ca5),
but a fresh run showed only 22/69 passing (47 failing). Re-opened investigation.

## Symptom

Fresh run after rebuild: 22/69. The bulk of the `test_submodule_switch_func`
generated subtests (git_pull / git_pull_ff / git_pull_ff_only / git_pull_noff:
"added submodule creates empty directory", "modified submodule ...", etc.) plus
tests 59-69 failed.

Verbose trace of test 1 ended with:

```
error: missing object 63c430421a9604f896526f8a04236bdfe3ac1fc1 while copying from remote: object not found
not ok 1 - git_pull: added submodule creates empty directory
```

`63c4304...` is a commit inside the **submodule** `sub1`, recorded as a gitlink
in the superproject tree of branch `add_sub1`.

## Root cause

`grit/src/commands/pull.rs` local-path fast path resolves the pulled branch tip
into `copy_roots` and then copied its reachable object closure with
`super::fetch::copy_reachable_objects(...)`. That traversal walks every tree
entry — including gitlink (`0o160000`) entries whose commit IDs live in the
nested submodule's object store, not the superproject's. Reading such an object
from the superproject odb fails with "missing object ... while copying from
remote", aborting the pull before the merge.

The prior agent's fix (851719ca5) added the recurse-submodule wiring but used
the gitlink-walking copy helper, introducing this regression for any pull of a
branch that adds/modifies a submodule.

Git's own `pack-objects.c` skips gitlinks during tree traversal
(`if (S_ISGITLINK(entry.mode)) continue;`), so gitlink-referenced commits are
never part of a superproject's transferred object closure.

## Fix

`grit/src/commands/pull.rs` (~line 750): switch the local-path copy from
`copy_reachable_objects` to the existing
`copy_reachable_objects_skipping_gitlinks` helper, which omits `0o160000` tree
entries during traversal. No new helper needed — it already existed in
`grit/src/commands/fetch.rs`.

## Result

`./scripts/run-tests.sh t5572-pull-submodule.sh` → 69/69 fully passing.
