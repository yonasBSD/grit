## Failing subtest

Subtest 10: `git diff HEAD does not segfault` (lines 153-159).

```sh
git checkout base &&
GIT_TEST_CHECK_CACHE_TREE=false &&
git reset --hard &&
test_must_fail git diff HEAD 2>err &&
test_grep "error: corrupted cache-tree has entries not present in index" err
```

## Root cause — TEST-BODY bug (portability / authoring)

Upstream `git/t/test-lib.sh:1744` pre-exports `GIT_TEST_CHECK_CACHE_TREE=true`.
So in upstream the bare reassignment `GIT_TEST_CHECK_CACHE_TREE=false` mutates
the already-exported variable and therefore propagates to the `git reset --hard`
and `git diff HEAD` subprocesses, disabling the cache-tree check for the reset
(so it succeeds) while `git diff HEAD` still verifies the cache-tree ungated
(so it fails with the expected message).

The ported `tests/test-lib.sh` OMITS that export block (and editing test-lib.sh
is forbidden / owned by the orchestrator). With the variable unset, a bare
`GIT_TEST_CHECK_CACHE_TREE=false` does NOT export to child processes, so grit's
`git reset --hard` subprocess sees the variable unset, defaults the cache-tree
check ON, fails the reset, and aborts the `&&` chain before `git diff HEAD`.

This is the cwd/env-leak class of authoring bug: the test author relied on an
ambient harness export that the ported harness does not provide.

## Differential proof vs real git 2.52.0 (/opt/homebrew/bin/git)

Reproduced the full test-10 chain in /tmp on fresh, byte-identical inputs with
BOTH the harness `git` (target/release/grit) and `/opt/homebrew/bin/git`, with
`GIT_TEST_CHECK_CACHE_TREE=false` actually exported to the subprocesses:

| step                         | real git 2.52.0 | grit            |
| ---------------------------- | --------------- | --------------- |
| `git checkout base`          | exit 0          | exit 0          |
| `reset --hard` (env=false)   | exit 0          | exit 0          |
| `git diff HEAD`              | exit 128        | exit 128        |
| diff HEAD stderr             | `error: corrupted cache-tree has entries not present in index` | identical |

grit MATCHES real git byte-for-byte. The grit behavior is correct; only the
test's env-propagation mechanism was broken. => TEST bug, sanctioned to fix.

## Fix (test-body only, mechanism not value)

Changed the bare assignment to an explicit `export` so the override reaches the
`git` subprocesses, exactly matching the test author's intent and upstream
behavior. `export` is scoped to the test body's subshell, so it does not leak to
later subtests (verified: tests 14/15, which depend on the default-ON check, still
pass; full file deterministically 16/16 across re-runs).

No expected VALUE was changed. No grit Rust source was modified.
