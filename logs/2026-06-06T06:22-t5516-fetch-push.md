## Lesson learned
Running `--run=<N>` in isolation is MISLEADING for t5516: many subtests depend
on the `setup` test populating `$the_first_commit`/`$the_commit`. With `--run=56`
alone, those vars are empty so `mk_test` issues a delete refspec and the test
follows a different path. ALWAYS validate with a full file run.

## Fix 1: push rejects empty branch name entries (subtests 56, 57)
Upstream `handle_config` in remote.c returns -1 for any `branch.<empty>.X`
(empty subsection), which the config machinery turns into a fatal
`bad config variable 'branch..X' in file ... at line N` while loading remote
config during push. grit was missing this; the no-arg push proceeded and the
remote rejected with `not a valid ref`.

Added `reject_empty_branch_subsection(&config)` in `grit/src/commands/push.rs`,
called right after config load in `run()`. It scans config entries for keys of
the form `branch..{remote,pushremote,merge}` (canonical key has an empty middle
component) and emits the upstream fatal error with file/line, exiting 128.

Result: 56 and 57 now pass. 98/124.

## Fix 2: push refmap for colon-less CLI refspecs (subtests 105, 107)
Mirrors Git `set_refspecs`/`refspec_append_mapped` (builtin/push.c). A colon-less
command-line ref (e.g. `main`) that uniquely names a local head is rewritten via:
1. `remote.<name>.push` as a refmap (test 105: `+refs/heads/*:refs/remotes/src/*`
   maps `main` -> `+refs/heads/main:refs/remotes/src/main`).
2. else if `push.default = upstream`, the branch's single `branch.<b>.merge` ref
   (test 107: `branch.main.merge refs/heads/trunk` -> `main:refs/heads/trunk`).

Added `map_cli_refspecs_via_refmap` + `query_push_refmap_dst` in push.rs and call
it in `run()` (made `args` mut) before computing `push_refspecs_from_config`.
Uses existing `count_refspec_match_push` (DWIM local-head match) and `match_glob`.

Result: 105, 107 pass; 106 still passes. 99/124.

## Fix 3: fetch into bare respects core.logallrefupdates (subtest 113)
grit's fetch explicit-destination CLI path (`git fetch .. main:one`) wrote the
local ref via write_ref but NEVER wrote a reflog, so a bare repo with
`core.logallrefupdates true` got no reflog. Added:
- `fetch_reflog_action_prefix(args)`: mirrors Git `default_rla` -- `GIT_REFLOG_ACTION`
  or `fetch <remote> <refspecs...>`.
- `write_fetch_explicit_reflog(...)`: writes `"<rla>: <action>"` where action is
  `storing {head,tag,ref}` (new ref, by remote ref name) / `fast-forward` /
  `forced-update`. Uses append_reflog with force_create=false so
  `should_autocreate_reflog` honors logallrefupdates (creates for refs/heads when
  true; skips when unset+bare).
Called in the explicit-dest path after apply_single_ref_update (non-atomic).

t5510-fetch fully passes (215/215) -- no fetch-reflog regression. 100/124.

## Fix 4: push with legacy .git/branches/<name> remote (subtests 81, 82)
The fetch side already read `.git/branches/<name>`; the push side bailed
"remote not found". Mirrored Git `read_branches_file` (remote.c) Cogito-compatible
push: refspec `HEAD:refs/heads/<frag>` where `<frag>` is the `#branch` part of the
file or the repo's default branch name. Added `read_branches_push_remote` +
`BranchesPushRemote` in push.rs and a fallback in `run()`: when `resolve_remote_urls`
fails for a bare name, try the branches file; set URL (push-url-rewritten) and inject
`HEAD:refs/heads/<frag>` as the default refspec when none given. 103/124.

## Fix 5: fetch will not clobber existing tag without --force (subtests 87, 88)
`git fetch origin tag <t>` expands to `refs/tags/<t>:refs/tags/<t>` (non-forced) and
flowed through the explicit-destination CLI path, which only did a fast-forward check.
Tags are not fast-forwardable, so when the moved tag's new target was a descendant of
the old, `is_ancestor` returned true and grit clobbered the tag. Added an explicit
tag-clobber guard in that path: if dst is under refs/tags/, the local tag exists and
differs, and the refspec is not forced (`+`/`--force`/tagOpt force), reject with
`! [rejected] (would clobber existing tag)` and push to tag_clobber_failures (which
bails "some local refs could not be updated"). The `+refs/tags/*:refs/tags/*` retry
passes force and succeeds. t5510-fetch still 215/215. 105/124.
