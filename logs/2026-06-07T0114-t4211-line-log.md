# t4211-line-log — ticket 9ea46c

Final: **53/53 passing** (was 30/53 at claim).

## Root causes & fixes

Three independent bugs in `git log -L` (line-log), all in `grit/src/commands/log.rs`
(+ one in `grit/src/main.rs`):

### 1. Double blank line between line-log commits (tests 3-12, 15-20, 37)
`run_line_log`'s non-graph loop printed a blank line *before* each commit
(`log_wants_blank_line_between_commits`) AND another blank *after* each commit's
diff. Result: two blank lines between regular commits, where Git uses one.
Fix: removed the post-diff blank; the single pre-commit separator is the only
separator (regular commits) and the merge case is handled separately (see #3).

### 2. Attached `-L<value>` form swallowed as a revision (tests 2, 41)
`-L1,1:b.c` (no space) was captured by clap's hyphen-tolerant `revisions`
positional instead of the `-L` option, so `line_range` stayed empty and the
incompatibility checks (`-L` + pathspec / `--follow` / `--raw`) never ran.
Fix in `main.rs` `preprocess_log_args`: generalized the existing `-L:` rewrite
to all `-L<value>` attached forms → `-L <value>` (`arg.starts_with("-L") &&
arg.len() > 2`). The validation already lived in `run_line_log`; it just never
got the chance to fire.

### 3. Line-log walk not topologically ordered (tests 14, 43, 44)
Git forces `--topo-order` for line-level traversal
(`revision.c`: `if (revs->line_level_traverse) { ...; revs->topo_order = 1; }`).
`run_line_log` used `walk_commits` (commit-date order), which only coincides with
topo order when timestamps strictly increase. t4211's "fancy rename following"
commits all share one timestamp, so date order interleaved the main/side lineages
whereas Git shows the whole side branch before rejoining main.
Fix: new helper `topo_order_line_log_commits` re-orders the already-filtered
walk set via the shared `rev_list` machinery with `OrderingMode::Topo` (verified
to match `git log --topo-order` byte-for-byte), projecting the filtered set onto
the topo order. The commit *set* is unchanged (preserves --before/--skip/etc.);
only the order changes.

### Merge separator (test 14 final blank)
After #1, a merge commit (no patch in line-log) lost its second trailing blank.
Git emits the empty diff preamble (one blank where the patch would be) after a
merge's message. Re-added: one blank after a merge commit in the line-log patch
path (suppressed for the last commit). Combined with format_commit's
message-trailing blank this reproduces Git's two blank lines after a merge.
