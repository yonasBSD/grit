## Fixes (all in `grit/src/commands/`)

### update_ref.rs — subtests 28, 30–35
1. `update-ref --stdin` (and `-z`) now reject a badly formatted ref name with
   `fatal: invalid ref format: <ref>`, mirroring `parse_refname()` /
   `check_refname_format(REFNAME_ALLOW_ONELEVEL)` in `builtin/update-ref.c`.
   Added `validate_batch_refname()` called at the top of `process_batch_command`.
   For `-z`, the raw refname (including any trailing whitespace, e.g. `~a `) is
   extracted from the command field so the error text matches Git exactly
   (subtest 33 greps for the trailing space).
2. Rewrote the `-z` batch parser: `group_nul_commands()` now assembles each
   logical command from the correct number of NUL fields (the first argument
   shares the command field; `args-1` more fields follow), matching the C
   `command[]` arg-count table. Previously each NUL field was treated as a
   whole command, so multi-field `-z` commands errored before validation ran.
3. `validate_delete_refname()` now implements `refname_is_safe()` from refs.c:
   a delete of a one-level name like `my-private-file` is refused with
   `refusing to update ref with bad name '<ref>'` (subtest 28). Previously it
   accepted any name that passed `check_refname_format`, so it deleted the file.

### checkout.rs — subtest 37
`validate_new_branch_name()` now rejects `HEAD` (`refs/heads/HEAD`) and
leading-dash names, mirroring `check_branch_ref()` in refs.c, so
`checkout -B HEAD` fails as required.

### branch.rs — subtest 39
`delete_branch()` no longer dereferences a literal `HEAD` argument via
`symbolic_full_name()`. `git branch -d HEAD` now targets `refs/heads/HEAD`
instead of the symbolic-HEAD's branch (which made it refuse with
"used by worktree").

## Blocker (subtests 21–27, all symref/dangling-symref deletion)

grit's actual behavior is already correct and matches real Git 2.52. Each of
these subtests fails at the precondition `test_ref_exists refs/heads/badname`,
where `badname` is a dangling/broken symref. The local `tests/test-lib.sh`
defines `test_ref_exists` as `git rev-parse --verify -q "$ref"`, which (in both
grit AND real git) exits 1 on a dangling symref with
`warning: ignoring dangling symref`. Upstream `git/t/test-lib-functions.sh`
instead defines it as `git show-ref --exists "$ref"` (exit 0 here), and
`test_ref_missing` checks specifically for `show-ref --exists` exit code 2.

Verified against `/opt/homebrew/bin/git` 2.52.0:
- `rev-parse --verify -q` dangling symref → exit 1 (warning)
- `show-ref --exists` dangling symref → exit 0

grit matches Git exactly on both. The actual delete operations all succeed:
e.g. `git update-ref --no-deref -d refs/heads/badname` and
`git branch -d badname` (prints `Deleted branch badname (was refs/heads/broken...ref).`)
both work and leave the ref missing. Only the divergent local helper fails.

Cannot be fixed without modifying `tests/test-lib.sh` (forbidden) or changing
grit's `rev-parse --verify -q` to diverge from Git. Left for mop-up.
