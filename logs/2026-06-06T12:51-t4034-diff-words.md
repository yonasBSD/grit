# t4034-diff-words — bibtex word-diff alignment (ticket 31ea5c)

## Root cause
Word diff for the `bibtex` fixture produced a *different but equally minimal*
word-level alignment than upstream Git. Concretely, the tail
`year={1987},` → `year=1987, note={...The {\em pre}...}` matched the PRE `{`
to the FIRST `{` (the `note={` brace) instead of the SECOND `{` (the `{\em`
brace), so the green/red runs landed differently from Git's reference output.

grit's word diff went through `word_diff_ops_imara` (imara-diff Myers +
`postprocess_lines`). imara's Myers and Git's xdiff Myers both produce minimal
edit scripts, but pick different tie-breaks on streams with many repeated tokens
(`{`, `}`, `,`, `=`). Crucially, Git runs `xdl_cleanup_records` *before* the
Myers split — it discards records (words) that have no match on the other side —
which reshapes the effective stream and changes which `{`/`}` align. imara has no
equivalent preprocessing, so it cannot reproduce Git's choice via algorithm or
direction tweaks (verified: forward, reversed, and Histogram all diverge).

## Fix
Ported Git's xdiff Myers pipeline faithfully into `grit-lib/src/diff.rs`
(new private `git_xdiff` module), restricted to what the word-diff path needs
(no rename / ignore-whitespace flags):
- record classification (`DISCARD`/`KEEP`/`INVESTIGATE`) by cross-file
  occurrence counts,
- `xdl_trim_ends` + `xdl_cleanup_records` (incl. `xdl_clean_mmatch`),
- `xdl_recs_cmp` divide-and-conquer + `xdl_split` middle-snake finder
  (constants `XDL_SNAKE_CNT`, `XDL_K_HEUR`, `XDL_MAX_COST_MIN`, etc.).
`word_diff_ops_imara` now interns word tokens to ids, runs `git_xdiff::changed_flags`,
converts the per-token `changed` flags to `similar::DiffOp`s, then applies the
existing `apply_change_compact_to_ops` (indent heuristic off, as Git does for
word diff). Removed the old imara-based body. The public function name/signature
is unchanged, so all word-diff callers are unaffected.
