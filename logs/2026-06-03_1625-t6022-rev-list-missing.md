# t6022-rev-list-missing

- Claimed after completing `t6600-test-reach.sh`.
- Current CSV baseline: 40 total, 16 passing, 24 failing.
- Initial focus: inspect upstream missing-object behavior for `rev-list` and group failures by
  missing commit, tree, blob, promisor, and ignore-missing modes before making code changes.

- Direct debug run showed the first cluster failing because `--missing=allow-any`/`print` still
  aborted when revision traversal reached missing commits, and because segmented `--objects` output
  ignored parent-closure subtraction for ranges like `HEAD ^HEAD~1`.
- Implemented missing-tolerant revision closure walks for non-error missing modes and applied the
  existing parent-reachable object subtraction to segmented object output.
- Direct debug run improved `t6022-rev-list-missing.sh` from 16/40 to 26/40.
- Official harness `./scripts/run-tests.sh t6022-rev-list-missing.sh --verbose` confirms 26/40 and
  refreshed `data/test-files.csv` plus dashboards.
- Remaining failures: missing tree/blob object cases, missing tree/blob tips with an existing tip,
  `--missing=print-info`, and `-z` missing-info output.
- Verification: `cargo fmt`, `cargo check -p grit-cli`, `cargo test -p grit-lib --lib`,
  `cargo clippy --fix --allow-dirty`, and `cargo build --release -p grit-cli` ran. Clippy reported
  the existing warning backlog and unrelated auto-fixes; unrelated applied fixes were removed.

- Finished the remaining missing tree/blob clusters by resolving negative `--objects` specs with
  object-aware parsing and subtracting tree/blob object-root closures from collected object output.
- Fixed `rev:path` object roots so tree paths such as `HEAD:foo` are accepted as tree roots instead
  of being hard-coded as blobs.
- Added `--missing=print-info` and `-z` output formatting, including inferred path/type metadata
  for missing commit, tree, and blob objects.
- Direct debug run now passes all tests: `cd tests && sh t6022-rev-list-missing.sh` reports 40/40.
- Official harness `./scripts/run-tests.sh t6022-rev-list-missing.sh` confirms 40/40 and refreshed
  `data/test-files.csv` plus dashboards.
