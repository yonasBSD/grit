# t4201-shortlog — close out (subtests 8 & 9)

Ticket: 80b9b4. Final state: 32/32 passing.

## Starting state
Prior agent fixed subtest 7 (commit stores invalid-UTF-8 bodies verbatim).
Remaining failures: 8 (`shortlog from non-git directory`) and 9
(`shortlog can read --format=raw output`). Both pipe `git log` output into
`git shortlog -w` reading from stdin; the expected output is test 7's `expect`,
which contains the raw bytes `f8 9d 84 9e` (commit 4's invalid UTF-8, stored
under `i18n.commitEncoding=non-utf-8` — `logmsg_reencode` is a no-op so Git
emits the body verbatim).

## Root causes (two distinct bugs)

1. **`grit log` lossily decoded the body.** `parse_commit` already keeps verbatim
   bytes in `CommitData.raw_message`, but the log formatter wrote `CommitInfo.message`
   (a `String` produced by `from_utf8_lossy` for the unknown encoding), turning
   `f8 9d 84 9e` into U+FFFD (`ef bf bd`). So the `log` input fed to shortlog was
   already corrupted.

2. **`grit shortlog`'s stdin reader was String-based.** `read_from_stdin` did
   `reader.lines().map_while(|l| l.ok())`, which on an invalid-UTF-8 line errors
   and `map_while` *stops* — silently dropping every commit after commit 4 (output
   was only 7 lines instead of the full summary).

## Fixes

- `grit-lib/src/tab_expand.rs`: new `indent_and_expand_tabs_bytes(line, indent,
  tab_width)` — byte-level port of Git's `pp_handle_indent`/`strbuf_add_tabexpand`.
  Emits the body verbatim (no lossy UTF-8); for `tab_width>0` it aligns tabs by the
  display width of the preceding bytes and gives up alignment once a pre-tab segment
  is not well-formed UTF-8 (matching `pp_utf8_width < 0`). Added 3 unit tests.
- `grit/src/commands/log.rs`: added `raw_message: Option<Vec<u8>>` to `CommitInfo`
  (populated from `CommitData.raw_message` at all 5 construction sites) and a
  `write_commit_body` helper used by the `raw`/`medium`/`full`/`fuller` arms of
  `format_commit`. It emits `raw_message` bytes verbatim when present, otherwise the
  existing `message.lines()` string path (byte-identical for valid-UTF-8 commits).
- `grit/src/commands/shortlog.rs`: rewrote `read_from_stdin` to read raw bytes
  (`read_to_end`), split on LF (Git `strbuf_getline_lf`), match the `Author:`/`author`
  prefix at the byte level, and store onelines as `&[u8]` (`insert_record` already
  takes bytes). Idents still go through the String-based mailmap (test idents are
  ASCII; Git treats idents as bytes for grouping only).
