# t1309-early-config — read_early_config repo-version warning

Ticket: 89f2d2 (config thread B). Baseline: 9/10. Result: 10/10.

## Failing subtest
not ok 6 - ignore .git/ with incompatible repository version
  test_with_config "[core]repositoryformatversion = 999999" 2>err &&
  test_grep "warning:.* Expected git repo version <= [1-9]" err
-> grit produced empty stderr (no warning).

## Root cause
The subtest runs `test-tool config read_early_config early.config`, which routes
through `run_test_tool_config` in grit/src/main.rs. That `read_early_config`
branch resolved the git dir with `Repository::discover(None).ok()`.

`Repository::discover` runs `validate_repository_format` on the found gitdir
(repo.rs:358) and returns `Err` for `repositoryformatversion = 999999`. So
`.ok()` yielded `None`, and `ConfigSet::read_early_config` was called with
`git_dir = None`. With no git_dir it never reaches the per-repo branch that calls
`early_config_ignore_repo_reason` and prints the
`warning: ignoring git dir '...': Expected git repo version <= 1, found N`
message. Hence empty stderr.

This diverges from Git: `read_early_config` uses
`discover_git_directory_reason`, which still *records* the gitdir under
`GIT_DIR_INVALID_FORMAT` (setup.c), then warns and skips that repo's local
config while still reading the non-repo (global/xdg/system) layers.

## Fix
grit/src/main.rs `run_test_tool_config`, `read_early_config` branch: locate the
gitdir with the validation-free `test_tool_config_locate_git_dir()` (already used
by the other `test-tool config` subcommands). It tries `Repository::discover`
first (valid repos), then falls back to
`precompose_config::locate_git_dir_from_cwd`, which walks up for `.git` honoring
GIT_DIR and GIT_CEILING_DIRECTORIES but without format validation. The downstream
`ConfigSet::read_early_config` then sees the gitdir, calls
`early_config_ignore_repo_reason`, emits the warning, and skips the bad repo's
local config — exactly as before for valid repos and ceiling cases.

The library function `early_config_ignore_repo_reason` (repo.rs) and the warning
text already existed (landed with the t1302 fix); only the CLI wiring needed the
gitdir to survive format validation.
