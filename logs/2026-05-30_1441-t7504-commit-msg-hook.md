## Root cause
Both merge and `rebase -i` reword build commits in-process and skipped the
commit-msg hook that the upstream commit pipeline runs.

## Changes

### grit/src/commands/merge.rs (subtest 28)
Added `run_merge_commit_msg_hooks(repo, msg, will_launch_editor, no_verify, cleanup_mode) -> Result<String>`,
mirroring upstream `builtin/merge.c:prepare_to_commit`:
1. Write the built message to `MERGE_MSG`.
2. Run `prepare-commit-msg <MERGE_MSG> merge` (always, even with `--no-verify`).
3. When `!no_verify`, run `commit-msg <MERGE_MSG>`; bail on non-zero exit.
4. Re-read `MERGE_MSG` and re-apply cleanup so a hook that rewrites the file wins.

Wired into every merge-commit-building path right before the commit is
serialized: `do_real_merge`, the trivial/in-index path, `do_octopus_merge`,
`do_strategy_ours`, `do_strategy_theirs`. The `--no-commit`/conflict paths that
write `MERGE_MSG` and stop are untouched (the hook runs from the later
`git commit`). `--no-verify` still skips commit-msg (subtests 16/17 stay green).

### grit/src/commands/rebase.rs (subtest 30)
Added `run_reword_commit_msg_hook(repo, git_dir, message, config) -> Result<String>`,
mirroring upstream `sequencer.c` reword (`git commit --amend` with `VERIFY_MSG`):
write `COMMIT_EDITMSG`, run `commit-msg <COMMIT_EDITMSG>`, and on success re-read +
re-apply `rebase_commit_msg_cleanup`. Reword always verifies upstream, so this runs
unconditionally. Wired into all three reword sites: the trivial-replay path and the
post-merge path in `cherry_pick_for_rebase`, and the `--continue` reword handler.
Imported `run_commit_hook` + `CommitHookEnv` from `grit_lib::hooks`.
