## Work done (grit/src/commands/add_patch.rs)

### 1. Rewrote the hunk loop to mirror `git/add-patch.c::patch_update_file`
- Added a per-hunk `Decision { Undecided, Use, Skip }` state (git's
  UNDECIDED/USE/SKIP_HUNK) replacing `accepted: Vec<bool>`.
- Built the prompt navigation suffix dynamically and order-sensitively

  `(was: y)`/`(was: n)` annotation for already-decided hunks.

- Print a trailing newline on every exit from the file (git's final `putchar('\n')`).

### 2. Natural hunk splitting (`natural_hunk_ranges`)
The initial `hunk_ranges` are now the hunks `git diff -U<context>` would emit (changes
separated by an equal run > 2*context start a new hunk), so navigation works without `s`
first. Boundary equal runs are SHARED between adjacent hunks so each renders its
surrounding context; equal ops never consult `accepted` during blending, so the overlap is
harmless to staging.

### 3. Correct absolute hunk headers (`render_hunk_with_offsets`)
`partial_unified_for_op_range` rebuilt `@@` offsets from 1, giving wrong summary/headers
after a split. New renderer emits absolute `@@ -o,c +o,c @@` (and `-0,0` for pure
insertions), matching `git diff` and the `g`-list summaries (`-2,4 +3,8 ... +21`).

### 4. Full split (`split_hunk_into_all`) and no-op edit detection
- `s` now splits a hunk into ALL of its `splittable_into` sub-hunks (shared boundary
  context), not just the first gap.
- `e` returns an `EditResult { Unchanged, Edited, Aborted }`. A no-op editor (`EDITOR=:` /
  `touch`) now STAGES the hunk as-is (git's `hunk->use = USE_HUNK`) instead of leaving it
