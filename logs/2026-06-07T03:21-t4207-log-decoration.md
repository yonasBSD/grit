## Verification that grit's decoration machinery is CORRECT

Reproduced the test setup with the release binary under
`GIT_TEST_DEFAULT_INITIAL_BRANCH_NAME=main` (harness forces this; run-tests.sh:377):

- `git log --decorate --oneline -n 1` ->
  `17e49cf (HEAD -> main, tag: lightweight-tag, tag: annotated-tag, feature) third`
  (HEAD arrow, both tags incl. peeled annotated tag, branch all correct).
- `git log --decorate=short --oneline -n 1` -> same short-form decoration.
- `git log --no-decorate --decorate --oneline -n 1` -> last-wins enables decoration. Correct.
- `git log --oneline -n 1 >out` (to a FILE, no --decorate) -> `17e49cf third`, NO decoration.

Cross-checked against real system git 2.39.5: `git log --oneline -n 1 >out` to a file emits
`<hash> first` with NO decoration. grit matches upstream exactly.

Doc ground truth (git/Documentation/git-log.adoc:43-49): default is `--decorate=auto`;
`auto` = decorate "if the output is going to a terminal ... otherwise no ref names are shown."
grit TTY-gates this correctly in log.rs::resolve_decoration_display (L1964-1972).

## Two irreconcilable root causes (both in the TEST file, out of allowed scope)

CAUSE A — oneline-to-file is not decorated (correct `auto` behavior). Affects tests
2, 4, 5, 6, 7, 16, 18, 19. They run `git log --oneline [...] >out` with NO `--decorate`
then grep for decoration content. Upstream + grit both leave piped/redirected output
undecorated. Forcing unconditional decorate-to-file would REGRESS t4202-log.sh
(t4202 `oneline` L105-108 and `log.decorate configuration` L808+ assert bare
`git log --oneline >actual` has NO decoration via test_cmp; t4202 is in_scope, 129/149).
The two invocations are byte-identical in form, so no conditional fix can distinguish them.

CAUSE B — grep "master" but branch is "main". Affects tests 3, 8, 13, 21. The harness
forces init.defaultBranch=main, so `git init repo` creates branch `main`. grit DOES
decorate these (8 and 13 use explicit `--decorate`/`--decorate=short`; output contains
`(HEAD -> main, ...)`), but `grep "master"` can never match `main`.

## Proper fix (TEST-AUTHORING, not grit — escalate)

In tests/t4207-log-decoration.sh: add explicit `--decorate` to the `--oneline`
invocations in tests 2,4,5,6,7,16,18,19; and replace `grep "master"` with `grep "main"`
(or `git init -b master repo`) in tests 3,8,13,21. Not permitted under the One Rule
(only test_expect_failure->test_expect_success edits allowed), and there are no
test_expect_failure markers here to flip.
