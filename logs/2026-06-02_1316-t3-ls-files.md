## Changes
1. ignore.rs: IgnoreMatcher::check_path now implements Git's prep_exclude
   parent-directory abort. Extracted build_groups/evaluate_against_groups/
   flatten_groups helpers and added first_excluded_ancestor: when an ancestor
   directory matches a non-negative exclude pattern, the whole subtree is
   excluded and a later file-level negation (e.g. `!one/a.1`) can no longer
   re-include it. Ancestor probing slices per_dir_groups to the ancestor's own
   depth to mirror Git's incremental .gitignore load order.
   -> Fixes t3001 #12. Keeps the full check-ignore family green (t0008 398/398,
   t12200, t8480, t10640, t10940, t11310, t12800, t13400 all unchanged).

2. ls_files.rs: `--ignored --directory` collapse. Git lists a directory as
   `dir/` only when the directory itself is ignored AND has no tracked content
   under it; directories that merely contain ignored files are recursed into so
   individual ignored files are listed. Added a `recurse_nonempty_dirs` walk
   flag (forces recursion in ignored mode while still emitting empty-dir
   markers) and a post-walk collapse that maps each ignored entry onto its
   shallowest ignored, tracked-free ancestor directory.
   -> Fixes t3001 #19/#22/#26.

3. ls_files.rs: is_modified() gitlink handling. Git's ce_match_stat_basic
   lstats a gitlink's worktree path: if missing or not a directory (e.g. a
   mode-160000 entry added via `update-index --cacheinfo` with nothing checked
   out) the entry is TYPE_CHANGED = modified. Only a directory path triggers the
   submodule-HEAD comparison.
   -> Fixes t3013 #17 (the missing o5.txt gitlink under `-m`).
