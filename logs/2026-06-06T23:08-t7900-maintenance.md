# t7900-maintenance — geometric repack MIDX fix

Ticket: e42df4 (test, t7)
Date: 2026-06-06T23:08 UTC

## Symptom

`./scripts/run-tests.sh t7900-maintenance.sh` reported 71/72. The single failing
subtest was #29 "geometric repacking task". With stderr captured, the failing
geometric `git maintenance run --task=geometric-repack` printed:

```
error: could not load pack N
error: corrupt object: could not load pack N
error: command failed: repack -d -l --geometric=2 --quiet --write-midx
```

The test discards stderr (`2>/dev/null`), so the harness only saw the non-zero
exit break the `&&` chain.

## Root cause

`run_geometric` in `grit/src/commands/repack.rs` wrote the multi-pack-index via
`write_multi_pack_index_with_options` WITHOUT an explicit pack list. In that mode
`grit-lib/src/midx.rs::write_multi_pack_index_with_options` reads the *existing*
MIDX and fails ("could not load pack N", grit-lib/src/midx.rs:2116-2138) if any
pack it references no longer has a `.pack` on disk.

A prior geometric run wrote a MIDX covering packs A,B,C,D, then deleted C,D as
redundant via `remove_geometry_redundant` WITHOUT rewriting/clearing the MIDX. So
the on-disk MIDX kept referencing the deleted C,D. The next geometric run, when it
went to write a fresh MIDX, inherited those stale entries and errored.

Upstream git (git/repack-midx.c `write_midx_included_packs` -> `midx_included_packs`)
never inherits the existing MIDX's pack list: it writes the MIDX from an explicit
`multi-pack-index write --stdin-packs` list = kept packs + newly written packs +
surviving above-split packs + cruft packs. Packs being deleted (and packs a prior
run already deleted) are naturally excluded.

## Fix

In `grit/src/commands/repack.rs`:

- Added `geometric_midx_included_idx_names(...)` helper that builds the explicit
  `.idx` include list mirroring `midx_included_packs`: kept packs first, then
  newly written packs, then every other surviving local pack on disk
  (scanned from the pack dir, EXCLUDING the below-split stems this run rolls up).
  Scanning the directory rather than only grit's geometry-ranked `pack-*` packs is
  required because grit's geometry skips non-`pack-*` local packs (e.g. the
  `loose-*` pack the loose-objects task writes) — those always survive and must
  appear in the MIDX. Cruft (`*.mtimes`) packs are kept too.
- Both geometric MIDX-write sites (the normal `wrote_pack` path and the
  `!should_run_pack_objects` path) now pass `pack_names_subset_ordered =
  Some(include)` plus `preferred_pack_name` derived from `pref_stem`, instead of
  the previous `preferred_pack_idx` + directory-scan-and-inherit behavior. With a
  subset set, the writer skips the inherit-existing-MIDX consistency check and only
  indexes packs that actually exist.

## Why the first attempt regressed test 71

A first version of the helper included ONLY geometry-ranked packs (kept + new +
above-split `pack-*`). That excluded the `loose-*` pack, so the default-strategy
geometric repack in test 71 ("repacking loose objects is quiet") produced an empty
include list and errored "no pack files to index." Fixed by scanning the directory
for all surviving local `.idx` files.
