## Work done

### ignore-blank-lines (DONE for 46-52)
Root cause: `--ignore-blank-lines` was implemented as post-hoc "drop whole hunk
if all changes ignorable" (`suppress_ignored_hunks_in_patch`). Git instead marks
each xdiff change-record's `ignore` flag and runs `xdl_get_hunk` to drop ignorable
changes that are far from substantive ones while KEEPING adjacent ignorable
changes and recomputing hunk boundaries/line-numbers.

Implemented a faithful port of Git's xdiff change-record machinery
(`xdl_mark_ignorable_lines` + `xdl_get_hunk` + `xdl_emit_diff`) in
`grit-lib/src/diff.rs`:
- `histogram_unified_body_ignore` (body only, ignore predicate per change record)
- `unified_diff_histogram_ignore_with_prefix_and_funcname` (full patch + funcname)

Key constant: `max_ignorable = ctxlen` (NOT inter-hunk-context); `max_common =
2*ctxlen + interhunk`. The leading-ignorable-drop loop must walk ALL leading
ignorable changes (advance `*xscr` past each whose successor is absent or
`>= max_ignorable` away), not break early.

Wired into `grit/src/commands/diff.rs` `write_patch_with_prefix`: when only
`--ignore-blank-lines`/`-I` is active (no `-b/-w/...`, no `-W` function-context),
route through the new function. Other cases keep the old post-hoc suppression.

Result: 107 -> 112. Tests 46,47,48,50,52 now pass.

## 136 FIXED -> t4015 is 136/136 (FULLY PASSING)
Root cause was three independent bugs in grit's function-context machinery:
1. `unified_diff_with_prefix_and_funcname_and_algorithm` returned the histogram
   body printer BEFORE checking `function_context`, so `-W` was silently dropped
   for histogram diffs (the `--no-index` path). Fixed by checking
   `function_context` first.
2. `unified_diff_with_function_context` grouped changes with
   `similar::group_diff_ops(radius)` (splits at gap > 2*radius = 12), which
   over-merged the leading `+blank` (a separate function) with the function-body
   changes. Switched to the gap-correct `group_diff_ops_gap` (splits at gap >
   2*ctx+inter = 6, like git's xdl_get_hunk).
3. The post-expansion old-side end was `base_e1 + ctx`, double-counting context
   (`base_e1` already includes the group's trailing context), over-extending the
   leading hunk so it overlapped/merged the body hunk. Changed to `i1_end + ctx`
   (git's `xche->i1 + xche->chg1 + lctx`). Added an overlap-merge of the
   resulting ranges for the cases where function expansion genuinely overlaps.
With these, `--no-index --function-context` emits the two separate hunks git
emits, and the post-hoc suppress_ignored_hunks_in_patch drops the all-ignorable
leading `+blank` hunk. Verified no regression: t4051-diff-function-context
42/42, t4061-diff-indent 33/33, t4013-diff-various 230/230, t4032 37/37.
(The histogram_unified_body_ignore_fc `-W` scaffold added earlier is unused at
function_context=true and harmless; could be removed in cleanup.)
