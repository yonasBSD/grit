## Work done

### ignore-blank-lines (DONE for 46-52)
Root cause: `--ignore-blank-lines` was implemented as post-hoc "drop whole hunk
if all changes ignorable" (`suppress_ignored_hunks_in_patch`). Git instead marks
each xdiff change-record's `ignore` flag and runs `xdl_get_hunk` to drop ignorable
changes that are far from substantive ones while KEEPING adjacent ignorable
changes and recomputing hunk boundaries/line-numbers.

Implemented a faithful port of Git's xdiff change-record machinery
(`xdl_mark_ignorable_lines` + `xdl_get_hunk` + `xdl_emit_diff`) in
`grit-lib/src/diff.rs`:
- `histogram_unified_body_ignore` (body only, ignore predicate per change record)
- `unified_diff_histogram_ignore_with_prefix_and_funcname` (full patch + funcname)

Key constant: `max_ignorable = ctxlen` (NOT inter-hunk-context); `max_common =
2*ctxlen + interhunk`. The leading-ignorable-drop loop must walk ALL leading
ignorable changes (advance `*xscr` past each whose successor is absent or
`>= max_ignorable` away), not break early.

Wired into `grit/src/commands/diff.rs` `write_patch_with_prefix`: when only
`--ignore-blank-lines`/`-I` is active (no `-b/-w/...`, no `-W` function-context),
route through the new function. Other cases keep the old post-hoc suppression.

Result: 107 -> 112. Tests 46,47,48,50,52 now pass.
