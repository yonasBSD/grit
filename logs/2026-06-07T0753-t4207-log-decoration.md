## The two unfixable-in-grit causes

(A) Tests 2,4,5,6,7,16,18,19 run `git log --oneline >out` with NO `--decorate` (output to a FILE),
then grep for decorations. Upstream default is `--decorate=auto`, and
`git/builtin/log.c:auto_decoration_style()` (L75-78) returns decoration ONLY when
`session_is_interactive()` = `isatty(1) || pager_in_use()` (L70-72). To a redirected file that is
false → NO decoration. grit mirrors this exactly in
`grit/src/commands/log.rs::resolve_decoration_display` L1965-1973 (`IsTerminal::is_terminal(&stdout())`).
Forcing unconditional decorate-to-file would REGRESS t4202-log.sh (currently 143/149): its `oneline`
test (L105-108) and `log.decorate configuration` test (L808+) assert `git log --oneline >actual`
has NO decoration. That invocation is byte-identical to t4207's, so no conditional grit fix can
satisfy both.

(B) Tests 3,8,13,21 grep `master`, but the harness forces `GIT_TEST_DEFAULT_INITIAL_BRANCH_NAME=main`
(run-tests.sh:378 → test-lib.sh:322-324 sets `init.defaultBranch=main`). The branch is `main`;
`grep master` can never match. Tests 8/13/21 use explicit `--decorate` and grit DOES emit the
decoration with `main` — the grep target is simply wrong.

## Proper fix (out of allowed scope)

`test_expect_success`, so there is no in-scope edit. No grit Rust change is warranted. Ticket left OPEN
