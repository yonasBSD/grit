# t7001-mv

Ticket: afb336

## Start

- Claimed ticket for `tests/t7001-mv.sh`.
- Reproducing current failures before editing code.

## Findings

- Reproduced `t7001-mv.sh` at 51/54 with failures in subtests 49, 51, and 52.
- Subtest 49 preserved the moved-away populated submodule directory, but checkout skipped Git's
  `rmdir` warning and then recreated the old submodule gitfile in an empty placeholder directory.
- `git status -s sub2` leaked dirty submodule entries because short-format submodule synthesis did
  not apply the user's pathspec, and status argv preprocessing forced ordinary pathspecs behind
  `--`.
- Directory-expanded `git mv` rows for submodules were marked index-only, so `.gitmodules` and the
  moved submodule's `core.worktree` were not refreshed when moving a parent directory.

## Changes

- Made checkout use Git-style `rmdir`-or-warn behavior for preserved dropped gitlinks.
- Avoided refreshing submodule gitfiles for empty checkout placeholder directories; `git submodule
  update` initializes those later.
- Fixed `status` pathspec parsing and applied pathspec filtering to short-format synthesized
  submodule dirtiness.
- Updated `git mv` to refresh `.gitmodules`, submodule gitfiles, and `core.worktree` for gitlinks
  moved as part of an index-only directory expansion.

- `./scripts/run-tests.sh t7001-mv.sh`: 54/54 passing.
