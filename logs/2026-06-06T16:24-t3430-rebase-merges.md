# t3430-rebase-merges (ticket 029fa4)

Subsystem: rebase-core (interactive rebase / sequencer, `--rebase-merges`).

## Fresh run baseline
4/34 at start (after earlier shared-machinery fixes from sibling tickets).

## Root causes found & fixed (grit/src/commands/rebase.rs)

1. `merge -C <ref>` where `<ref>` is an annotated tag (e.g. `merge -C H`, with `H` an
   annotated tag) crashed the whole rebase with `corrupt object: commit missing author
   header`. The Completed / Conflict arms of the `MergeReuseMessage` step, plus
   `rewrite_merge_head_for_replay_opts`, read `merge_oid` and `parse_commit`'d it directly.
   Fix: `peel_to_commit_for_merge_base` before parsing in all three spots.

2. Todo-list generation for `--rebase-merges` produced wrong text:
   - `merge -C <hash> <label>  # <subj>` had a double space before ` # ` (built
     `"merge -C {hash} "` then pushed `label + ' '` then ` # subj`). Rebuilt to join with
     single spaces.
   - `reset <label> # <hash> # <subj>` instead of `reset <label> # <subj>` — was using the
     full `pick <hash> # <subj>` line as the comment. Added `reset_comment_subject` to take
     just the subject.

3. ORIGINAL-TODO / git-rebase-todo lacked the instruction help comment block, so after
   `grep -v '^#'` the expected trailing blank line was missing. Added `append_rebase_todo_help`
   (mirrors git `append_todo_help` in rebase-interactive.c) and wired it into
   `run_interactive_rebase_with_initial_todo` (new `help_revs_onto` param).
