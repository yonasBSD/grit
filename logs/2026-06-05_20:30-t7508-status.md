# t7508 status

Ticket: 2befa1

Remaining failures from the ticket were subtests 13, 65, and 119-121.

Investigated the commit-template failures first. The failing assertions were seeing stale `output`
from the preceding `git status` command, which means the fake editor did not overwrite the file in
the full harness run. A gated editor-resolution trace showed `VISUAL=:` and `EDITOR=.git/editor`,
but commit launch resolved the editor as `:` because it was using the non-launch resolver mode.
Changed commit editor launch to use launch-mode editor resolution, which ignores harness
`VISUAL=:` / `EDITOR=:` placeholders while preserving explicit `GIT_EDITOR=:` as a no-op.

For subtests 119-121, `git --no-optional-locks status` correctly preserved the magic index mtime,
but a normal `git status` also preserved it. The status refresh path only wrote the index when
entry stat fields changed. Git also rewrites when entries are racy relative to the index file mtime,
which advances the index mtime even if cached stat data did not change. Added the same racy
timestamp write condition used by `update-index --refresh` to `status`.

Final harness run: `./scripts/run-tests.sh t7508-status.sh` passes 126/126 and updates
`data/tests/t7/t7508-status.toml` to `fully_passing = true`.
