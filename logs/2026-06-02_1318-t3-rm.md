## What changed in rm.rs

1. **SIGPIPE / broken pipe (subtests 35, 36).** `rm '<path>'` lines are now
   emitted through a fallible `writeln!` to a single locked stdout handle. On a
   closed pipe the `io::Error(BrokenPipe)` propagates up; `main.rs` already maps
   broken-pipe errors to exit code 128+13, matching Git's SIGPIPE convention,
   instead of the previous `println!` panic. Dry-run never locks the index, so no
   `index.lock` is left behind.
   (A direct `libc::signal(SIGPIPE, SIG_DFL)` reset is impossible here — the
   workspace sets `unsafe_code = "forbid"`, which `allow` cannot override.)

2. **Empty pathspec (subtest 82).** `git rm ""` now errors with
   `fatal: empty string is not a valid pathspec...` and exits 128, mirroring
   Git's `parse_pathspec`.

3. **Dangling vs. real symlinked leading path (subtests 70, 72, 73).** The
   leading-symlink check now bails only when the symlink *resolves* to an
   existing directory. A dangling leading symlink (`d -> nonexistent`) falls
   through to index-only removal; a symlink to a real dir still refuses.

4. **Submodule removal (subtests 39-42, 45, 46, 49, 51, 52, 53, 55).**
   - Gitlink modification check rewritten to mirror Git's `ce_compare_gitlink` +
     `bad_to_remove_submodule`: an empty/unpopulated submodule dir is *not*
     modified (removable without -f); a different submodule HEAD, a dirty work
     tree (`git status --porcelain` inside), or an embedded `.git` directory each
     require `--force`.
   - On full removal of a gitlink, its `[submodule "<name>"]` section is dropped
     from `.gitmodules` and the file re-staged (`remove_path_from_gitmodules` +
     `stage_updated_gitmodules`); a missing section warns but is non-fatal; a
     missing `.gitmodules` is silent; `--cached` leaves `.gitmodules` untouched.
   - Pre-check: refuse when `.gitmodules` is tracked but has unstaged worktree
     modifications (`is_staging_gitmodules_ok`).
   - Conflicted (unmerged) submodules: gitlink mode is now detected at any stage
     (1/2/3, not just 0), so they match without `-r`, get the `.gitmodules`
     cleanup, and are safety-checked against their stage-2 ("ours") OID.
