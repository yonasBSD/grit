# t5616-partial-clone — manual prefetch of missing objects

Date: 2026-06-06T09:21 UTC
File: tests/t5616-partial-clone.sh
Result: 47/47 fully passing (was 46/47).

## Failing subtest

Test 14 "manual prefetch of missing objects":

    git -C pc1 fetch --filter=blob:none origin &&
    git -C pc1 rev-list --quiet --objects --missing=print main..origin/main >revs &&
    ... sort >observed.oids &&
    test_line_count = 6 observed.oids   # grit produced 7

After `fetch --filter=blob:none`, grit reported 7 missing blobs in `main..origin/main`
where upstream reports 6. The extra missing object was `5cc725050055b08433ac3e866bb20f4910a56282`,
the final version of `file.1.txt` at `origin/main`.

## Root cause

The blob was fetched dynamically by an earlier step (test 10 `git diff ... -- file.1.txt`,
which lazily fetches missing blobs). grit wrote that lazily-fetched object as a plain **loose
object** with no `.promisor` marker. In a partial clone the blob is logically a *promisor object*
(it lives, one hop down, under a tree that came from the promisor pack).

Later, the auto-maintenance / `gc` run after `fetch` runs the partial-clone repack path
(`grit/src/commands/gc.rs::run_promisor_merge_repack_gc`): it repacks objects that are literally
in `.promisor` packs into a new promisor pack, then runs `pack-objects --all
--exclude-promisor-objects` for everything else. `5cc7250` was a loose (non-promisor) object, so
it was NOT included in the new promisor pack; and because it is a promisor object (reachable via a
promisor tree), `--exclude-promisor-objects` dropped it from the main pack too. The old regular
pack holding it was then deleted as superseded — the blob vanished entirely.

Upstream Git avoids this because its lazy fetch (and `cat-file`/`diff` dynamic fetch) writes
fetched objects into a `.promisor`-marked pack, so they survive gc inside the promisor pack.
Confirmed by running the identical test-14 sequence with real git 2.52: `5cc7250` ends up **in a
promisor pack** and is present after gc.

## Fix

`grit/src/commands/promisor_hydrate.rs`, `try_lazy_fetch_promisor_objects_batch` (the batch lazy
fetch used by `diff`/`blame` prefetch). The local-promisor fast path copied each fetched object as
a loose object and returned. Changed it to collect the successfully-copied OIDs and write them into
a `.promisor`-marked pack via `write_promisor_pack_for_local_oids` (mirroring the single-object
path `try_lazy_fetch_promisor_object`). Did NOT call `register_promisor_default_filter` in this
path — a plain (unfiltered) promisor remote must not gain a `blob:none` filter just from a lazy
copy (would break t0410 test 14's later `fetch server2`).

## Dead ends / notes

- First attempted to change `grit-lib/src/rev_list.rs` `--exclude-promisor-objects` to only drop
  *missing* objects. That broke t0410 test 23 ("rev-list stops traversal at promisor commit, tree,
  and blob") which requires PRESENT promisor-pack objects to be excluded from the object walk.
  Reverted. Git's `is_promisor_object` set DOES include present promisor-pack objects + one hop;
  the real divergence was object *placement* (loose vs promisor pack), not rev-list semantics.
