## Root cause
Grit had no delta-island machinery at all. `repack -i` / `--delta-islands` was
ignored (the `-i` flag fell into repack's `rest`/trailing args), and
`pack-objects` had no `pack.island` handling, so delta selection never restricted
cross-island deltas and never biased base preference toward superset islands.

## Changes
- New module `grit-lib/src/delta_islands.rs` â€” a port of Git's `delta-islands.c`:
  - Compiles `pack.island` regexes (left-anchored with `^`), reads `pack.islandcore`.
  - Assigns each ref to an island by last-matching regex (last-one-wins), island
    name = non-empty capture groups joined by `-`.
  - Deduplicates islands by the running-sum-of-leading-8-oid-bytes hash, assigns one
    bitmap bit per surviving island.
  - Propagates marks tagâ†’target, commitâ†’tree+parents (fixpoint over ancestry so a
    shared root accumulates the union of all descendant islands), treeâ†’entries.
  - Exposes `in_same_island(trg, src)` (trg island set âŠ† src island set),
    `delta_cmp(a, b)` (`island_delta_cmp` superset bias), and core-island membership.
- `grit/src/commands/repack.rs`: parse `-i`/`--delta-islands`, forward
  `--delta-islands` to pack-objects (also honors `repack.useDeltaIslands`).
- `grit/src/commands/pack_objects.rs`: `--delta-islands` flag; load islands once;
  - delta-base selection (reuse path + common-prefix path) now skips bases that
    violate `in_same_island`, and the common-prefix path prefers a base whose island
    set is a superset (`delta_cmp < 0`) even when it isn't the largest object;
  - `pack.islandcore` layering: core-island blobs sort first in the pack write order.
