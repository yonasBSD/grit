# t3206-range-diff — ticket 9d753c

## Summary

Ticket reported subtest 46 "ranges with pathspecs" failing:
`git range-diff topic...mode-only-change -- other-file` produced 0 lines instead
of the expected 2-line range-diff limited to commits touching `other-file`.

## Findings

On a fresh build + run the file is **fully passing: 48/48**. The underlying bug
was already fixed by prior commits on the workspace branch:

- `6c88d29db fix: include pathspec mode-only range-diff commits`
- `4cb5790c6 fix: emit gitlink hunks in log patches`
- `e0dcb7701 fix: make t3206 range-diff pass`

The pathspec is plumbed through `parse_invocation` in
`grit/src/commands/range_diff.rs` (the `-- <pathspec>` suffix is collected into
`log_extra` and appended to the underlying `grit log -p` invocation that builds
each side's patch list), so the log/diff that constructs the per-side commit list
is correctly path-limited.
