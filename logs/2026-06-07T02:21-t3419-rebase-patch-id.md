# t3419-rebase-patch-id.sh — fix mode-only change dropped during rebase

Ticket: 7c9ac4 (created fresh; no prior open/closed ticket existed).

## Status
- Before: 7/8 (subtest 8 "do not drop patch modechange" failing).
- After: 8/8 fully passing.

## Symptom
`git checkout modeother^{} && git rebase other` dropped the executable-bit
change. HEAD ended at `other`; `git diff HEAD~` showed the full content diff
instead of the expected mode-only diff:

```
diff --git a/file b/file
old mode 100644
new mode 100755
```

## Diagnosis
`modeother` = `other` + a top commit that is `chmod +x file` amended on top of the
same scrambled content. Rebasing `modeother` onto `other` replays exactly that
one commit. Patch-id cherry detection correctly did NOT drop it (mode change makes
the patch-id differ — verified via `rev-list --cherry-mark` and `patch-id`).

The commit was instead dropped by the rebase's empty-pick detection
(`merged_tree_oid == head_tree_oid`, grit/src/commands/rebase.rs ~8427). The
3-way merge produced `file` with mode 100644 instead of 100755, so the merged
tree equaled the upstream tree and the pick was treated as empty.

Three-way merge inputs (cherry-pick semantics): base = picked commit's parent
(file 100644, content A), ours = `other` (file 100644, content B), theirs =
picked (file 100755, content B). Content is identical on both sides; only theirs
changed the mode.

Root cause: `try_content_merge_ext` in grit/src/commands/merge.rs computed the
clean-merge result mode as `let mode = ours.mode;` unconditionally. Git's
`merge_3way` (merge-ort.c lines ~2201): if `a->mode == b->mode || a->mode ==
o->mode` then `result->mode = b->mode` (take theirs); else keep ours. Here
ours==base, so the merged mode must be theirs' (100755).

## Fix
grit/src/commands/merge.rs, `try_content_merge_ext`, clean-merge branch:

```rust
let mode = if ours.mode == theirs.mode || ours.mode == base.mode {
    theirs.mode
} else {
    ours.mode
};
```

This is the single load-bearing change. Explicit-favor and binary branches
already pick the correct side's mode and were left untouched.
