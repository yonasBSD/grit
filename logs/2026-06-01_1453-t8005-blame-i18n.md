# t8005-blame-i18n

- Reproduced the reported state from `/tmp/t8005-blame-i18n.verbose`: setup passed, but blame incremental emitted replacement-character numeric entities for the older non-UTF-8 author/summary lines.
- Root cause: the panic workaround changed top-level argv collection to `args_os().to_string_lossy()`, which avoided invalid-UTF-8 panics but corrupted EUC-JP/SJIS `--author` and `-m` bytes before `git commit` encoded the commit object.
- Fix: hydrate raw Unix argv bytes for `git commit --author` and `-m`/`--message`, then decode those bytes with the active `i18n.commitencoding` before existing commit serialization.
- Verification: `cargo build --release -p grit-cli` passed; `./scripts/run-tests.sh t8005-blame-i18n.sh` passed 5/5; `cargo check -p grit-cli` and `cargo test -p grit-lib --lib` passed with the existing warning backlog.
