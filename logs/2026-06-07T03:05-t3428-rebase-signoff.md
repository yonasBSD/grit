# t3428-rebase-signoff — MOP-UP ROUND 1 (ticket 99a747)

Fresh re-run: 5/7. Failing subtests 6 and 7.

## Subtest 7 — `rebase -i --signoff` edit-then-amend with staged changes

Repro (clean HOME, no `core.editor`): after `rebase -i` stops at an `edit`, staging a
new file (`git add a`) and running `git rebase --continue` died with
`error: you have staged changes in your working tree` instead of folding the staged
change into the edited commit and continuing.

Root cause: `do_continue()` compared the current `HEAD` against `stopped-sha`
(the *original* commit OID) to decide whether the user had made their own commit
after `edit`. But after an `edit`, `HEAD` is the *replayed* commit, whose OID always
differs from the original `stopped-sha`, so the check always reported "user made
own commit" and refused to amend.

Git's `commit_staged_changes` compares `HEAD` against the OID written by
`intend_to_amend()` — the replayed HEAD at the edit stop, stored in
`rebase-merge/amend`. Grit never wrote that file.

Fix (grit/src/commands/rebase.rs):
- In the `RebaseReplayStep::Edit` clean-stop path, write the replayed HEAD (`new_oid`)
  to `rb_dir/amend` (mirrors `intend_to_amend()`).
- In `do_continue`, read `head_at_stop` from `rb_dir/amend` (fall back to the original
  OID when absent) and compute `user_made_own_commit = edit_continue && head_oid != head_at_stop`.
  Use that flag for the three comparison sites (early replay, message template, reflog).
- Remove the `amend` file on the amend/early-return paths.

## Subtest 6 — `rebase -m --signoff --continue` missing `# Conflicts:` block

The continue editor (`GIT_EDITOR="sed -n /Conflicts:/,/^$/p >actual"`) saw an empty
message — no `# Conflicts:` block.

Root cause: on a merge-backend pick conflict, grit writes the bare message to
`rebase-merge/message` and the message-plus-`# Conflicts:`-footer to `MERGE_MSG`.
`read_rebase_continue_message` preferred the bare `message` file, so the editor
template lacked the conflicts hint. Git's `git commit` during `--continue` uses
`MERGE_MSG` as the template.

Fix: `read_rebase_continue_message` now prefers `MERGE_MSG` (with the conflicts hint)
over the bare `message` file. The hint is comment lines, stripped by `commit.cleanup`,
so the committed message is unchanged; only the editor template gains the block.
