## Fix this session: pack.preferBitmapTips valueless-key error + Total-line gating

Tests 74 / 150 / 231 ("pack.preferBitmapTips", the second occurrence in each of the 3
`test_bitmap_cases` runs) only require config/output behavior, NOT a real bitmap reader:

```
[pack]
	preferBitmapTips        # value-less
```
then `GIT_TEST_NAME_HASH_VERSION=1 git repack -adb 2>actual` must produce exactly:
```
error: missing value for 'pack.preferbitmaptips'
```

Two grit bugs blocked this:

1. `repack -adb`'s pack-objects file-output path always printed
   `Total N (delta...), reused...` on `!quiet`. Git only prints that summary when
   progress is enabled (`builtin/pack-objects.c`: `progress = isatty(2)`, summary guarded
   by `if (progress)`). Since the test redirects stderr to a file (not a TTY), Git
   suppresses it. Fixed: gate the **file-output** Total line on
   `!quiet && (stderr.is_terminal() || --progress || --all-progress-implied || GIT_PROGRESS_DELAY)`.
   The `--stdout` Total line (server/fetch path, t5500) is unchanged.

2. grit never read `pack.preferbitmaptips`, so it never emitted the nonbool error.
   Git's bitmap writer reads it via `repo_config_get_string_multi`
   (`pack-bitmap.c:bitmap_preferred_tips`); a value-less key fails with
   `config_error_nonbool` -> `error: missing value for '<key>'`. Added: in the
   bitmap-writing path of pack-objects (where the `.bitmap` placeholder is written),
   check `config.get_all_raw("pack.preferBitmapTips")` for any `None` and emit the error.

File changed: `grit/src/commands/pack_objects.rs`.
