## Root cause (confirmed independently)
The failure is a **harness porting bug**, not a grit bug.

Subtest 12 body (tests/t1050-large.sh:72):
```sh
for l in .git/objects/$OIDPATH_REGEX
do
    test_path_is_missing "$l" || return 1
done
```

`OIDPATH_REGEX` is **unset** in the ported `tests/test-lib.sh`. Upstream sets it at
`git/t/test-lib.sh:1611`:
```sh
OIDPATH_REGEX=$(test_oid_to_path $ZERO_OID | sed -e 's/0/[0-9a-f]/g')
```
The ported `tests/test-lib.sh` sets `OID_REGEX`, `_x05`, `_x35`, `_x40`, `ZERO_OID`,
`EMPTY_TREE`, `EMPTY_BLOB` (lines 560-568) but **omits the `OIDPATH_REGEX` line**.
`test_oid_to_path` itself IS present (`tests/test-lib-functions.sh:1810`), so only the
single assignment line is missing.

With `OIDPATH_REGEX` empty, `.git/objects/$OIDPATH_REGEX` globs to the bare existing
directory `.git/objects/`, so `test_path_is_missing` fails unconditionally â€” regardless
of grit. With `OIDPATH_REGEX` set to `[2-hex]/[38-hex]`, the glob matches no loose
objects (there are none), so the loop body never runs and the subtest passes.
