# t7005-editor — editor default-fallback fix

Ticket: 66e838 (# t7005-editor: 10/12 -> 12/12)

## Symptom
Subtests "Using vi" (5) and "Using editors with overrides" (10) failed on the
`i=$vi` iteration. With `TERM=vt100`, `vi` on PATH, and no
GIT_EDITOR/core.editor/VISUAL/EDITOR configured, `git commit --amend` should
launch the compiled DEFAULT_EDITOR `vi`. grit instead errored:
  error: Terminal is dumb, but EDITOR unset

## Root cause
`grit/src/editor.rs::resolve_commit_launch_editor` had an extra gate after the
dumb-terminal check:

```rust
if !std::io::stdin().is_terminal() {
    return None;
}
```

Upstream `git/editor.c::git_editor()` has NO such `isatty` check. It returns
NULL only when `terminal_is_dumb && !editor`; otherwise it always falls back to
`DEFAULT_EDITOR` ("vi"). In the test harness `git commit --amend` runs with
stdin not a TTY, so grit hit this gate, returned None, and the commit launch
path emitted "Terminal is dumb, but EDITOR unset".

## Fix
Removed the `!stdin().is_terminal()` gate (and the now-unused
`std::io::IsTerminal` import). When the terminal is not dumb and nothing is
configured, fall back to "vi" unconditionally, matching `git_editor()`.

Affects the commit/add_patch/rebase launch paths (all call
`resolve_commit_launch_editor`); all now match upstream parity.

## Result
./scripts/run-tests.sh t7005-editor.sh => 12/12 (was 10/12).
cargo test -p grit-lib --lib => 276 passed; only the 2 known pre-existing
ignore::gitignore_glob_tests failures remain (unrelated to editor).
