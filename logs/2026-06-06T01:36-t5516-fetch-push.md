## Root cause #1 (massive): `git init --template=` created `hooks/`, `info/`, `description`

The harness `mk_empty` helper does:
```sh
git init --template= "$repo_name" &&
mkdir "$repo_name"/.git/hooks &&        # FAILS when grit already made hooks/
git -C "$repo_name" config receive.denyCurrentBranch warn
```

Upstream Git's `create_default_files` (git/setup.c) only creates `objects/`, `refs/`
and friends; `hooks/`, `info/`, and `description` are *template content* installed by
`copy_templates`. With `--template=` (empty), no template is copied, so none exist and
the subsequent `mkdir .git/hooks` succeeds.

grit's `grit/src/commands/init.rs::create_git_dir` unconditionally created `hooks` and
`description`, and created `info/exclude` via its default-template branch. Fix: gate
`hooks/` creation and the `description` write behind `has_template_content`
(`template_dir.is_some() || !skip_default_templates`). `info/exclude` was already gated by
`skip_default_templates`.

Result: 6 -> 73/124.

## Root cause #2: push dst DWIM against remote refs (`count_refspec_match`)

`git push remote main:origin/main` (dst `origin/main` is a short name that should match the
existing `refs/remotes/origin/main`) bailed "destination is not a full refname". grit's
`resolve_destination_ref_for_push` rejected any slash-containing non-`refs/` dst.

Implemented Git's `count_refspec_match` (remote.c) weak/strong matching plus `guess_ref`
fallback in push.rs: match the short dst against the remote's advertised refs; one strong
match (or single weak) is unambiguous. For colon-less pushes, the dst value is the *full
source ref name* (so a remote with both refs/heads/frotz and refs/tags/frotz is not ambiguous
for a branch source). Fixed tests 26, 32, 33 with no regressions. 73 -> 74/124.

## Root cause #3: `@` source alias and config-push HEAD/@ resolution

`git push remote @` (and `+@`, `@:dst`) failed: grit never rewrote `@`->HEAD. Also
`remote.<name>.push = HEAD`/`@` in config went through a normalize_ref(src) path that produced
`refs/heads/HEAD`. Fixed both: rewrite `@`->HEAD in CLI parse_refspec output; route the
config-push branch through resolve_push_src_for_refspec (handles HEAD/@/oid/DWIM) with the
same effective-dst HEAD resolution. Fixed 45,46,47,48,49,50. 74 -> 80/124.
