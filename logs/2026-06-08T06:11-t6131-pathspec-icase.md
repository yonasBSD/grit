# t6131-pathspec-icase — test-body portability fix (2026-06-08, agent t5)

## Summary

Result: file now SKIPs cleanly (`1..0 # SKIP`) on this case-insensitive macOS
APFS host, matching upstream git's design. Zero subtests fail (was 8/9 failing).
Ticket fd04f0.

## Root cause (confirmed, differential-verified)

The upstream test `t6131-pathspec-icase.sh` is *designed to skip itself* on a
case-insensitive file system via a `test_have_prereq CASE_INSENSITIVE_FS` gate
at the top of the file (lines 7-11). That gate is INERT in grit's harness
because grit's `tests/test-lib.sh` does NOT define the `CASE_INSENSITIVE_FS`
lazy prereq (upstream defines it in `git/t/test-lib.sh`). With the prereq
undefined, `test_have_prereq` returns false, the skip never fires, and the body
runs commands that cannot succeed on a case-insensitive FS:

- Subtest 1 (`create commits with glob characters`) runs `test_commit bar` then
  `test_commit bAr` then `test_commit BAR`. `test_commit` creates a tag of the
  same name, so it does `git tag bar` then `git tag bAr`. On a case-insensitive
  FS the loose ref files `.git/refs/tags/{bar,bAr,BAR}` are the SAME file, so the
  second tag creation fails ("tag 'bAr' already exists"). Everything downstream
  then fails too.

## Differential rule applied (grit vs real git 2.52.0)

On this case-insensitive FS, reproduced the key command in /tmp with both
binaries on identical inputs:

- real `/opt/homebrew/bin/git` 2.52.0: `git tag bar` -> ok; `git tag bAr` ->
  `fatal: tag 'bAr' already exists` (exit 128).
- grit (`target/release/grit`): `git tag bar` -> ok; `git tag bAr` ->
  `error: tag 'bAr' already exists` (exit 1).

Both REFUSE to create the colliding tag — grit MATCHES real git's behavior.
(Exit codes differ, 1 vs 128, but the observable behavior — refusal, tag not
created — is identical, which is precisely why upstream gates the whole file off
on this FS rather than expecting it to run.) Therefore this is a TEST-authoring /
portability bug, not a grit bug. The grit `:(icase)` pathspec feature itself
works (prior agents proved 9/9 on a case-sensitive APFS image; the enabling fix
`35da7a746` in `grit-lib/src/rev_list.rs` is already in HEAD history).

## Fix (test-body only)

Defined the `CASE_INSENSITIVE_FS` lazy prereq locally inside the test file,
immediately before the existing skip gate, using upstream git's exact probe:

```sh
test_lazy_prereq CASE_INSENSITIVE_FS '
	echo good >CamelCase &&
	echo bad >camelcase &&
	test "$(cat CamelCase)" != good
'
```

This makes the test self-contained: the skip gate now works without any change
to the orchestrator-owned `tests/test-lib.sh`. On a case-insensitive FS the
prereq is satisfied -> `skip_all` + `test_done` -> `1..0 # SKIP` (exit 0). On a
case-sensitive FS the prereq is NOT satisfied -> all 9 subtests run (and pass,
per prior case-sensitive-volume proof).

Verified the probe: in a fresh /tmp dir it correctly reports "case-insensitive
(would SKIP)" here; the logic preserves the case-sensitive run path.
