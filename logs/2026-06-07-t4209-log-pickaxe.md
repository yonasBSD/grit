# t4209-log-pickaxe — author/committer case-sensitivity

Ticket: 7abeb2

## Problem
`git log --author=<pat>` (and `--committer`) matched case-INSENSITIVELY by default
in grit. Real git matches case-SENSITIVELY unless `-i`/`--regexp-ignore-case` is
given. Failing subtest: `not ok 13 - log --author (nomatch)` — `--author=person`
must NOT match author "Another Person".

## Root cause
`grit/src/commands/log.rs::ident_pattern_ignore_case()` unconditionally returned
`true`, hard-forcing case-insensitive matching for the `--author`/`--committer`
header greps. In git, `revs->grep_filter.ignore_case` is a SINGLE flag shared by
all grep patterns (header included): `grep.c` honors `opt->ignore_case` uniformly,
and `revision.c:2670` sets it only for `-i`/`--regexp-ignore-case`. So header
greps are case-sensitive by default, exactly like `--grep`.

## Fix
`ident_pattern_ignore_case(regexp_ignore_case)` now returns `regexp_ignore_case`
(was hard `true`). Updated the two explanatory comments to match.

## Verification
- Real git 2.39.5: `--author=person` => empty; `--author=Person` => hash;
  `-i --author=person` => hash. Confirms case-sensitive default + `-i` folds case.
- `./scripts/run-tests.sh t4209-log-pickaxe.sh` => 48/48 (was 47/48).
