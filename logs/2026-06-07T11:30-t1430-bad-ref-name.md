## Fix: subtest 33 (update-ref --stdin -z fails create with bad ref name)
Expected stderr: `fatal: invalid ref format: ~a ` (note the TRAILING SPACE).
grit produced `fatal: invalid ref format: ~a` (space trimmed).

Root cause: `validate_batch_refname()` in grit/src/commands/update_ref.rs hardcoded
whitespace-tokenization of the refname even under `-z`. Per `parse_refname()` in
git/builtin/update-ref.c, with `-z` (line_termination == '\0') the refname is
"everything up to the next NUL", preserving trailing whitespace; only without `-z`
is it the first whitespace-delimited C-quoted arg. For `create ~a \0refs/heads/main\0`
the bad name is therefore `~a ` (with the trailing space).

Change: pass `null_terminated` through to `extract_raw_refname(rest, null_terminated)`
instead of forcing `false`. The `-z` branch of `extract_raw_refname` already returns
the whole field. Verified subtests 34 and 35 (which have no trailing space in the
refname field) still pass.

Result: 32→33; subtests 33,34,35 all pass.

## Blocker: subtests 21-27 (unfixable on grit side) — independently re-confirmed

`test_ref_missing`. The COPIED tests/test-lib.sh (line 1500) overrides these helpers:
    test_ref_exists () { git rev-parse --verify -q "$1" >/dev/null 2>&1; }
This override is sourced AFTER tests/test-lib-functions.sh (sourced at line 102),
which defines the upstream helper using `show-ref --exists` (exit 0 for broken refs).
The override therefore wins.

Reproduced each precondition manually:
- dangling symref badname -> broken...ref: `rev-parse --verify -q` exits 1, `show-ref --exists` exits 0.
- broken...ref (direct ref, valid OID, bad NAME): `rev-parse --verify -q` exits 1, `show-ref --exists` exits 0.
- broken...symref -> refs/heads/main (valid target, bad NAME): `rev-parse --verify -q` exits 1, `show-ref --exists` exits 0.

Verified against git C source that grit matches real git exactly:
- git/builtin/update-ref.c parse_refname: `-z` keeps trailing whitespace (subtest 33 fix basis).
- git/refs.c refs_resolve_ref_unsafe lines 2093-2096 & 2152-2157: a refname failing
  check_refname_format returns NULL unless RESOLVE_REF_ALLOW_BAD_NAME is set (rev-parse
  --verify does NOT set it); and even when set, the OID is cleared and REF_ISBROKEN is
  marked (lines 2140-2143), so it never resolves to an object. Hence real git's
  `rev-parse --verify -q` ALSO exits non-zero on these broken/dangling refs.

The actual delete operations all succeed in grit (update-ref --no-deref -d, branch -d
produce correct output / leave the ref missing). Only the divergent local helper
precondition fails. Making 21-27 pass would require either editing tests/test-lib.sh
(forbidden) or diverging grit's rev-parse --verify from real git (wrong). Documented;
ticket stays open as the remaining 7 are a known test-harness portability divergence.
