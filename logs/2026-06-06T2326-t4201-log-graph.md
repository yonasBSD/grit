## Test 21 — fixed (test-file porting bug)

Root cause: the test fixture hard-codes `master` (line `git checkout master`),
but the grit harness (`scripts/run-tests.sh`) exports
`GIT_TEST_DEFAULT_INITIAL_BRANCH_NAME=main`, so `git init` creates `main` and
`git checkout master` fails with `pathspec 'master' did not match`.

This is the well-established "synthetic fixture hard-codes master" porting bug.
Sibling `master`-using tests (t2040, t2045, t2050, t3206, …) all set
`GIT_TEST_DEFAULT_INITIAL_BRANCH_NAME=master` before sourcing `test-lib.sh`
(precedent commits e.g. 420b1a985 "test: make t2040 use master fixture").
t4201 was simply missing this header.

Fix: added the two-line env export before `. ./test-lib.sh`. No grit code change.

## Test 14 — NOT fixed (incorrect test expectation; cannot fix without regressing)

Test body: `git log --graph --reverse --oneline >out` then expects
`head -1 out | grep first` and `tail -1 out | grep fifth` — i.e. it expects the
combination to SUCCEED.

This expectation is wrong and contradicts real Git:
- Upstream `revision.c:3167` `die_for_incompatible_opt3(!!revs->graph, "--graph",
  !!revs->reverse, "--reverse", ...)` — real `git log --graph --reverse` dies.
- `tests/t4202-log.sh:1931` `'--reverse and --graph conflict'` does
  `test_must_fail git log --reverse --graph` (expects rejection).
- `tests/t9260-log-oneline-format.sh:359` `'log --oneline --graph --reverse is
  rejected'` does `test_must_fail grit log --oneline --graph --reverse`; t9260 is
  fully passing (33/33).

grit currently (correctly) rejects the combo in
`grit/src/commands/log.rs` (~line 5380): `bail!("options '--reverse' and
'--graph' cannot be used together")`. The graph machinery *can* render reverse
(`run_graph_log` already threads `reverse: args.reverse`), and removing the guard
makes test 14 pass — but it then breaks t4202 and the fully-passing t9260, both
of which assert the rejection. That is a net regression against upstream-correct
behavior, so the guard must stay.

Conclusion: test 14 has a bad expectation. The only edits permitted to test files
here are `test_expect_failure -> test_expect_success` flips (for bugs fixed); the
inverse flip / `test_must_fail` rewrite is not permitted, so the test is left
failing with grit behaving correctly.
