# t1600-index.sh — duplicate GIT_INDEX_VERSION warning

Ticket: bbb708 (subtests 2 & 3 failing, 5/7 -> 7/7)

## Symptom

Subtests 2 (`bogus GIT_INDEX_VERSION issues warning`) and 3 (`out of bounds
GIT_INDEX_VERSION issues warning`) expected the two-line warning

```
warning: GIT_INDEX_VERSION set, but the value is invalid.
Using version 3
```

exactly once, but grit emitted it twice on `git add` when no index existed yet.

## Root cause

`grit/src/commands/add.rs` built the working index twice for the fresh-index
case:

- `raw_index = Index::new()` (sparse-index baseline) — `Index::new()` calls
  `get_index_format_from_env()`, which warns and returns the fallback version.
- `index = Index::new_from_config(&config)` — `new_from_config` also calls
  `get_index_format_from_env()`, warning a second time.

C Git resolves the format once via `get_index_format_default()`
(read-cache.c:1629), called a single time when a fresh in-memory index is
initialized, so the warning is printed once.

## Fix

- Added `Index::empty(version)` to `grit-lib/src/index.rs`: a constructor that
  builds an empty index at a caller-supplied version and never reads
  `GIT_INDEX_VERSION` (so it never warns).
- Reordered `add.rs` to build `index` first (single env resolution via
  `new_from_config`), then derive `raw_index` from `Index::empty(index.version)`.
  The `raw_index` is only an empty sparse-index diff baseline, so its version is
  irrelevant — it just needs to not re-resolve/re-warn.

## Result

- `./scripts/run-tests.sh t1600-index.sh` -> 7/7.
- `cargo test -p grit-lib --lib` -> 276 passed, only the 2 known pre-existing
  `ignore::gitignore_glob_tests` failures remain (unrelated).
- No new clippy warnings in changed files.
