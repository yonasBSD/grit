# t5607-clone-bundle — work log

Ticket 216d76. Fresh run: 10/16 pass. Failing subtests: 2, 3, 8, 10, 12, 15.

## Diagnoses (all reproduced)

- **2 "verify" needs a worktree**: `grit bundle verify` succeeds outside a repo. Git
  errors `need a repository to verify a bundle` and exits 1 (builtin/bundle.c
  cmd_bundle_verify checks `startup_info->have_repository`). Fix: `run_verify` must
  require a repo.

- **3 annotated tags excluded by rev-list options**: ls-remote on a bundle prints refs
  in bundle-header order, but git's bundle transport `get_refs_from_bundle` prepends
  each ref (`ref->next = result`) producing REVERSED order, and ls-remote does no default
  sort. Fix: reverse bundle ref output in `run_bundle_ls_remote` (ls_remote.rs).

- **8 ridiculously long subject in boundary**: `git fetch <bundle.bdl>` (plain filename,
  no slash) errors "does not appear to be a git repository". fetch.rs dispatch only
  treats `.`/`/`-containing or directory names as path remotes; a bare bundle filename
  falls through to config lookup and bails. Fix: treat an existing file arg as a path
  remote (url_override).

- **10 failed bundle creation does not leave cruft**: `git bundle create fail.bundle
  main..main` succeeds (empty). Git's write_bundle_refs skips refs whose commit was not
  SHOWN by the rev walk; with no commits shown ref_count==0 -> "Refusing to create empty
  bundle". Fix: in run_create, drop refs whose commit target is not in listed.commits.

- **12 clone bundle fsckObjects**: clone-from-bundle never fsck-checks objects. Need:
  fetch.fsckObjects / transfer.fsckObjects (fetch overrides transfer), fetch.fsck.<id>
  severity. Use grit_lib::fsck_standalone::fsck_object; fail with "missingEmail".

- **15 v3 rejects unknown capabilities**: header parser fails with "truncated bundle
  header" because the test file has no pack/blank line. Git's read_bundle_header_fd treats
  v3 `@cap` lines: unknown -> `error: unknown capability '<cap>'` and aborts. Fix
  parse_bundle_header to reject unknown `@` capabilities (v3) before requiring blank line.

## Result

All 16/16 subtests pass. Fixes landed:

- bundle.rs: `run_verify` requires a repo ("need a repository to verify a bundle");
  `parse_bundle_header` handles `@object-format=`/`@filter=` in any version and rejects
  unknown `@cap` in v3 ("unknown capability '...'"); `run_create` drops refs whose tip is
  an unshown commit on a plain range (not --all/--max-count/since/until) and errors
  "Refusing to create empty bundle." (no file left behind).
- ls_remote.rs: `run_bundle_ls_remote` prints refs in reversed header order (matching
  git's bundle transport prepend + no default ls-remote sort).
- fetch.rs: a bare existing-file remote arg (e.g. `git fetch foo.bundle`) is treated as a
  path URL (url_override) so the bundle is unbundled instead of bailing.
- clone.rs: bundle clone runs content fsck when fetch/transfer.fsckObjects is set
  (fetch overrides transfer); `fetch.fsck.<id>=ignore` demotes a check; failure prints
  "missingEmail" and exits non-zero.
