# t5545-push-options — gitlink strict-connectivity fix

## Problem
Subtest `5545.4 'push options and submodules'` failed. Pushing a superproject
that contains a submodule reported `remote unpack failed: unpack-objects
abnormal exit`.

Root cause: grit's strict pack-connectivity walk treated gitlink (submodule,
mode `160000`) tree entries as ordinary objects that must be present in the
pack or destination ODB. Submodule commits live in the submodule repository,
not in the superproject pack, so the strict check produced a false positive
`strict: missing object <oid> referenced by tree` and aborted the unpack.

Upstream git skips gitlinks in the connectivity walk
(`git/fsck.c:374`: `if (S_ISGITLINK(entry.mode)) continue;`).

## Fix
File: `grit-lib/src/unpack_objects.rs`
- Added `use crate::index::MODE_GITLINK;`.
- In `strict_verify_packed_references_map` (receive path / `unpack-objects
  --strict`), skip tree entries whose `mode == MODE_GITLINK` before the
  resolve check.
- Same skip in the public `strict_verify_packed_references` (`index-pack
  --strict`), keeping the two strict paths consistent so large-pack receives
  via index-pack do not regress.
- Added unit test `test_strict_skips_gitlink_tree_entries`: a tree with a
  gitlink entry pointing at an absent submodule oid passes strict
  verification, while a regular-file entry pointing at an absent blob still
  errors (guards the relaxation stays narrow).

No changes needed in push.rs, receive_ingest.rs, pack_objects.rs, or
push_submodules.rs — the pack generation was already correct; only grit's
strict checker was wrong.
