### 1. Implement `git add -i` main menu (new file `grit/src/commands/add_interactive.rs`)
Mirrors `git/add-interactive.c`: status / update / revert / add untracked / patch / diff
sub-commands plus the `What now>` loop and `list_and_choose` file selection.

- `get_modified_files()` computes per-file `+add/-del` for index-vs-HEAD (staged) and
  worktree-vs-index (unstaged) using `diff_index_to_tree` / `diff_index_to_worktree` and
  `count_changes_with_algorithm`.
- Key bug found: for worktree diffs `new_oid` is a hash of an on-disk file NOT written to the
  ODB, so reading the blob returns empty (gave `+0/-1` instead of `+2/-0`). Fixed by reading
  the worktree file directly for worktree-side diffs (`read_new_side(from_worktree=true)`).
- `revert` reuses `reset::tree_to_flat_entries` (made `pub(crate)`) to reset index entries to HEAD.
- `patch` delegates to `add_patch::run_add_patch_with_reader` (new variant) so the SAME buffered
  stdin reader is threaded through — otherwise the inner `io::stdin().lock()` lost buffered input
  and HUNG (this hang caused a harness-wide FATAL that zeroed the whole run).
- `diff` renders the cached patch via `diff::write_patch_from_pairs`.

Wired into `add::run` replacing the not-implemented warning. Made `resolve_patch_context` /
`resolve_patch_interhunk` `pub(crate)`.

### Remaining root-causes for the mop-up agent (NOT yet fixed)
1. **add -p hunk navigation (`j k J K g /`)** — tests 45-49,112,114-117 (~9 tests). grit's hunk
   loop uses a simple auto-incrementing `hunk_cursor` + `accepted: Vec<bool>`. Git tracks a
   per-hunk *decision* (UNDECIDED/USE/SKIP), shows `(N/M)` that does NOT auto-advance past
   undecided, annotates `(was: y)`, and `j/J/k/K` move next/prev (J/K roll over), `g`/`g N`/`g1`
   goes to a hunk, `/pattern` searches. Implementing this is a real rewrite of the `'hunk_loop`
   in grit/src/commands/add_patch.rs — biggest self-contained win left but high regression risk
   to the ~30 passing add -p tests. The nav letters currently fall to the "Unknown command" arm.
2. **Mode-change as hunk 0** — test 33. grit shows mode change as a standalone `(1/1) Stage mode
   change` pre-prompt (add_patch.rs ~line 299). Git makes it hunk 1 of N: `(1/2) Stage mode
   change [...,k,K,j,J,g,/,p,P,?]` then `(2/2) Stage this hunk`. Folding it into the hunk loop
   risks regressing the passing tests 30/31/32.
3. **git diff `* Unmerged path X`** — test 56. `git diff`/`diff --cached` must print
   `* Unmerged path <name>` for stage-1/2/3 entries (in grit/src/commands/diff.rs, a contended
   shared file — coordinate). add -p already correctly skips unmerged.
4. **git diff empty-file deletion header** — test 42 expects `git diff --cached` to OMIT the
   `--- a/empty` / `+++ /dev/null` lines when the deleted file is empty (just the
   `diff --git`/`deleted file mode`/`index` header). grit emits the `---`/`+++` lines. diff.rs.
5. **-U/--inter-hunk-context across patch commands** — tests 91-110. `checkout -p`/`restore -p`/
   `reset -p`/`stash -p`/`commit -p` ignore `-U`/`--inter-hunk-context` (checkout_patch in
   checkout.rs takes no context arg; add -p has `let _ = opts.inter_hunk_context;`). The
   inter-hunk-context *merging* of adjacent hunks is unimplemented everywhere.
6. **Colorized add -p / add -i** — tests 58-67, 85, 99/101/.... `add -i`/`add -p` never emit
