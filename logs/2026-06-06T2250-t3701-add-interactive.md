### 1. Implement `git add -i` main menu (new file `grit/src/commands/add_interactive.rs`)
Mirrors `git/add-interactive.c`: status / update / revert / add untracked / patch / diff
sub-commands plus the `What now>` loop and `list_and_choose` file selection.

- `get_modified_files()` computes per-file `+add/-del` for index-vs-HEAD (staged) and
  worktree-vs-index (unstaged) using `diff_index_to_tree` / `diff_index_to_worktree` and
  `count_changes_with_algorithm`.
- Key bug found: for worktree diffs `new_oid` is a hash of an on-disk file NOT written to the
  ODB, so reading the blob returns empty (gave `+0/-1` instead of `+2/-0`). Fixed by reading
  the worktree file directly for worktree-side diffs (`read_new_side(from_worktree=true)`).
- `revert` reuses `reset::tree_to_flat_entries` (made `pub(crate)`) to reset index entries to HEAD.
- `patch` delegates to `add_patch::run_add_patch_with_reader` (new variant) so the SAME buffered
  stdin reader is threaded through — otherwise the inner `io::stdin().lock()` lost buffered input
  and HUNG (this hang caused a harness-wide FATAL that zeroed the whole run).
- `diff` renders the cached patch via `diff::write_patch_from_pairs`.

Wired into `add::run` replacing the not-implemented warning. Made `resolve_patch_context` /
`resolve_patch_interhunk` `pub(crate)`.
