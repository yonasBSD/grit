## Independent re-verification of the R1 blocker (bundle-uri over HTTP)
The prior agent (Comment 2) concluded #113/#114/#115 are UN-PASSABLE in this environment and
NOT a grit bug. I re-verified every link of that chain from scratch rather than trusting it:

1. **REAL_GIT selection** (`tests/lib-httpd.sh` lines 29-55): candidate loop tries
   `/usr/bin/git` first → exists → wins. `/usr/bin/git` = **Apple Git 2.39.5 (Apple Git-154)**.
   Homebrew git **2.52.0** lives at `/opt/homebrew/bin/git` and is never in the candidate list,
   so it is never selected.

2. **Advertisement source**: for these 3 tests `BUNDLE_URI_PROTOCOL` is unset (they pass only
   `-c protocol.version=2 -c transfer.bundleURI=true`). The generated smart-HTTP
   `git-upload-pack` wrapper (lib-httpd.sh lines 242-255) routes the `--advertise-refs`
   capability listing to **REAL_UP** (real git's upload-pack), i.e. Apple Git.

3. **Apple Git 2.39.5 does NOT advertise `bundle-uri`** even with
   `uploadpack.advertiseBundleURIs=true` + `bundle.*` configured. Verified directly:
   `GIT_PROTOCOL=version=2 /usr/bin/git upload-pack --advertise-refs repo.git` →
   `version 2 / agent / ls-refs / fetch / server-option / object-format / object-info` — NO
   bundle-uri. (The capability was added upstream in git 2.41+.) Contrast verified: homebrew
   2.52.0 on the same repo DOES emit `000fbundle-uri`.

4. **grit client is correct**: `crate::file_upload_pack_v2::server_advertises_bundle_uri(&caps)`
   gates discovery (callers in `grit/src/http_smart.rs:1577`, `grit/src/fetch_transport.rs:1379`,
   `:2061`). Cap not advertised → grit skips discovery → spawns no `git-remote-https` child →
   the `child_start` trace event the tests grep for never fires → tests fail.

## Conclusion
No correct grit-side fix makes these pass here:
- Forcing grit to fabricate bundle-uri discovery against a server that did not advertise the
  capability would be WRONG (diverges from upstream git behavior, regression risk on other
  bundle-uri tests like t5558/t5732).
- Selecting homebrew git as REAL_GIT requires editing `tests/lib-httpd.sh` — a forbidden harness
  change that risks regressions across all HTTP-dependent test files.

t5601 is grit-complete at **112/115**; the 3 remaining failures are an environment limitation
(REAL_GIT = Apple Git 2.39.5 too old to advertise bundle-uri), not a grit defect. No code change
made this round; no new commit (TOML already at 112/115). Leaving ticket OPEN since not 115/115.
