## Changes

### grit/src/commands/branch.rs
`branch_used_by_other_worktree` now delegates to
`worktree_refs::branch_occupied_by_other_worktree` (ignores the current
worktree) instead of `branch_occupied_any_worktree`. Git only forbids
renaming/deleting/force-updating a branch checked out in ANOTHER worktree;
renaming the branch you are currently on is allowed (HEAD is already updated).
The current-worktree case is still guarded separately at the call sites
(`name == current` checks for create/delete/rename), so no regression.

### grit/src/commands/gc.rs
- Tri-state `effective_detach`: `--detach`=1, `--no-detach`=0, else
  `gc.autodetach` (default true) under `--auto`; non-auto never detaches.
- Auto message: print "Auto packing the repository in background for optimum
  performance." when `effective_detach > 0`, the foreground variant otherwise.
- `quiet_effective` no longer forced quiet purely because `--auto`; a foreground
  auto gc (`--no-detach` / `gc.autodetach=false`) now runs with progress.
- `report_last_gc_error()` mirrors git/builtin/gc.c:791-832: when detaching, a
  recent (mtime >= now - gc.logexpiry, default `1.day.ago`) non-empty `gc.log`
  prints the warning and skips the gc (exit 0). `gc.logexpiry` parsed with
  `grit_lib::git_date::approx::approxidate_careful`.
- `acquire_gc_pid` now returns `Ok(None)` (silent exit 0) when `--auto` and the
  `gc.pid` lock is held by a live process ("be quiet on --auto"); non-auto still
  bails.
- `gc.bigPackThreshold` parsed with `parse_byte_size_with_suffix` so `2g`/`1m`
  are honored instead of silently dropping to 0 (= keep-largest-pack).
- `run_commit_graph_for_gc` takes a `want_progress` flag; passes `--progress`
  when `!quiet && effective_detach <= 0` (foreground auto), matching
  git/builtin/gc.c:1056-1058 (`!opts.quiet && !daemonized`).

### grit/src/commands/repack.rs
The `--filter` follow-up side pack (`write_filtered_pack`) only joins
`new_pack_names` (the LOCAL keep set) when its destination resolves to the
local `objects/pack` dir. With an explicit `--filter-to` pointing at another
repository, the pack lives there, so it is no longer treated as a local pack to
retain — a same-named leftover local pack is now correctly removed by
`repack -d`. New helper `filter_dest_is_local`.

### grit/src/commands/pack_objects.rs
`write_pack_via_stdin_objects` only records the side pack into the local
`grit-extra-packs` when `--filter-to` resolves to the local pack dir (new
helpers `side_pack_is_local` / `paths_refer_to_same_dir`). Defensive: the hot
path for t6500-16 turned out to be the repack follow-up above.

### grit/src/commands/commit_graph.rs + grit-lib/src/commit_graph_write.rs
On the `--reachable` write path, emit
`Collecting referenced commits: <n>, done.` when progress is enabled, matching
git/commit-graph.c:1884-1892. `<n>` = count of distinct commit OIDs that refs
point to directly (after peeling tags) — new lib fn
`count_referenced_commit_tips` (+ `peel_to_commit`). Both this and the existing
"Computing commit graph generation numbers" line now respect delayed-progress:
they display only for a TTY or `GIT_PROGRESS_DELAY=0` (fast ops stay silent
under the default 2s delay), so `gc --auto` with `gc.autodetach=false` (#8) does
not gain spurious progress output. Removed the unconditional sleep.

## #32 — host-portability blocker (not a grit bug)
`tests/test-tool chmtime` uses GNU `stat -c %Y`, which fails on this macOS BSD
host (`stat: illegal option -- c`). It prints "Failed to stat .git/gc.log:
skipping" and leaves the mtime UNCHANGED, so `gc.log` can never be aged past
`gc.logexpiry`. All three gc invocations therefore skip (log stays recent), the
pre-existing extra pack is never consolidated, and `test_line_count = 1 packs`
sees 2. The grit-side logic is correct (the `warning:` line is emitted in every
step). Fixing this would require editing the shared `tests/test-tool` harness
helper, which is out of scope. Documented in the harness-gotchas memory.
