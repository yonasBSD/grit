Grit harness test progress — baseline-normalized

Passing tests as a share of the full suite (denominator = 42,442), with measurement-methodology re-baselines removed.

Solid green = normalized progress: the recorded passing count, fixed to a 42,442 denominator and chain-linked across the two harness re-baselines so no work is gained or lost at a counting change. Dashed gray = as-recorded (fixed denominator only), shown for comparison.

The harness changed how it counts a passing test twice — Apr 14 (37,628→18,625 across the same files) and Jun 1 (~22k→~34k). Assuming those tests were not actually broken/fixed at those instants, each era is rescaled to the current methodology: Apr 2–13 ×0.760, Apr 14–May 31 ×1.536, Jun 1–6 ×1.0. Factors come from the end-of-day counts straddling each boundary (amber lines), so the green curve is continuous there. The result: ~13% (Apr 2) → ~67% (mid-Apr) → steady ~77–80% (May) → 98.2% (Jun 6) → 99.6% (Jun 7) / 99.5% (Jun 8). Apr 1 is omitted (that day counted test files, not tests). Purple bars show commits/day (right axis); the Jun 7–8 progress and all commit counts are taken across every branch (--all), since recent work continued on refs not reachable from this branch’s tip. Source data: docs/test-progress-history.csv (date, passing, percent_raw, percent_normalized).
