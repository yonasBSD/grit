# Domain-rule modules under `grit/src/` (audit, read-only)

## Scope and method
The `grit` binary crate is meant to hold only CLI concerns (argv parsing, env/config reading, subprocess spawning, exit-code mapping, user-facing output). The domain engine lives in `grit-lib`, which already establishes the boundary pattern: pure logic with typed `thiserror` errors and no process-global state (e.g. `grit-lib/src/protocol.rs`, `transport_path.rs`, `receive_pack.rs`, `ls_remote.rs`, `fetch_negotiator.rs`, `upload_filter.rs`).

The `grit/src/commands/*` files are the CLI dispatch layer (argv â†’ output â†’ exit code) and are out of scope by definition. The findings below are the **top-level** `grit/src/*.rs` modules that carry domain rules which belong in `grit-lib`. Several top-level files are already thin CLI shims and are correctly placed (listed at the end so they are not flagged in error).

Note: there is no transport/network module in `grit-lib` today (its `ls-remote` is local-only by design), so most transport findings imply **new** `grit-lib` modules. Where a logical target already exists I name it; otherwise I propose a `grit-lib` module name.

## Transport negotiation (largest cluster)

### `grit/src/fetch_transport.rs` (2077 lines)
- **Domain logic:** Wire-protocol negotiation for fetch/clone over local upload-pack, `git://`, and ssh: v0/v1 ref-advertisement parsing (`read_advertisement`), want/have computation (`collect_wants`, `collect_wants_cli`, `push_want_unique`), the SkippingNegotiator-driven negotiate-and-receive-pack loop (`fetch_via_upload_pack_skipping`, `fetch_upload_pack_negotiate_pack_bytes_with_streams`), pkt-line payload framing (`read_pkt_payload_raw`), shallow-fetch option modeling (`UploadPackShallowOptions`), and a duplicated `parse_git_url`/`GitDaemonUrl` (also in `git_daemon_url.rs`). CLI-only parts: subprocess spawning of `grit upload-pack`, trace2 emission, leading-shell-env parsing for SSH command templates.
- **Target:** new `grit-lib/src/fetch_protocol.rs` (negotiation, want/have, advertisement parsing) building on existing `grit-lib::fetch_negotiator` and `grit-lib::pkt_line`. The wire-parsing/negotiation functions move; only the process-spawning/tracing wrappers stay in the binary.

### `grit/src/http_smart.rs` (1504 lines)
- **Domain logic:** Smart-HTTP protocol-v2 client logic: `ls-refs` response modeling (`LsRefEntry`, `http_ls_refs`), negotiate-only common-commit computation (`http_negotiate_only_common`), the v2 fetch request/response handling and pack extraction (`http_fetch_pack`), and default-branch selection from the advertisement (`remote_default_branch_from_advertised`). CLI-only: trace2 `child_start` dedup, agent header from process state.
- **Target:** new `grit-lib/src/smart_http.rs` (v2 request building, advertisement/response parsing) sitting on `grit-lib::protocol` and `grit-lib::fetch_negotiator`. The HTTP transport itself stays in the binary via the client below.

### `grit/src/http_push_smart.rs` (547 lines)
- **Domain logic:** Pure smart-HTTP wire logic for `git-receive-pack`: advertisement parsing (`read_receive_pack_advertisement`, `ReceivePackAdvertisement`/`ReceivePackAdvertisedRef` with capability/oid lookup), push-command and status-report modeling (`PushCommand`, `PushStatusEntry`, `PushStatusReport`), and the `send_receive_pack_stream` encode/decode over generic `Read`/`Write`. Confirmed no env/Command/socket use â€” it is wire framing only.
- **Target:** existing `grit-lib/src/receive_pack.rs` (extend it with client-side advertisement/status parsing) or a sibling `grit-lib/src/send_pack.rs`.

### `grit/src/file_upload_pack_v2.rs` (779 lines)
- **Domain logic:** Protocol-v2 wire encode/decode for local `file://`: capability-block reading (`read_v2_capability_block`, `read_pkt_lines_until_flush`), capability predicates (`server_advertises_bundle_uri`, `v2_fetch_supports_sideband_all`), v2 fetch/bundle-uri request construction (`write_v2_fetch_request`, `write_bundle_uri_command`, `cap_lines_for_bundle_request`), and section-boundary skipping. CLI-only: spawning `grit upload-pack`, reading `GIT_DEFAULT_HASH`.
- **Target:** new `grit-lib/src/protocol_v2.rs` (the request-building and capability/response parsing functions). Spawn helpers stay in the binary.

### `grit/src/ssh_transport.rs` (876 lines)
- **Domain logic:** SSH remote URL grammar â€” scp-style vs `ssh://`/`git+ssh://` parsing including IPv6 bracket handling and port extraction (`SshUrl`, `parse_ssh_url`, `is_configured_ssh_url`), and the `build_git_ssh_argv` argument construction rules (matching Git `connect.c`). - **Target:** new `grit-lib/src/ssh_url.rs` (URL parsing + argv construction), alongside existing `grit-lib::transport_path`.

### `grit/src/git_daemon_url.rs` (117 lines)
