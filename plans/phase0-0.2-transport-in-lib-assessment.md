# Task 0.2 — Transport Negotiation Code Split: Assessment & Refactor Plan

## Summary

Essentially **all** fetch/push transport negotiation logic currently lives in the **binary crate** (`grit/src/`), not in `grit-lib/`. `grit-lib` contains only a few transport *adjacent* helpers (pkt-line framing, negotiator, config policy, local ls-remote). There is no `grit-lib::transport` module and `Repository` has no `fetch_pack`/`push_pack` methods. The CLI commands call transport logic as `crate::*` free functions.

### What `grit-lib` already has (transport-adjacent, but not negotiation)

- `grit-lib::pkt_line` (210) — wire framing
- `grit-lib::fetch_negotiator` (277) — `SkippingNegotiator` (already consumed by `fetch_transport`)
- `grit-lib::protocol` (220) — capability/protocol constants
- `grit-lib::receive_pack` (51) — config policy only (`receive_unpack_limit`, `pack_object_count`, `should_use_unpack_objects`)
- `grit-lib::ls_remote` (328) — **local** ls-remote over a `git_dir`/`Odb`, NOT network
- `grit-lib::promisor`, `shallow`, `fetch_head`, `url_rewrite`, `transport_path` — small policy/path helpers

## Dependency direction (good news)

- `grit` depends on `grit-lib` one-way (`grit/Cargo.toml: grit-lib = { path = "../grit-lib" }`). No reverse dependency, so moving code *into* `grit-lib` is structurally allowed.
- The transport functions **already take plain `Repository` / `&Path git_dir` / `Odb` arguments** and already import `grit_lib::{repo::Repository, odb::Odb, refs, objects, pkt_line, fetch_negotiator}`. They are not deeply entangled with CLI state.

## Concrete blockers to moving these into `grit-lib`

1. **Error type mismatch.** All transport modules use `anyhow::Result` / `bail!` / `Context` (48–58 occurrences each). `grit-lib` uses its own `thiserror`-based `grit_lib::error::Result` and does **not** depend on `anyhow`. This is the single largest mechanical conversion.
2. **HTTP client.** `http_smart`/`http_push_smart`/`http_client` depend on `ureq` (in `grit/Cargo.toml` only) plus `base64`. `grit-lib` does not pull these. Moving HTTP requires adding the dependency to `grit-lib` (or abstracting it behind a transport trait).
3. **Binary-crate cross-deps that must move or be abstracted:**
   - `crate::version_string` (defined in `grit/src/main.rs:56`) — for the user-agent string.
   - `crate::trace_packet`, `crate::trace2_transfer`, `crate::wire_trace`, `crate::protocol_wire`, `crate::trace_run_command_git_invocation` (`main.rs:276`) — tracing/wire helpers. Some (`protocol_wire`, `wire_trace`) have zero crate/grit_lib refs and move cleanly; the trace2 ones have a few.
   - `crate::grit_exe` (27 lines, no deps) — locates the `grit` executable to spawn sub-processes; trivially movable.
   - `crate::file_upload_pack_v2` and `crate::commands::index_pack` — `fetch_transport` calls these. `index_pack::run` is CLI-shaped (`Args` → `Result<()>`) and only depends on `grit_lib::{repo,config,pack_rev,unpack_objects}`, so it can be refactored to a lib entry.

## Ordered Refactor Plan

1. **Create `grit-lib/src/transport/mod.rs` skeleton.** Add `pub mod transport;` to `grit-lib/src/lib.rs` with submodules `pkt_session`, `upload_pack`, `receive_pack`, `http`, `ssh`, `ext`. No behavior yet.

2. **Move the zero/low-dependency wire helpers first** (lowest risk): `protocol_wire.rs`, `wire_trace.rs`, `grit_exe.rs`, and `trace_packet.rs`/`trace2_transfer.rs` into `grit-lib::transport`. Re-export from `grit/src/main.rs` as `pub use grit_lib::transport::...` so existing `crate::*` call sites keep compiling.

3. **Resolve the `anyhow` boundary.** Decide one of: (a) add `anyhow` to `grit-lib`'s deps (lowest churn), or (b) introduce a `transport::Error` variant in `grit_lib::error` and convert `bail!`/`Context` to it. Recommend (a) initially to unblock the move, then optionally migrate to native errors. This must precede moving `fetch_transport`/`http_smart`.

4. **Move `pack_objects_upload.rs` and a library-shaped `index_pack` entry.** Refactor `commands::index_pack::run` into a `grit_lib::transport`-callable `index_pack_into(repo, pack_bytes, opts)` function; leave the CLI `commands/index_pack.rs` as a thin wrapper. Do the same for `file_upload_pack_v2`.

5. **Move SSH and ext transports** (`ssh_transport.rs`, `ext_transport.rs`) into `grit-lib::transport::{ssh,ext}`. These are mostly URL parsing + `std::process` spawning; only `version_string`/trace refs need rewiring (pass version as a parameter or move `version_string` into lib).

6. **Move `fetch_transport.rs`** into `grit-lib::transport::upload_pack`. Then add the public method **`Repository::fetch_pack(&self, remote: &RemoteSpec, opts: &FetchPackOptions) -> Result<FetchPackOutcome>`** in `grit-lib/src/repo.rs` (the second `impl Repository` block at `repo.rs:1713` is the natural home) that internally dispatches to the moved `upload_pack`/`ssh`/`ext` driver based on URL scheme.

7. **Move the HTTP transport** (`http_client.rs`, `http_smart.rs`, `http_push_smart.rs`) into `grit-lib::transport::http` after adding `ureq`/`base64` to `grit-lib/Cargo.toml`. Fold HTTP into the same `Repository::fetch_pack` dispatch and add **`Repository::push_pack(&self, remote, updates, opts) -> Result<PushOutcome>`** that wraps `send_receive_pack` / `send_receive_pack_stream` / SSH `spawn_git_ssh_receive_pack`. Promote the `pub(crate)` push items to `pub`.

8. **Repoint the CLI.** Replace `crate::fetch_transport::*` / `crate::http_smart::*` / `crate::http_push_smart::*` call sites in `commands/{fetch,push,clone,pull,ls_remote,send_pack}.rs` to call `repo.fetch_pack(...)` / `repo.push_pack(...)`. Keep `commands/*` focused on arg parsing, refspec resolution, ref-transaction updates, and FETCH_HEAD/output formatting; the network session belongs to the library.

9. **Delete the now-unused `crate::*` re-exports** from `grit/src/main.rs` and the moved modules from `grit/src/`. Run the full test suite (the repo has extensive `tests/` plus `bin/test_httpd.rs` for HTTP integration) after each of steps 5–8.

## Recommended initial target API in `grit-lib::transport`

```rust
impl Repository {
    pub fn fetch_pack(&self, remote: &RemoteSpec, opts: &FetchPackOptions) -> Result<FetchPackOutcome>;
    pub fn push_pack(&self, remote: &RemoteSpec, updates: &[RefUpdate], opts: &PushPackOptions) -> Result<PushPackOutcome>;
}
```
where `RemoteSpec` carries the URL (scheme-dispatched to http/ssh/ext/git/local), so the four transport backends become an internal implementation detail of `grit-lib::transport` rather than something the CLI selects.
